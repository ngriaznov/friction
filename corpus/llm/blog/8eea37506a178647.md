**Building a Small Kubernetes Cluster at Home with Three Raspberry Pi 4 Boards**

As a developer and sysadmin, I've always been fascinated by containerization and orchestration tools like Docker and Kubernetes. But with my limited budget and space constraints, I never thought I could set up a full-fledged Kubernetes cluster at home. That was until I stumbled upon the Raspberry Pi 4 board, which made me think: "Why not?!"

In this blog post, I'll walk you through the process of building a small Kubernetes cluster using three Raspberry Pi 4 boards. We'll cover the networking setup, storage challenges, and what I ended up running on it.

**The Hardware**

Before we dive into the setup process, let's take a look at the hardware components:

* Three Raspberry Pi 4 boards (2GB or 4GB models work fine)
* Power supplies for each board
* MicroSD cards for booting each board
* Ethernet cables
* A 5-port Gigabit Ethernet switch or a network router with multiple ports

**Networking Setup**

The first challenge I faced was setting up the networking. With three boards, we need to ensure that they can communicate with each other and access the outside world (for updating packages, etc.). Since the Raspberry Pi 4 boards have built-in Wi-Fi, we could use that as a fallback option. However, for a production-grade setup, I wanted to go with wired Ethernet.

Here's how I connected everything:

1. Connected each board to a switch using an Ethernet cable.
2. Set up a single IP address range (e.g., 192.168.1.x) on the switch and assigned static IPs to each board:
	* Node 1: 192.168.1.10
	* Node 2: 192.168.1.11
	* Node 3: 192.168.1.12 (or whatever range you choose)
3. Set up the switch to route traffic between the nodes and assign a gateway IP address.

**Storage Headaches**

Next, we need to think about storage for our cluster. With only three boards, we can't rely on a single high-capacity storage device. We'll use the onboard SD card slots as a temporary solution, but in a production environment, you'd want more robust storage solutions like SAS or SATA drives.

For this setup, I used 32GB microSD cards for each board, which is sufficient for now. However, keep in mind that these cards will fill up quickly if you're storing large Docker images or data volumes. Consider upgrading to larger SD cards or exploring other storage options (like USB drives) if your use case demands it.

**Installing Kubernetes**

Now that our networking and storage are set up, let's install the Kubernetes cluster on each board:

1. Install Raspbian OS (the official Raspberry Pi operating system) on each microSD card.
2. Install Docker on each board:
	* `sudo apt update`
	* `sudo apt install docker.io`
3. Configure Docker to use a private registry (optional but recommended):
	* `sudo mkdir /etc/docker/registries.d`
	* `sudo echo "https://registry.docker.io" > /etc/docker/registries.d/docker-registry.conf`
4. Install Kubernetes on each board:
	* `curl -sSL https://packages.cloud.google.com/apt/doc/apt-key.gpg | sudo apt-key add -`
	* `echo "deb http://apt.kubernetes.io/ kubernetes-xenial main" > /etc/apt/sources.list.d/kubernetes.list`
	* `sudo apt update`
	* `sudo apt install kubectl`

**Forming the Cluster**

With Kubernetes installed on each board, we can now form our cluster. We'll use a combination of Docker and Kubernetes to create a highly available and scalable environment.

1. On Node 1 (controller node), run:
	* `kubectl apply -f https://raw.githubusercontent.com/kubernetes/website/master/content/en/examples/controller-manager/controllers.yaml`
2. On Node 2 and Node 3, join them to the cluster using the `kubeadm` tool:
	* `sudo kubeadm join <controller-node-IP>:8443`

**What I Ended Up Running on It**

After setting up my Kubernetes cluster, I wanted to test it with a real-world application. I chose to deploy a simple web server (NGINX) and a demo database (MariaDB).

Here's the YAML file for the deployment:
```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: nginx-deployment
spec:
  replicas: 3
  selector:
    matchLabels:
      app: nginx
  template:
    metadata:
      labels:
        app: nginx
    spec:
      containers:
      - name: nginx
        image: nginx:latest
        ports:
        - containerPort: 80

---

apiVersion: v1
kind: Service
metadata:
  name: nginx-service
spec:
  selector:
    app: nginx
  ports:
  - port: 80
    targetPort: 80
```
I applied the YAML file using `kubectl` and watched as my web server (NGINX) and database (MariaDB) deployed across all three nodes.

**Conclusion**

Setting up a small Kubernetes cluster at home with three Raspberry Pi 4 boards was a fun project that taught me about the challenges of networking, storage, and scaling. While this setup is far from production-grade, it demonstrates how containerization and orchestration tools can be used in a DIY environment.

If you're interested in trying out Kubernetes on your own hardware, I encourage you to follow along with this blog post. Remember to keep an eye on your storage usage and consider upgrading to more robust solutions as needed.

Happy building!
