**Title: Creating a Chrome Extension to Block Autoplaying Videos on News Sites**

In recent years, the proliferation of video content on news websites has led to an increase in autoplaying videos, which can be both intrusive and annoying for users. To combat this, I decided to build a small Chrome extension that blocks these autoplaying videos while allowing legitimate video content to play smoothly. This blog post will guide you through the process of creating such an extension, highlighting the necessary manifest permissions and addressing a peculiar bug that only appeared on sites with lazy-loaded iframes.

### Step 1: Setting Up Your Extension

Firstly, you need to set up the basic structure of your Chrome extension. Create a new directory for your project and inside it, create two files: `manifest.json` and `background.js`.

#### Manifest Permissions

The `manifest.json` file is crucial as it tells Chrome what permissions your extension needs and how it should behave. For our video-blocking extension, we need to include the following permissions:

```json
{
  "manifest_version": 3,
  "name": "Video Blocker",
  "version": "1.0",
  "description": "Blocks autoplaying videos on news sites.",
  "permissions": [
    "activeTab",
    "scripting"
  ],
  "background": {
    "service_worker": "background.js"
  },
  "content_scripts": [
    {
      "matches": ["*://*.newswebsite.com/*"],
      "js": ["contentScript.js"]
    }
  ]
}
```

- **activeTab**: This permission allows the extension to interact with the active tab, which is necessary for injecting scripts into specific web pages.
- **scripting**: Required for running background scripts that manipulate page content.

#### Background Script

The `background.js` file will contain logic to intercept and block autoplaying videos. Since we're targeting news sites, we'll use a content script (`contentScript.js`) to handle the blocking on those specific domains.

```javascript
chrome.runtime.onMessage.addListener(
  function(request, sender, sendResponse) {
    if (request.action === 'blockAutoplay') {
      // Logic to block autoplaying videos will go here
    }
  });
```

### Step 2: Implementing Video Blocking

In `contentScript.js`, you'll write the logic to detect and block autoplaying video elements. This involves checking for video players that are set to autoplay and then preventing them from starting.

```javascript
document.addEventListener('DOMContentLoaded', function() {
  const videosToBlock = document.querySelectorAll('[autoplay]');

  videosToBlock.forEach(video => {
    video.setAttribute('aria-busy', 'true'); // Temporarily disable the video
    video.pause(); // Ensure it's not playing
  });
});
```

### Step 3: Dealing with Lazy-Loaded Iframes

One bug I encountered was specific to sites that use lazy-loading for iframes containing videos. These iframes might only be loaded when they come into view, which means our initial script wouldn't detect them until after they've been loaded and attempted to autoplay.

To fix this, we need to listen for the `load` event on these iframes:

```javascript
document.addEventListener('DOMContentLoaded', function() {
  const iframeLoaders = document.querySelectorAll('iframe[src*="video"]');

  iframeLoaders.forEach(iframe => {
    iframe.onload = () => {
      // Re-run our video blocking logic after iframe loads
      blockAutoplayingVideos();
    };
  });
});
```

This ensures that whenever an iframe containing a video is loaded, the script runs again to check for any newly added autoplaying videos.

### Conclusion

Building this Chrome extension taught me valuable lessons about web content manipulation and handling asynchronous loading patterns on modern websites. By carefully defining permissions in the manifest and implementing robust logic to detect and block autoplaying videos, even those loaded through lazy-loaded iframes, we can significantly improve user experience on news sites without sacrificing legitimate video playback.

Remember, extensions are powerful tools that can enhance productivity and privacy online, but they require careful consideration of security and permission management to protect user data. Happy coding!
