**The Great Autoplay Blocker: Building a Small Chrome Extension to Save Your Sanity**

Are you tired of clicking on a news article, only to have some annoying video start blasting its audio without warning? You're not alone! I've had my fair share of frustration with autoplaying videos on news sites, and I decided to take matters into my own hands. In this post, I'll walk you through building a small Chrome extension that blocks these pesky videos.

**Getting Started**

To build the extension, you'll need to create a new directory for your project and initialize it with `npm init`. Then, install the necessary dependencies by running `npm install browserify` (we'll get to why we're using Browserify in a minute). Create a new file called `background.js` where we'll write our logic.

**Manifest Magic**

In Chrome extensions, you need to declare your extension's permissions and functionality in the manifest file. For this extension, we only need two permissions: `activeTab` and `tabs`. The former allows us to access the current tab, while the latter is necessary for us to detect when a new tab is opened.

```javascript
{
  "manifest_version": 2,
  "name": "Autoplay Blocker",
  "version": "1.0",
  "permissions": ["activeTab", "tabs"],
  "content_scripts": [
    {
      "matches": ["*://*.news-site.com/*"],
      "js": ["inject.js"]
    }
  ]
}
```

Here, we're telling Chrome to inject a script called `inject.js` into any page that matches the specified pattern (in this case, news sites).

**The Blocker**

Now it's time to write some code! In `inject.js`, we'll use a simple content script to detect when a video is about to autoplay. We can do this by listening for the `load` event on all iframes and checking their attributes.
```javascript
document.addEventListener("DOMContentLoaded", function () {
  const iframes = document.querySelectorAll('iframe');

  iframes.forEach(function (iframe) {
    iframe.addEventListener('load', function (event) {
      const videoSrc = iframe.src;
      const isVideo = videoSrc && videoSrc.includes('.mp4') || videoSrc.includes('youtube.com');

      if (isVideo) {
        event.target.contentDocument.body.style.display = 'none';
      }
    });
  });
});
```
This code listens for the `load` event on all iframes, checks their sources to see if they're videos, and then hides the iframe's content if it is a video.

**The Bug That Bit Me**

When I first deployed this extension, everything seemed to work great. But then I noticed that some sites with lazy-loaded iframes were still autoplaying videos! After some debugging, I realized that the `load` event wasn't being triggered for these iframes because they weren't loading immediately.

To fix this issue, I added a separate check for iframe attributes like `data-src`, which is often used by lazy loaders.
```javascript
if (iframe.hasAttribute('src') || iframe.hasAttribute('data-src')) {
  // ...
}
```
With this tweak, my extension now blocks autoplaying videos on all news sites.
