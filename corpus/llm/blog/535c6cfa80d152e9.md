**From Gerrit to GitHub: Why Our Engineering Team Made the Switch**

As an engineering team of five, we've always been passionate about writing clean, efficient, and well-documented code. However, our code review process was holding us back. We were using Gerrit as our code review tool, but after some time, we realized it wasn't meeting our needs. So, we made the switch to GitHub pull requests. In this post, we'll share what we missed about Gerrit and what got easier with the transition.

**What We Missed About Gerrit**

Gerrit is a powerful code review tool that's widely used in the open-source community. However, for our team, it had some limitations. Here are a few things we missed:

* **Ease of use**: Gerrit has a steep learning curve, especially for new team members. It requires a significant amount of configuration and setup before you can start using it effectively.
* **Integration with other tools**: While Gerrit is highly customizable, integrating it with our existing toolchain was more complicated than we anticipated. We had to set up additional plugins and scripts to get everything working smoothly.
* **Feedback loop**: With Gerrit, the feedback loop between reviewers and authors can be slow. It often took several iterations before changes were made and approved.

**What Got Easier**

Switching to GitHub pull requests was a breath of fresh air for our team. Here are some benefits we've experienced:

* **Simplified setup**: Setting up GitHub pull requests was a breeze compared to Gerrit. We simply enabled the feature in our repository settings, and it worked seamlessly.
* **Native integration with other tools**: As a popular platform, GitHub has excellent integrations with many other tools, including CI/CD pipelines, project management software, and more. This made it easy for us to integrate GitHub pull requests into our existing workflow.
* **Faster feedback loop**: With GitHub pull requests, the feedback loop is much faster. Reviewers can leave comments directly on the code, and authors can make changes quickly without having to navigate a separate review interface.

**The Benefits of GitHub Pull Requests**

So, what makes GitHub pull requests so appealing? Here are some benefits we've experienced:

* **Streamlined collaboration**: With GitHub pull requests, our team members can collaborate more easily. Reviewers can leave comments and suggestions directly on the code, making it easier for authors to make changes.
* **Improved code quality**: The visibility of code reviews has improved significantly with GitHub pull requests. We can see exactly what's changed and why, which helps us catch bugs and errors earlier in the development process.
* **Reduced overhead**: With GitHub pull requests, we've reduced the overhead associated with code reviews. Reviewers don't need to create separate Gerrit accounts or navigate a complex review interface.

**Conclusion**

Switching from Gerrit to GitHub pull requests was one of the best decisions our engineering team has made in recent memory. The transition was smooth, and the benefits have been significant. We've reduced the overhead associated with code reviews, improved code quality, and streamlined collaboration among team members. If you're considering switching from Gerrit or another code review tool, we highly recommend giving GitHub pull requests a try.

**Additional Tips for a Smooth Transition**

If you're planning to switch from Gerrit to GitHub pull requests, here are some additional tips to keep in mind:

* **Plan ahead**: Before making the switch, plan how you'll integrate GitHub pull requests into your existing workflow. This may involve setting up CI/CD pipelines or project management software.
* **Communicate with your team**: Make sure all team members understand the new process and any changes to your workflow.
* **Be patient**: It may take some time for everyone to adjust to the new tool, so be patient and don't hesitate to ask for help if you need it.

By following these tips and embracing the benefits of GitHub pull requests, you can streamline your code review process and improve collaboration among team members.
