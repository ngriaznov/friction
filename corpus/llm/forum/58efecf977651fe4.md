Without explicit annotations, Rust uses its built-in rules to infer lifetimes based on the function signature.

The three lifetime elision rules in Rust are:

1. **Single method with one self parameter**: If you have a single method with no other parameters besides `self`, Rust will assign the lifetime of `self` to the reference. This is because `self` usually refers to an instance variable, which has a well-defined scope.
2. **Multiple methods, but only one takes &self**: In this case, Rust assigns the lifetime of `&self` to all references in the function signature. 3. **Multiple methods, and at least one takes &self**: Here's where things get more interesting. If you have multiple methods, and at least one of them takes `&self` or has a reference parameter, Rust will assign the shortest lifetime to all references in the function signature.
