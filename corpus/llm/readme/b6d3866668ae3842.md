# invoicer4j README

## Introduction

invoicer4j is a lightweight Java library designed to facilitate the generation of PDF invoices from a template plus data objects. It leverages an existing PDF layout engine, making it easy to integrate into your projects for creating professional-looking invoices.

## Maven Dependency Coordinates

To include invoicer4j in your project, add the following dependency to your `pom.xml` file:

```xml
<dependency>
    <groupId>com.alibaba.cloud</groupId>
    <artifactId>invoicer4j</artifactId>
    <version>1.0.2</version>
</dependency>
```

## Minimal Code Sample

Below is a minimal code sample that demonstrates how to construct an invoice with line items and render it to a file.

### Step 1: Define the Invoice Template

First, you need to define your invoice template in XML format. Here's an example of a simple invoice template:

```xml
<?xml version="1.0" encoding="UTF-8"?>
<invoice>
    <header>
        <companyName>ABC Corp</companyName>
        <address>123 Main St, Anytown USA</address>
    </header>
    <details>
        <lineItem>
            <productCode>SKU001</productCode>
            <description>Description of Product 1</description>
            <quantity>5</quantity>
            <price>19.99</price>
        </lineItem>
        <lineItem>
            <productCode>SKU002</productCode>
            <description>Description of Product 2</description>
            <quantity>3</quantity>
            <price>49.95</price>
        </lineItem>
    </details>
    <footer>
        <totalAmount>167.85</totalAmount>
    </footer>
</invoice>
```

### Step 2: Construct the Invoice and Render it to a File

Here’s how you can construct an invoice from this template and render it to a file:

```java
import com.alibaba.cloud.invoicer4j.InvoiceGenerator;
import com.alibaba.cloud.invoicer4j.model.LineItem;

public class InvoiceExample {
    public static void main(String[] args) throws Exception {
        // Define the data object for your invoice line items
        LineItem item1 = new LineItem("SKU001", "Description of Product 1", 5, 19.99);
        LineItem item2 = new LineItem("SKU002", "Description of Product 2", 3, 49.95);

        // Create a list of line items
        java.util.List<LineItem> lineItems = new java.util.ArrayList<>();
        lineItems.add(item1);
        lineItems.add(item2);

        // Define the total amount
        double totalAmount = 167.85;

        // Define the template file path (XML format)
        String templatePath = "path/to/invoice-template.xml";

        // Generate and save the invoice to a PDF file
        InvoiceGenerator.generateInvoiceToFile(lineItems, totalAmount, templatePath, "invoice.pdf");
    }
}
```

## Template Definition

invoicer4j templates are defined using a simple XML format. The root element `<invoice>` can contain various elements such as `<header>`, `<details>`, and `<footer>`. Inside these sections, you define the structure of your invoice.

### Example Template Structure:

- **Header Section**:
  - Contains company name and address information.
  
- **Details Section**:
  - Used to list individual line items with product codes, descriptions, quantities, and prices.
  
- **Footer Section**:
  - Includes total amount or other summary information.

## Supported Currencies

invoicer4j supports a wide range of currencies. The library automatically formats the amounts according to the defined currency when rendering the invoice. You can specify the desired currency using an ISO 4217 code (e.g., `USD`, `EUR`).

### Example Usage:
```java
InvoiceGenerator.generateInvoiceToFile(lineItems, totalAmount, templatePath, "invoice.pdf", CurrencyCode.EUR);
```

## Locale-Aware Number Formatting

invoicer4j ensures that number formatting is locale-aware. This means that the library will adjust decimal separators and thousands separators based on the specified locale when rendering the invoice.

### Example Usage:
```java
InvoiceGenerator.generateInvoiceToFile(lineItems, totalAmount, templatePath, "invoice.pdf", Locale.US);
```

## Known Limitations

1. **Multi-Page Invoices**:
   - Currently, invoicer4j does not support generating multi-page invoices directly from a single template. If your invoice spans multiple pages, you may need to split the content across templates or handle pagination manually.
   
2. **Customization Capabilities**:
   - The current version supports basic customization through XML templates but lacks advanced features like conditional logic within the template.

3. **Template Validation**:
   - There is no built-in validation for the XML templates, so invalid templates might lead to unexpected results or exceptions during rendering.

## Conclusion

invoicer4j provides a straightforward way to generate PDF invoices from templates and data objects. With support for basic localization, currency handling, and simple template definitions, it can be quickly integrated into your projects. For more advanced use cases, consider exploring the library’s capabilities and limitations further.
