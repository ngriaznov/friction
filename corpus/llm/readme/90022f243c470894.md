# csvtype README
=====================

csvtype is a Rust crate that enables efficient and accurate inference of CSV file structures by leveraging statistical analysis.

## Installation

Add `csvtype` to your `Cargo.toml`:

```toml
[dependencies]
csvtype = "0.1.0"
```

## Usage

### Example: Inferring Schema and Iterating Typed Rows

First, import the necessary modules:
```rust
use csvtype::{CsvRecord, ColumnSchema};
use std::fs::File;
use std::io::BufReader;

fn main() {
    let file = File::open("data.csv").unwrap();
    let reader = BufReader::new(file);

    // Create a new CsvReader instance with the desired sampling size
    let reader = csvtype::CsvReader::new(reader, 100).unwrap();

    // Infer column types and create a typed record iterator
    let schema = reader.infer_schema().unwrap();
    let record_iter = reader.iterated(schema);

    for row in record_iter {
        println!("{:?}", row);
    }
}
```

In this example, we open a CSV file named `data.csv` and create a new `CsvReader` instance with a sampling size of 100 rows. We then use the `infer_schema()` method to infer column types based on the sampled data. Finally, we call `iterated()` to get an iterator over typed records.

### Sampling Size

The sampling size affects the accuracy of inferred types. A larger sampling size provides more reliable results but can be slower due to increased computation time. The ideal sampling size depends on the specific use case and dataset characteristics.

### Manual Override

You can override an inferred type manually per column using the `override` method:

```rust
let reader = csvtype::CsvReader::new(reader, 100).unwrap();
let schema = reader.infer_schema().unwrap();

for (i, col) in schema.columns.iter().enumerate() {
    if i == 0 { // Integer column
        reader.override(col.clone(), ColumnSchema::Integer);
    } else if i == 1 { // Float column
        reader.override(col.clone(), ColumnSchema::Float);
    }
}

let record_iter = reader.iterated(schema);

for row in record_iter {
    println!("{:?}", row);
}
```

In this example, we override the inferred type for integer and float columns manually.

## Notes

csvtype uses statistical analysis to infer column types. The accuracy of these inferences depends on several factors:

*   **Dataset size**: Larger datasets generally provide more reliable results.
*   **Data distribution**: Datasets with skewed or uniform distributions can affect inference accuracy.
*   **Column correlation**: Correlated columns may lead to incorrect type inferences.

To improve accuracy, consider increasing the sampling size or collecting a larger representative subset of data. Additionally, manual override of inferred types can provide more precise control over column structures.
