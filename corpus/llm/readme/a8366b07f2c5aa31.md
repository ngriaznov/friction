## csvtype: Typed CSV Iteration in Rust

`csvtype` is a Rust crate that simplifies working with CSV files by inferring a schema of column types and providing a typed record iterator. Instead of dealing with raw strings, you can iterate over your CSV data as structured records with clearly defined types for each column. This makes parsing, processing, and analyzing CSV data significantly easier and more robust.

**Installation:**

Add the following to your `Cargo.toml`:

```toml
[dependencies]
csvtype = "0.1.0" // Replace with the latest version
```

**Example Usage:**

```rust
use csvtype::{CsvReader, CsvType};

fn main() {
    let reader = CsvReader::from_path("data.csv").unwrap();
    let schema = reader.infer_schema().unwrap();

    println!("Schema: {:?}", schema);

    for record in reader.typed_records(&schema).unwrap() {
        println!("{:?}", record);
    }
}
```

**Explanation:**

1. **`CsvReader::from_path("data.csv")`**: Creates a `CsvReader` instance from the specified CSV file path.
2. **`reader.infer_schema()`**: Infers the column types based on sampling rows from the CSV file. This returns a `Result<Schema, Error>`, where `Schema` represents the inferred type schema for each column.
3. **`reader.typed_records(&schema)`**: Creates a typed record iterator using the inferred schema. Each element in this iterator is a struct representing a row with fields corresponding to the columns and their inferred types.

**Sampling Size and Accuracy:**

The accuracy of the type inference depends on the sampling size used by `infer_schema()`. A larger sample size generally leads to more accurate results, but it also takes longer to process. You can adjust the sampling size using the `sample_size` parameter in `CsvReader::from_path()`.

**Manual Type Overrides:**

If you need to override an inferred type for a specific column, you can provide a custom `Schema` struct when calling `reader.typed_records()`. This allows you to fine-tune the schema based on your specific requirements.


```rust
// Example: Override the "age" column type to u32
let schema = Schema {
    name: CsvType::String,
    age: CsvType::U32, // Manually override age type
    ...
};

for record in reader.typed_records(&schema).unwrap() {
    // ... process the typed record
}
```



**Benefits of Using `csvtype`:**

* **Type Safety:** Avoid runtime errors by working with explicitly typed data.
* **Improved Readability:** Code becomes more concise and easier to understand when dealing with structured records instead of raw strings.
* **Enhanced Data Analysis:** Typed data allows for more efficient and accurate analysis using Rust's powerful libraries.
