# Feedwell README

## Introduction

Feedwell is a self-hosted RSS and Atom feed reader designed to provide a lightweight web UI experience. It's built to run behind any WSGI server, making it flexible and easy to deploy on various hosting environments. This README will guide you through the installation process, feature overview, setup instructions, database backend options, and how to schedule feed polling using cron.

## Features

### Folder Organization
Feedwell allows users to organize their feeds into folders, enabling a more structured approach to managing subscriptions. You can easily move feeds between folders based on your preferences or topics of interest.

### Read/Unread Tracking
With Feedwell, you can track which articles have been read and which are still unread. This feature helps in keeping tabs on new content without having to manually note down what you've already seen.

### Full-Text Search Across Cached Articles
Feedwell supports full-text search across all cached articles. Whether you're looking for a specific keyword or phrase within the text of an article, Feedwell's search functionality makes it quick and easy to find exactly what you need without leaving the application.

### OPML Import/Export
FeedExceptions are compatible with OPML (Outline Processor Markup Language) files, allowing you to import your existing feed subscriptions from other readers or export them for backup purposes. This feature ensures a smooth transition between different RSS/Atom feed readers while preserving all your current subscriptions and settings.

### Database Backend Options

#### SQLite by Default
Feedwell uses SQLite as its default database backend due to its simplicity and ease of use. It's perfect for small-scale deployments where performance requirements are modest, and maintenance overhead is minimal.

#### PostgreSQL Support
For larger installations or those requiring higher scalability and reliability, Feedwell also supports PostgreSQL. This option provides better performance handling large datasets and concurrent users, making it ideal for high-traffic environments or enterprise-level deployments.

## Installation from Source with a Virtualenv

### Prerequisites
Before you begin, ensure you have Python 3.x installed on your system along with pip (Python package installer). You'll also need to install `virtualenv` if you haven't already:

```bash
pip install virtualenv
```

### Step-by-Step Installation

1. **Clone the Repository**
   Start by cloning the Feedwell repository from GitHub:
   
   ```bash
   git clone https://github.com/yourusername/feedwell.git
   cd feedwell
   ```

2. **Create a Virtual Environment**
   Navigate to the project directory and create a virtual environment for isolation of dependencies:
   
   ```bash
   python -m venv .venv
   source .venv/bin/activate  # On Windows use `.venv\Scripts\activate`
   ```

3. **Install Dependencies**
   Install all necessary Python packages listed in `requirements.txt`:
   
   ```bash
   pip install -r requirements.txt
   ```

4. **Configure Settings**
   Create a configuration file named `.env` in the project root with your database settings and any other custom configurations required by Feedwell. For example, to use PostgreSQL:

   ```
   DATABASE_URL=postgresql://user:password@localhost/feedwell_db
   SECRET_KEY=random_secret_key_here
   ```

5. **Run Database Migrations**
   Initialize the database schema based on your selected backend (SQLite or PostgreSQL):
   
   ```bash
   python manage.py migrate
   ```

6. **Create an Admin Account**
   After installation, you'll need to create an admin account to access the administrative interface:
   
   ```bash
   python manage.py createsuperuser
   ```

## First-Run Setup

Upon completing the installation and database migration, visit your Feedwell instance in a web browser (e.g., `http://localhost:8000`). You'll be prompted to create an admin account. Follow these steps:

1. **Log In**
   Use the credentials you created during the `createsuperuser` step.

2. **Configure Feeds**
   After logging in, navigate to the "Feeds" section where you can import existing feeds via OPML files or add new subscriptions directly from within Feedwell.

3. **Set Up Folder Organization**
   Organize your feeds into folders as needed for better management and navigation through your reading list.

4. **Enable Read/Unread Tracking**
   Ensure that the read/unread tracking feature is enabled in settings to start marking articles as read or unread.

## Scheduling Feed Polling with Cron

To keep your feed reader up-to-date, you'll want to schedule polling for new content at regular intervals using cron jobs:

1. **Open crontab Editor**
   Open a terminal and run:
   
   ```bash
   crontab -e
   ```

2. **Add a New Job**
   Add the following line to schedule Feedwell to poll feeds every hour (adjust frequency as needed):
   
   ```
   0 * * * * /path/to/your/virtualenv/bin/python /path/to/feedwell/manage.py feed_poll >> /path/to/feedwell/feedpoll.log 2>&1
   ```

   Replace `/path/to/your/virtualenv/bin/python` and `/path/to/feedwell/manage.py` with the actual paths to your virtual environment's Python executable and Feedwell's manage script, respectively.

3. **Save and Exit**
   Save the changes and exit the editor. Cron will now automatically run the feed polling command at the specified interval (every hour in this example).
