# confkeep: Preserving Configuration File Formatting

## Introduction

`confkeep` is a lightweight Python library designed to parse INI-style configuration files while preserving all formatting elements like comments and blank lines. This feature ensures that when you round-trip the file (read, modify, and write), your user's manual formatting remains intact.

Unlike Python's built-in `configparser`, which strips out comments and extra whitespace, `confkeep` maintains these details, providing a more faithful representation of the original configuration files.

## Installation

To install `confkeep`, you can use pip:

```sh
pip install confkeep
```

## Motivation

While `configparser` is robust for many scenarios, it lacks support for preserving comments and extra whitespace. This can be problematic when users have customized their configurations with specific formatting or inline comments. By using `confkeep`, you ensure that the user's customizations are not lost during programmatic modifications.

## Example Usage

Below is a short example demonstrating how to read a configuration file, modify one value, and write it back out:

```python
from confkeep import ConfigParser

# Load the configuration file
config = ConfigParser()
config.read('example.ini')

# Modify a value
config.set('section1', 'key2', 'new_value')

# Write the modified configuration back to the file
with open('example.ini', 'w') as configfile:
    config.write(configfile)
```

### File Contents Before and After

Here is an example of what your `example.ini` might look like:

```ini
# This is a comment on section 1
[section1]
key1 = value1
# This is another comment on key2
key2 = old_value
[section2]
key3 = value3
```

After running the above script, the file will be updated to:

```ini
# This is a comment on section 1
[section1]
key1 = value1
# This is another comment on key2
key2 = new_value
[section2]
key3 = value3
```

## Supported Syntax Quirks

`confkeep` supports several common quirks found in INI-style configuration files:

### Inline Comments

Inline comments allow you to place a `;` or `#` at the start of a line, and everything after that is treated as a comment. For example:

```ini
[section1]
key1 = value1 ; This is an inline comment
```

In this case, `confkeep` will preserve both the key-value pair and the inline comment when writing back to the file.

### Multi-Line Values

Sometimes values span multiple lines or include newlines within them. `confkeep` supports these cases by preserving newline characters:

```ini
[section1]
key1 = This is a multi-line value.
       It continues here.
```

After modification, this remains intact:

```ini
[section1]
key1 = This is a modified multi-line value.
       It continues here.
```

### Section Inheritance

While not directly supported in standard INI files, `confkeep` allows you to simulate section inheritance by using multiple sections with similar key-value pairs. Although this isn't true inheritance, it can be useful for maintaining consistent configurations across different sections.

## Conclusion

By using `confkeep`, you ensure that your users' customizations and formatting are preserved during programmatic modifications of configuration files. This library provides a more robust solution compared to the built-in `configparser` when dealing with complex or user-customized INI-style configurations.

Feel free to explore its capabilities further in your projects!
