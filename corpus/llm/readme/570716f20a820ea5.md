# Packlite README

## Introduction

Packlite is a Node.js command-line tool designed to simplify the process of bundling and minifying JavaScript and CSS files for simple static sites. Unlike full Single Page Application (SPA) frameworks, Packlite focuses on efficiency and ease of use, making it ideal for projects with straightforward requirements.

## Installation

To get started with Packlite, you need Node.js installed on your machine. Once Node.js is set up, install Packlite globally using npm:

```bash
npm install -g packlite
```

This command makes the `packlite` command available across your system.

## Configuration Example

After installation, create a configuration file named `packlite.config.json` in the root of your project. Here’s an example configuration that lists entry points for JavaScript and CSS files:

```json
{
  "entry": {
    "js": ["src/scripts/main.js", "src/scripts/utilities.js"],
    "css": ["src/styles/main.css", "src/styles/components.css"]
  },
  "output": {
    "js": "dist/bundle.js",
    "css": "dist/style.min.css"
  }
}
```

This configuration tells Packlite which files to bundle and where to output the resulting bundled files.

## Watch Mode

For local development, enabling watch mode is highly recommended. It allows you to automatically recompile your assets whenever changes are detected:

```bash
packlite --watch
```

This command keeps Packlite running in the background, watching for file changes and rebuilding the project as needed without manual intervention.

## Source Map Generation

Packlite supports generating source maps for easier debugging of bundled files. To enable this feature, add `--source-maps` to your build command:

```bash
packlite --source-maps
```

Source maps will be generated alongside your output files, mapping the minified code back to the original source.

## Build Time Comparison

Using Packlite significantly reduces build times compared to manually bundling with heavier tools like Webpack or Gulp. Here’s a brief comparison:

- **Manual Bundling**: Requires setting up configurations for loaders, plugins, and environment setups, which can be time-consuming and error-prone.
- **Packlite**: With its streamlined approach, Packlite reduces setup time from hours to minutes. It automatically handles bundling and minification with minimal configuration.

For example, a typical Webpack build might take several seconds to complete for small projects, whereas Packlite achieves the same result in mere fractions of that time, especially when combined with watch mode for iterative development.

## Conclusion

Packlite offers an efficient, straightforward solution for bundling and minifying assets for static sites. Its ease of use, quick setup, and minimal configuration make it a valuable tool for developers looking to streamline their workflow without the complexity often associated with full-fledged bundlers designed for SPAs.

For more detailed usage instructions or troubleshooting tips, refer to the Packlite documentation or community forums. Happy coding!
