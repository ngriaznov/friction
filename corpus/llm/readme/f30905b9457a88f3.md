# Backupctl README

## Overview

Backupctl is a collection of bash scripts designed to automate nightly backups for a LAMP (Linux, Apache, MySQL, PHP) stack server. The suite includes:

1. **MySQL dumps** - Regular database snapshots.
2. **Rsync of the web root** - Copies the entire web directory to ensure all files are backed up.
3. **Rotation of old archives** - Manages storage by keeping only a specified number of backups.

## Prerequisites

Before installing backupctl, ensure you have:

- A Linux server with Apache, MySQL, and PHP installed.
- Bash shell access.
- Root or sudo privileges for script execution.

## Environment Variables

Backupctl relies on specific environment variables to manage database credentials securely. Set these in your system's configuration files (e.g., `/etc/environment` or a `.bashrc` file):

```bash
export MYSQL_USER='your_mysql_user'
export MYSQL_PASSWORD='your_mysql_password'
export MYSQL_DATABASE='your_database_name'
```

These variables must be set before running any backupctl scripts. If not defined, the scripts will fail with an error message prompting you to configure them.

## Installation and Cron Setup

1. **Download/Clone Backupctl**: Place the script files in a directory of your choice (e.g., `/opt/backupctl`).

   ```bash
   git clone https://github.com/yourusername/backupctl.git /opt/backupctl
   ```

2. **Set Permissions**:

   ```bash
   chmod +x /opt/backupctl/*.sh
   ```

3. **Configure Cron Job**: Add a cron entry to run the backup script nightly (at 2 AM by default):

   ```bash
   crontab -e
   ```

   Then add:

   ```
   0 2 * * * /usr/bin/env bash /opt/backupctl/run_backup.sh > /dev/null 2>&1
   ```

   This command runs `run_backup.sh` every night at 2 AM.

## Backup Location and Retention

Backups are stored in a designated directory:

```bash
export BACKUP_DIR='/mnt/backup/server_name'
```

Ensure the directory exists and has write permissions for the user running backupctl. By default, backupctl retains only the last **7** backups due to rotation logic defined in `rotate_backups.sh`. To change this behavior:

1. Open `configure.sh` and modify the line:

   ```bash
   export RETENTION=7
   ```

   Adjust the number as needed for your storage capacity.

## Restore Script

To restore a previous backup, use the provided script:

```bash
./restore_latest.sh
```

This command will automatically select the most recent MySQL dump and the corresponding web root rsync archive, restoring both to their respective locations on the server. Ensure you have sufficient permissions and that no other processes are interfering with the restore operation.

## Conclusion

Backupctl simplifies managing nightly backups for a LAMP stack by automating database dumps, file system syncing, and archival rotation. By following the setup instructions and utilizing environment variables, you can ensure your data is safely backed up and easily recoverable in case of failure. For further customization or troubleshooting, consult the script comments and logs generated during each run.

Happy backing up!
