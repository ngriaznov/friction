# backupctl - Automated Backup Scripts for LAMP Stack Servers

## Overview
`backupctl` is a collection of bash scripts designed to automate nightly backups of a LAMP (Linux, Apache, MySQL, PHP) stack server environment. It handles the following tasks:
- Regularly backing up MySQL databases.
- Rsyncing the web root directory.
- Rotating and archiving old backup files.

## Prerequisites
Ensure that you have `bash` installed on your system as these scripts are written in bash. Additionally, this setup requires a basic understanding of LAMP stack components.

## Setup

### Environment Variables Configuration
Before using `backupctl`, configure the necessary environment variables:

```sh
export BACKUPCTL_HOME="/path/to/your/backupdir"
export MYSQL_USER="username"
export MYSQL_PASSWORD="password"
export MYSQL_DATABASE="databasename"
```

- **BACKUPCTL_HOME**: The directory where backups will be stored.
- **MYSQL_USER**, **MYSQL_PASSWORD**, **MYSQL_DATABASE**: Credentials for your MySQL database.

### Cron Job Configuration
To schedule automatic nightly backups, add the following cron entry to your crontab:

```sh
0 2 * * * /path/to/backupctl.sh >> /var/log/backupctl.log 2>&1
```

This cron job runs `backupctl` at 2 AM every day.

## Backup Directory Structure

Backups are written to a directory structure under `$BACKUPCTL_HOME`, e.g., `/path/to/your/backupdir`. The directory will look something like this:

```sh
/path/to/your/backupdir/
├── mysql_backups/
│   └── (date) - contains SQL dump files for each database.
└── web_root_backups/
    ├── (date1)
    │   └── webroot_backup.tar.gz  # Latest backup of the web root
    └── (date2)
        └── webroot_backup.tar.gz
```

## Retention Configuration

By default, `backupctl` retains three most recent backups for both MySQL and web root. If you wish to change this, modify the retention script in `cleanup.sh`. For example:

```sh
# Keep 3 latest backups only
find $BACKUPCTL_HOME/mysql_backups -type f ! -name 'current.sql' -mtime +7 -exec rm {} \;
find $BACKUPCTL_HOME/web_root_backups -type f -mtime +7 -exec rm {} \;
```

This script will delete backups older than 7 days.

## Restore Script

To reverse the process and restore your server to a previous state, use the `restore.sh` script. Here’s how it works:

1. **MySQL Restore**: You can specify which database dump file to use:
   ```sh
   ./restore-mysql.sh /path/to/backupdir/mysql_backups/2023-10-01/databasename.sql
   ```

2. **Web Root Restore**:
   ```sh
   cd /var/www/html  # Change to your web root directory
   tar -xvzf /path/to/backupdir/web_root_backups/2023-10-01/webroot_backup.tar.gz
   ```
   
3. **Cleanup Old Backups After Restore**: To ensure the backup dir doesn't fill up, run:
   ```sh
   ./cleanup.sh
   ```

## Conclusion

`backupctl` simplifies the process of managing backups for a LAMP stack server. By following these instructions and customizing to your specific needs, you can maintain a robust backup strategy that ensures data safety and easy recovery when needed.

Happy backing up!
