README
================

Multitail-go
-------------

A command-line tool that allows you to tail multiple log files at once, providing color-coded labels to identify the source of each line.

Installation
------------

To install multitail-go, run the following command:

```bash
go get github.com/your-username/multitail-go
```

Replace `github.com/your-username/multitail-go` with your actual GitHub username. This will fetch the latest version of the tool from the Go package repository.

Usage
-----

### Basic Usage

To start multitail-go, use the following command:

```bash
multitail-go /path/to/log1.log /path/to/log2.log /path/to/log3.log
```

Replace `/path/to/log1.log` with the paths to your log files. The tool will display each line of the logs in a separate window, prefixed with a color-coded label indicating its source file.

### Glob Patterns

To rotate through multiple log files at once, you can use glob patterns to specify files that match certain names or extensions. For example:

```bash
multitail-go -R '*' /path/to/log*.*
```

This will start multitail-go and tail all log files in the `/path/to/log` directory and its subdirectories.

### Filtering Lines

You can filter lines by a regular expression using the `-F` flag followed by your regex pattern. For example:

```bash
multitail-go -F 'error|exception' /path/to/log1.log /path/to/log2.log
```

This will only display lines that match either "error" or "exception".

### Color Coding

Colors are assigned to sources based on their file extension. The following colors are used:

* `.log` (default): Red for log files, green for error logs
* `.txt`: Blue for text files, yellow for HTML files
* `.csv`: Cyan for CSV files
* `.json`: Magenta for JSON files

If a file has no extension or an unrecognized extension, it will default to red.

### Customizing Colors

You can customize the colors used by multitail-go by modifying the `color.conf` file. This file is located in the same directory as the multitail-go executable and contains a list of file extensions mapped to colors. For example:

```ini
.log=red
.txt=blue
.html=yellow
.csv=cyan
.json=magenta
```

Add or modify entries as needed to change the color scheme.

Troubleshooting
--------------

If you encounter any issues with multitail-go, refer to the following common problems and solutions:

*   **Failed to open log file**: Make sure that the log file exists and is readable by your user account. If you have write access to the file but are unable to read it, check your permissions.
*   **Incorrect color coding**: Check your `color.conf` file for any typos or inconsistencies.

FAQ
----

*   **Q: Can I customize the number of lines displayed?**
    A: Yes, you can use the `-L` flag followed by your desired line count. For example:

        ```bash
multitail-go -L 100 /path/to/log1.log /path/to/log2.log
```

    This will display the first 100 lines of each log file.

*   **Q: Can I specify a different separator for the log files?**
    A: Yes, you can use the `-S` flag followed by your desired separator. For example:

        ```bash
multitail-go -S ',' /path/to/log1.log /path/to/log2.log
```

    This will display each log file with a comma-separated list of line numbers.
