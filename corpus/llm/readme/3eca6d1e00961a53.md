# README: csvtype

**csvtype** is a Rust crate designed to read CSV files and automatically infer the data types of each column (e.g., integer, float, date, boolean, string) by sampling rows from the file. Once the schema is inferred, it provides an iterator that yields records with typed fields instead of raw strings, making data processing more efficient and type-safe.

## Installation

To install `csvtype`, add the following line to your `Cargo.toml`:

```toml
[dependencies]
csvtype = "0.1"
```

Ensure you have Rust installed on your system, then run:

```bash
cargo build
```

to compile the crate and its dependencies.

## Example Usage

Below is a simple example demonstrating how to use `csvtype` to read a CSV file, infer its schema, and iterate over typed records.

```rust
use csvtype::CsvReader;

fn main() {
    // Path to your CSV file
    let path = "data/sample.csv";

    // Create a reader instance
    let mut reader = CsvReader::new(path);

    // Iterate over the rows with inferred types
    for record in reader.records() {
        println!("{:?}", record);
    }
}
```

### Inference and Sampling Size

The type inference process relies on sampling a subset of rows from the CSV file. The default sample size is set to 100 rows, which provides a good balance between accuracy and performance. However, if your dataset is very large or highly skewed (e.g., many categorical values), you might need a larger sample for more accurate type inference.

Increasing the sample size can improve accuracy but will also increase memory usage and processing time. Conversely, reducing the sample size may lead to less precise type predictions but will be faster to compute.

### Overriding Inferred Types

In some cases, the automatic type inference might not perfectly capture your data's true nature (e.g., a column intended for dates being inferred as strings). You can manually override the inferred type per column by using the `with_type_override` method:

```rust
use csvtype::CsvReaderBuilder;

let mut reader = CsvReaderBuilder::new()
    .has_header(true)
    .type_overrides(vec![
        ("your_column_name".to_string(), "date".to_string()),
    ])
    .from_path("data/sample.csv");
```

This feature is particularly useful for datasets with ambiguous data types or when you have domain-specific knowledge about the expected type of certain columns.

## Notes

- **Performance**: The crate leverages efficient Rust libraries under the hood, ensuring that even large CSV files are processed quickly.
- **Error Handling**: If a column cannot be inferred due to insufficient sample size or ambiguous data, an error will be raised during iteration, prompting you to adjust your sampling strategy or override types where necessary.
- **Compatibility**: `csvtype` is compatible with standard UTF-8 encoded CSV files. Non-standard encodings may require additional configuration.

By using `csvtype`, you can streamline the process of working with structured data in Rust applications, reducing runtime errors related to type mismatches and enhancing code readability and maintainability through typed records.
