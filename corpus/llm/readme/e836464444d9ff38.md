# csvtype
A Rust crate for reading CSV files and inferring a column type schema.

1. [Installation](#installation)

## Installation

Add the following line to your `Cargo.toml` file:

```toml
[dependencies]
csvtype = "0.1.0"
```

Then run `cargo build` in your project directory.

## Example Usage

Here's a basic example of how you might use csvtype:

```rust
use csvtype::{Reader, Record};

fn main() -> std::io::Result<()> {
    let reader = Reader::from_path("path/to/file.csv")?;
    for record in reader.records() {
        // Each record is a typed representation of the row.
        match record {
            Ok(record) => println!("{:?}", record),
            Err(err) => eprintln!("Error parsing record: {}", err),
        }
    }

    Ok(())
}
```

This example will print out each record as a `Record` object, which contains fields for each column type.

## Inference Accuracy

The accuracy of the inferred types is dependent on the sampling size. If you have large CSV files or highly variable data, increasing the sampling size can improve accuracy.

```rust
let reader = Reader::from_path("path/to/file.csv")
    .infer_types(1000) // Increase the sampling size to 1000 rows.
```

However, note that larger sampling sizes will increase processing time.

## Overriding Inferred Types

If you're certain of a column's type, you can override the inferred type manually:

```rust
let reader = Reader::from_path("path/to/file.csv")
    .infer_types(10)
    .override_type("column_name", Type::Date); // Override "column_name" to Date.
```

This example assumes that `Type` is imported from csvtype.
