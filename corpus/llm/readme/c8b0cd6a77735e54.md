# phodedupe: Duplicate Photo Finder Using Perceptual Hashing

## Overview

phodedupe is an open-source command-line tool written in Python designed to scan one or more directories for duplicate photos using perceptual hashing. Unlike traditional checksum comparisons, which fail to identify duplicates when images are resized or slightly re-compressed, perceptual hashing provides a robust solution by focusing on the content of the image rather than its exact binary representation.

## Why Perceptual Hashing?

Perceptual hashing offers several advantages over plain checksum comparison:

1. **Content-Based Identification**: It captures the essential features of an image, allowing it to identify duplicates even if they have been resized or slightly altered in compression.
2. **Robustness to Transformations**: Images that appear identical but differ in resolution or compression artifacts are still recognized as duplicates due to similar hash values.
3. **Efficiency**: Reduces false positives by ignoring minor changes, making the tool more practical for real-world use cases.

## Installation

phodedupe can be easily installed via pip, Python's package installer:

```bash
pip install phodedupe
```

Ensure you have Python and pip set up on your system. For Linux or macOS users, this is typically included by default. On Windows, you may need to add Python to your PATH environment variable.

## Basic Usage

To scan a directory for duplicate photos and generate a report before any deletions occur:

```bash
phodedupe /path/to/your/photos
```

This command will traverse the specified directory, identify duplicates based on perceptual hashes, and output a summary of found groups. No changes are made to the original files.

### Dry-Run Flag

To preview what would happen without actually deleting any files:

```bash
phodedupe /path/to/your/photos --dry-run
```

The `--dry-run` flag allows you to review the list of duplicates that would be identified, ensuring you're comfortable with the deletion process before proceeding.

## Configuration Options

### Hash Algorithm Choice

By default, phodedupe uses the BLAKE2b hash algorithm due to its speed and security. However, you can specify a different algorithm if needed:

```bash
phodedupe /path/to/your/photos --hash-algorithm=sha256
```

Supported algorithms include `blake2b`, `sha256`, and others available in Python's hashlib module.

### Similarity Threshold

To adjust how similar images are considered duplicates, you can set a similarity threshold:

```bash
phodedupe /path/to/your/photos --similarity-threshold=0.85
```

The threshold is a value between 0 and 1, where values closer to 1 require more exact matches (useful for high-resolution scans), while lower values allow broader matching of similar images.

## Conclusion

phodedupe leverages perceptual hashing to efficiently identify duplicate photos in large collections without being misled by minor changes such as resizing or slight compression artifacts. By integrating with pip, it offers a straightforward installation process and flexible configuration options to suit various use cases. Whether you're organizing personal photo libraries or managing media assets for professional projects, phodedupe provides a reliable solution for detecting and handling duplicates.
