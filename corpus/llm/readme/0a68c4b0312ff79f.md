## csvtype: Typed CSV Iteration in Rust

**A crate for efficient and type-safe CSV data processing.**

csvtype provides a convenient way to read CSV files and iterate over their rows as typed records, eliminating the need to manually guess column types and manage string parsing. It intelligently infers a schema based on sampling, offering an initial level of type safety before you can refine it further.

**Why use csvtype?**

* **Type Safety:** Work with data as integers, floats, dates, booleans, or strings – preventing common errors caused by misinterpreting CSV values as strings.
* **Simplified Iteration:** Iterate directly over typed records instead of manually parsing and converting each cell.
* **Reduced Boilerplate:**  Eliminates the need for custom parsing logic and error handling for different data types.
* **Performance:** Optimized for reading CSVs, leveraging Rust's performance characteristics.

**Cargo.toml Installation Snippet**

```toml
[dependencies]
csvtype = "0.1.2" # Replace with the latest version
```

Then, in your `src/main.rs` file:

```rust
mod main; // Import the module containing your application logic
```

**Example Usage:**

```rust
use csvtype::CsvRecordIterator;
use csvtype::CsvTypeInferer;

fn main() -> Result<(), Box<dyn std::error::Error>> {
    // Replace "your_file.csv" with your CSV file path
    let iterator = CsvTypeInferer::new("your_file.csv", 100)?; 

    for record in iterator {
        println!("{:?}", record); // Prints a typed record (e.g., [1, 3.14, "hello"])
    }

    Ok(())
}
```

**Schema Inference and Sampling Size:**

`csvtype` infers the schema by sampling a specified number of rows from your CSV file. This is done to quickly determine common data types without reading the entire file. The default sample size is 100, but you can adjust this using `CsvTypeInferer::new()`.  **Important:** A larger sample size generally leads to more accurate type inference, especially for datasets with diverse columns. However, a very large sample might be unnecessary and could increase processing time. Experiment to find the optimal balance for your data.

**Overriding Type Inference (Manual Type Specification):**

While `csvtype` attempts to infer types automatically, you can override this behavior for specific columns using the `column_types` parameter in the `CsvTypeInferer::new()` function.  This allows you to explicitly define the expected type of each column if the inference isn't accurate or if you have domain-specific knowledge about your data. 

```rust
use csvtype::{CsvRecordIterator, CsvTypeInferer};

fn main() -> Result<(), Box<dyn std::error::Error>> {
    let iterator = CsvTypeInferer::new("your_file.csv", 100)
        .column_types(vec![
            Some(csvtype::DataType::Integer), // Column 0 is an integer
            Some(csvtype::DataType::Float),  // Column 1 is a float
            None,                          // Column 2: let the crate infer this
        ])?;

    for record in iterator {
        println!("{:?}", record);
    }

    Ok(())
}
```

In this example, we've specified that column 0 should be an integer and column 1 a float.  The `None` value indicates that the crate will attempt to infer the type for this column automatically.

**Notes:**

* **Data Quality:**  `csvtype` relies on good data quality within your CSV file. Inconsistent formatting or unexpected values can negatively impact type inference accuracy.
* **Error Handling:** The crate provides basic error handling, but you should implement robust error handling in your application to gracefully handle potential issues such as invalid data types or missing values.
* **Performance Considerations**: For very large CSV files, consider using streaming techniques for improved memory efficiency, although the current implementation is optimized for moderate-sized datasets.
