# phodedupe: A Command-Line Tool for Detecting Duplicate Photos

## Introduction

`phodedupe` is an open-source command-line tool written in Python that helps you identify duplicate photos within one or more directories using perceptual hashing. This method ensures that resized or slightly re-compressed copies of the same image are caught, making it a powerful tool for photo management and cleanup.

## Why Perceptual Hashing Beats Plain Checksum Comparison

Perceptual hashing is superior to plain checksum comparison in several ways:

1. **Resilience to Transformations**: Perceptual hashes can identify images that have been resized, rotated, or slightly re-compressed, which would be missed by simple checksums.
2. **Visual Similarity**: The hash values are based on the visual content of the image, making them more robust against noise and minor alterations.
3. **Efficiency**: Perceptual hashing is generally faster to compute than full-image comparisons.

## Installation

To install `phodedupe`, you can use pip:

```bash
pip install phodedupe
```

Alternatively, if you want to contribute or just need the latest features, you can clone the repository and install it from source:

```bash
git clone https://github.com/yourusername/phodedupe.git
cd phodedupe
python setup.py install
```

## Basic Usage

### Scanning a Folder

To scan a folder for duplicate photos, use the following command:

```bash
phodedupe /path/to/folder --report report.txt
```

This will generate a report in `report.txt` detailing the groups of duplicates found.

### Reviewing and Deleting Duplicates

After generating the report, you can review it to decide which files to delete. The tool does not automatically delete any files; this is intended as a safety measure to prevent accidental data loss.

To perform a dry-run (which simulates deletions without actually deleting anything), use:

```bash
phodedupe /path/to/folder --report report.txt --dry-run
```

### Deleting Duplicates

Once you have reviewed the report and are sure which files should be deleted, run:

```bash
phodedupe /path/to/folder --report report.txt --delete
```

This will delete the duplicate files according to the groups identified in the report.

## Configuration Options

`phodedupe` offers several configuration options to customize its behavior. You can specify these using command-line flags or by creating a `config.ini` file.

### Hash Algorithm Choice

You can choose between different hashing algorithms:

```bash
phodedupe /path/to/folder --report report.txt --hash-algorithm pHash
```

The default algorithm is `pHash`, but you can also use `dHash` or `aHash`.

### Similarity Threshold

Set the similarity threshold to control how similar images must be to be considered duplicates:

```bash
phodedupe /path/to/folder --report report.txt --similarity-threshold 0.85
```

The default value is `0.9`, but you can adjust it based on your needs.

### Example Configuration File

Create a `config.ini` file with the following content to set these options:

```ini
[phodedupe]
hash_algorithm = dHash
similarity_threshold = 0.85
```

Then, run the tool using this configuration:

```bash
phodedupe /path/to/folder --report report.txt --config config.ini
```
