# Invoicer4J
Invoicer4J is a Java library designed to generate PDF invoices from a template plus a data object. It leverages an existing PDF layout engine, allowing users to create professional-looking invoices with minimal coding effort.

## Maven Dependency Coordinates
To use Invoicer4J in your project, add the following dependency to your `pom.xml` file:
```xml
<dependency>
    <groupId>com.example</groupId>
    <artifactId>invoicer4j</artifactId>
    <version>1.0.0</version>
</dependency>
```
## Minimal Code Sample
Here's an example of how to construct an invoice with line items and render it to a file:
```java
import com.example.invoicer4j.Invoice;
import com.example.invoicer4j.InvoiceTemplate;

public class InvoiceExample {
    public static void main(String[] args) {
        // Create an invoice template from the 'invoice.xml' file
        InvoiceTemplate template = new InvoiceTemplate("invoice.xml");

        // Create a sample invoice data object
        Invoice invoice = new Invoice();
        invoice.setClientName("John Doe");
        invoice.setInvoiceDate(new Date());
        invoice.addLineItem("Product A", 10.99, "USD");
        invoice.addLineItem("Product B", 5.49, "USD");

        // Render the invoice to a PDF file
        template.render(invoice, "invoice.pdf");
    }
}
```
In this example, we create an `InvoiceTemplate` object from the 'invoice.xml' file and a sample `Invoice` data object with line items. We then pass these objects to the `render()` method of the `template` object, which generates the PDF invoice.

## Template Definition
Templates for Invoicer4J are defined in a simple XML layout format. This format allows you to specify placeholders for text and images, as well as define the layout structure of your invoice. Here's an example template:
```xml
<?xml version="1.0" encoding="UTF-8"?>
<invoice>
    <header>
        <logo src="/logo.png"/>
        <date>${invoiceDate}</date>
        <clientName>${clientName}</clientName>
    </header>
    <body>
        <table>
            <tr>
                <th>Product</th>
                <th>Quantity</th>
                <th>Unit Price</th>
                <th>Total</th>
            </tr>
            ${lineItems}
        </table>
        <footer>
            <total>${total}</total>
            <currency>${currency}</currency>
        </footer>
    </body>
</invoice>
```
In this example, we define a basic invoice template with placeholders for the logo, date, client name, and line items. The `${}` syntax is used to indicate where data should be inserted.

## Supported Currencies and Locale-Aware Number Formatting
Invoicer4J supports multiple currencies and locale-aware number formatting through the use of internationalization (I18N) libraries such as ICU or OpenJDK's built-in I18N support. This allows you to generate invoices with correctly formatted numbers, dates, and currencies for different regions.

## Known Limitations
One known limitation of Invoicer4J is its current inability to handle multi-page invoices. While we are working on improving this feature, it is not yet supported in the library.

For more information on using Invoicer4J or contributing to the project, please see our [GitHub page](https://github.com/example/invoicer4j).
