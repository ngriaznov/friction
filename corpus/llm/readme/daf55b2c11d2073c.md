# README for confkeep

## Introduction

**confkeep** is a lightweight Python library designed to parse INI-style configuration files while meticulously preserving comments and blank lines. This ensures that when you perform programmatic edits on a file and then write it back out, the original formatting remains intact. This feature distinguishes **confkeep** from the standard library's `configparser`, which does not preserve comments or blank lines by default.

## Motivation

The primary motivation behind **confkeep** is to provide users with greater flexibility in managing configuration files without losing their carefully crafted formatting. Many applications rely on INI-style configuration files, but traditional parsing methods often strip out comments and whitespace, leading to confusion and potential errors when re-applying these configurations. **confkeep** addresses this by ensuring that all original structure of the file is preserved during read and write operations.

## Installation

You can install **confkeep** via pip, which makes it easy to integrate into your projects:

```bash
pip install confkeep
```

## Basic Usage Example

Below is a concise example demonstrating how to read a configuration file, modify one of its values, and then write the changes back while preserving all original formatting.

### Step 1: Reading the Configuration

```python
from confkeep import ConfigParser

# Initialize the parser with your config file path
config = ConfigParser('path/to/your/config.ini')

# Access a configuration value
host = config.get('server', 'host')
port = int(config.get('server', 'port'))
```

### Step 2: Modifying a Value

```python
# Modify an existing value
config.set('server', 'host', 'localhost.localdomain')

# Change another setting
config.set('database', 'name', 'new_database_name')
```

### Step 3: Writing Back to the File

Simply call `save()` on your `ConfigParser` instance, and **confkeep** will write all changes back to the file while preserving comments and blank lines.

```python
# Save the modified configuration back to disk
config.save()
```

## Supported Syntax Quirks

**confkeep** is designed with real-world usage in mind, supporting several syntax quirks that are commonly found in configuration files:

### Inline Comments

You can add inline comments directly after key-value pairs using a `;` (semicolon):

```ini
[server]
host = localhost.localdomain ; Use local host for testing
port = 8080
```

**confkeep** will preserve these inline comments exactly as they appear.

### Multi-line Values

For values that span multiple lines, **confkeep** supports the continuation syntax using a backslash (`\`) at the end of each line:

```ini
[database]
name = \
    very\_long\_database\_name\_that\_spans\_multiple\_lines
```

When reading or writing these values, **confkeep** ensures that the multi-line nature is maintained.

### Section Inheritance

While not natively supported by standard INI files, **confkeep** can handle section inheritance through a simple syntax where sections can inherit properties from another section:

```ini
[server::production]
host = prod-server.example.com
port = 80

[server]
host = localhost.localdomain
port = 8080
```

In this example, `server::production` inherits values from `[server]`, allowing for a flexible configuration management system.

**confkeep** offers a robust solution for parsing and editing INI-style configuration files while preserving their original formatting.
