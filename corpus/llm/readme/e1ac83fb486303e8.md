## invoicer4j - Generate Professional Invoices from Java

**invoicer4j** is a Java library designed to simplify the process of generating professional PDF invoices directly from your application logic. It leverages an existing, robust PDF layout engine under the hood (currently based on [Specify Underlying Engine Here - e.g., iText] – details about the underlying engine and its licensing are provided in the documentation), allowing you to focus solely on defining your invoice structure and data without needing deep expertise in PDF generation.

**Key Features:**

* **Simple Template Definition:** Use a straightforward XML-based template format to define the layout of your invoices. No complex coding required.
* **Data-Driven Generation:** Easily populate your templates with data from Java objects, ensuring consistent and accurate invoice creation.
* **Currency & Locale Support:** Built-in support for multiple currencies and locale-aware number formatting, allowing you to generate invoices appropriate for any region.
* **Easy Integration:** Minimal code required for basic invoice generation – perfect for quickly adding invoicing functionality to your applications.

**Maven Dependency**

To include invoicer4j in your project, add the following dependency coordinates to your `pom.xml` (for Maven projects):

```xml
<dependency>
    <groupId>com.example</groupId> <!-- Replace with your groupId -->
    <artifactId>invoicer4j</artifactId>
    <version>1.0.0-SNAPSHOT</artifactId>  <!-- Replace with the latest version -->
</dependency>
```

**Minimal Code Sample**

This example demonstrates creating a simple invoice and rendering it to a PDF file:

```java
import com.example.invoicer4j.*; // Replace with your groupId
import java.io.File;

public class InvoiceGenerator {
    public static void main(String[] args) throws Exception {
        // 1. Define the invoice data
        InvoiceData invoiceData = new InvoiceData();
        invoiceData.setInvoiceNumber("INV-2023-123");
        invoiceData.setDate(new java.util.Date());
        invoiceData.setCustomerName("John Doe");
        invoiceData.setAddress("123 Main St");

        // 2. Define the line items
        invoiceData.addLineItem(new InvoiceLineItem("Product A", 10.00, 5));
        invoiceData.addLineItem(new InvoiceLineItem("Product B", 20.00, 3));

        // 3. Load the invoice template (XML)
        InvoiceTemplate template = new InvoiceTemplate("invoice_template.xml"); // Path to your XML template

        // 4. Generate the invoice PDF
        File outputFile = new File("invoice.pdf");
        InvoiceGeneratorUtil.generatePdf(invoiceData, template, outputFile);

        System.out.println("Invoice generated at: " + outputFile.getAbsolutePath());
    }
}
```

**Template Definition (XML)**

The templates are defined in XML format and specify the layout elements like text blocks, images, and positioning information.  A basic `invoice_template.xml` might look like this:

```xml
<?xml version="1.0" encoding="UTF-8"?>
<invoice>
    <header>
        <logo/> <!-- Placeholder for logo image -->
        <title>Invoice</title>
    </header>
    <customerInfo>
        <name>Customer Name: </name>
        <value>{invoiceData.getCustomerName()}</value>
        <address>Address: {invoiceData.getAddress()}</address>
    </customerInfo>
    <details>
        <number>Invoice Number: </number>
        <value>{invoiceData.getInvoiceNumber()}</value>
        <date>Date: {invoiceData.getDate().toString("yyyy-MM-dd")}</date>
    </details>
    <items>
        <title>Items:</title>
        <lineItemTemplate ref="line_item_template.xml"/> <!-- Use a separate template for line items -->
    </items>
    <footer/> <!-- Placeholder for footer information -->
</invoice>
```

**Supported Currencies & Locale-Aware Formatting**

invoicer4j supports multiple currencies through configurable locale settings.  You can specify the desired locale (e.g., `en_US`, `de_DE`, `fr_FR`) to ensure correct number formatting, date/time displays and currency symbols. The library leverages Java's built-in locale support for this purpose.

**Known Limitations - Multi-Page Invoices**

Currently, invoicer4j primarily focuses on single-page invoice generation. Handling multi-page invoices directly within the library is not yet supported. While you *can* achieve multi-page layouts using template design and by manually adding page breaks, it requires careful planning and might necessitate more advanced PDF manipulation techniques outside of the core invoicing functionality.  Future releases may include improved support for multi-page document generation.
