# multitail-go: A Go Command-Line Tool for Tailing Multiple Log Files

## Introduction

`multitail-go` is a powerful command-line tool written in Go that allows you to tail multiple log files simultaneously, making it easier to monitor and debug applications by viewing logs from different sources side by side. It supports color-coded labels for each file, glob pattern support for handling rotated logs, regex-based filtering of lines, and more.

## Installation

To install `multitail-go`, follow these steps:

1. **Ensure Go is installed**: Make sure you have the latest version of Go installed on your system. You can download it from [https://golang.org/dl/](https://golang.org/dl/).

2. **Install multitail-go**:
   ```sh
   go install github.com/alibaba/multitail-go@latest
   ```

3. **Verify the installation**:
   ```sh
   multitail-go --version
   ```
   This should display the version of `multitail-go` installed on your system.

## Basic Usage

To use `multitail-go`, you can provide a list of file paths as arguments to the tool:

```sh
multitail-go /path/to/app.log /path/to/db.log
```

This command will tail both logs and prefix each line with a color-coded label indicating which log it belongs to.

## Glob Pattern Support

`multitail-go` supports glob patterns for handling rotated or renamed log files. This is particularly useful in environments where log files are rotated daily, weekly, etc.

For example:

```sh
multitail-go /var/log/app/*.log
```

This command will tail all `.log` files in the `/var/log/app/` directory that match the pattern `*.log`.

## Filtering Lines with Regex

You can filter lines based on a regular expression using the `-r` or `--regex` option. This is useful for filtering out specific patterns of logs.

For example, to filter out all lines containing "error" in `/path/to/app.log`, you would use:

```sh
multitail-go -r 'error' /path/to/app.log
```

## Color Assignment

Colors are assigned to sources based on their file paths. By default, `multitail-go` uses a predefined color scheme where each source is assigned a different color.

You can customize the color assignment by specifying your own colors using the `-c` or `--color` option followed by a JSON object that maps file paths to colors.

For example:

```sh
multitail-go -c '{"/path/to/app.log":"red", "/path/to/db.log":"blue"}' /path/to/app.log /path/to/db.log
```

This command will assign the color red to `/path/to/app.log` and blue to `/path/to/db.log`.

## Advanced Usage

### Combining Options

You can combine multiple options for more advanced usage. For example:

```sh
multitail-go -c '{"/var/log/app/*.log":"green", "/var/log/db/*.log":"yellow"}' -r 'error' /var/log/app/*.log /var/log/db/*.log
```

This command will tail all logs in `/var/log/app/` and `/var/log/db/`, filter out lines containing "error", and color the app logs green and db logs yellow.

### Using Environment Variables

You can also set environment variables to configure `multitail-go`. For example:

```sh
export MULTITAIL_GO_COLOR='{"/path/to/app.log":"red"}'
export MULTITAIL_GO_REGEX='error'

multitail-go /path/to/app.log
```

This sets the color for `/path/to/app.log` and the regex filter before running `multitail-go`.

## Conclusion

`multitail-go` is a versatile tool that simplifies the process of monitoring multiple log files. Its features, such as color-coded labels, glob pattern support, and regex-based filtering, make it an invaluable asset for developers and system administrators.
