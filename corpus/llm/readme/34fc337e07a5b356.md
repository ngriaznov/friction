## Lumen: A Static Site Generator for Photography Portfolios

**Lumen** is a static site generator designed specifically for photographers looking to create beautiful, responsive online portfolios. Built with Ruby and leveraging the power of modern CSS frameworks, Lumen provides a streamlined workflow focused on showcasing your images and allowing you to concentrate on your photography. This gem simplifies the process of building a polished portfolio website without the need for complex templating engines or extensive JavaScript.

**Why Choose Lumen?**

* **Focused on Photography:** Designed with photographers’ needs in mind – optimized image handling, EXIF data extraction, and elegant gallery presentation.
* **Minimalist Workflow:**  No complicated templates or custom code required. Just images and Markdown.
* **Fast & Responsive:** Generates static HTML files that load quickly and look great across all devices.
* **Easy Configuration:** Manage your entire site with a single YAML configuration file.

---

**Getting Started**

1. **Installation:**

   ```bash
   gem install lumen
   ```

2. **Project Setup:**

   Create a new directory for your portfolio:

   ```bash
   mkdir my-portfolio
   cd my-portfolio
   ```

3. **Initialize the Project:**

   Run the following command to initialize the Lumen project:

   ```bash
   lumen init
   ```

---

**Folder Convention & File Structure**

Lumen expects a very simple folder structure, making it easy to maintain and understand:

* **`images/`**: This directory is crucial.  You'll create a subdirectory *within* this directory for *each gallery* you want to display on your portfolio. For example: `images/landscape`, `images/portrait`, `images/wedding`.
* **`_site/`**: Lumen generates the final, static HTML files here. You won’t directly modify these files.
* **`config.yml`**: This single YAML file manages all site-wide configurations (title, theme, output directory).

---

**File Structure within Galleries:**

Each gallery directory needs only a single `index.md` file containing the page content.  This file *must not* include front matter.

Example: Inside `images/landscape`:

```markdown
# Landscape Photography

This is where I showcase my landscape photography. Explore the vast beauty of nature!

![Image 1](images/landscape/image1.jpg)
![Image 2](images/landscape/image2.png)
```

---

**Build Command & Output**

To build your portfolio, run:

```bash
lumen build
```

This command will:

* Process all Markdown files in the `images/` directory.
* Generate thumbnails based on EXIF data (see below).
* Create static HTML files within the `_site/` directory.

The generated output is located in the `_site` folder.  You can then deploy this folder to any web server or CDN.


---

**Local Preview Server**

To preview your portfolio locally before deploying, run:

```bash
lumen serve
```

This will start a local development server and automatically open your portfolio in your default browser. Changes you make to the Markdown files will be reflected immediately.  You can stop the server by pressing `Ctrl+C`.


---

**Thumbnail Generation & EXIF Data Extraction**

Lumen automatically generates thumbnails from your images using their EXIF data (specifically, the width and height metadata). 

* **Supported Image Formats:** JPEG, PNG, TIFF
* **EXIF Data Requirements:** The image *must* have width and height information embedded within its EXIF tags for thumbnail generation to work correctly.  If this is missing, a default thumbnail size will be used (configurable in `config.yml`).

---

**EXIF-Based Caption Extraction**

Lumen cleverly extracts captions from your images by parsing the EXIF description field. This eliminates the need to manually add captions to your Markdown files – truly simplifying your workflow.  The extracted text is directly used as alt text for the image, improving accessibility and SEO.


---

**Configuration (config.yml)**

All site-wide configuration options are controlled through a single `config.yml` file. This file resides in the root of your project directory. Here's an example:

```yaml
title: "John Doe Photography"  # Your portfolio title
theme: "default"           # Choose from 'default', 'dark', or create your own theme (see documentation)
output_dir: "_site"         # The directory where the generated HTML files are placed
thumbnail_size: [300, 200]    # Thumbnail dimensions in pixels. Defaults to [400, 300].
```

* **`title`**:  The title that appears in the browser tab and potentially as a header on your site.
* **`theme`**: Allows you to switch between pre-defined themes for color palettes and styling. More themes can be added by creating custom CSS files.
* **`output_dir`**: The directory where the generated HTML files are placed.  Defaults to `_site`.
* **`thumbnail_size`**:  Specifies the dimensions (width, height) of the thumbnails generated from your images.  Adjust this to suit your preferences.
