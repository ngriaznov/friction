## wavtag: A C Library for WAV Metadata Manipulation

wavtag is a small, efficient C library designed to read and write metadata chunks (LIST/INFO tags) in WAV audio files *without* modifying the underlying audio sample data. This allows you to easily change artist names, titles, comments, and other information without impacting the sound itself.

**Key Features:**

* **Non-destructive Metadata Editing:** Modifies WAV file metadata directly, preserving the original audio data.
* **LIST/INFO Tag Support:** Primarily focuses on LIST and INFO tags, which are the most common metadata formats in WAV files.
* **Simple API:**  Easy to use functions for opening files, reading tags, and writing modified tags back.
* **Portability Considerations:** Handles byte-order (endianness) automatically for cross-platform compatibility.

**Build Instructions (using Make):**

1. **Dependencies:** Ensure you have a C compiler (e.g., GCC) installed.
2. **Create `Makefile`:** Create a file named `Makefile` with the following content:

```makefile
CC = gcc
CFLAGS = -Wall -fPIC  # Required for shared libraries if needed

all: wavtag.so

wavtag.so: wavtag.c
	$(CC) $(CFLAGS) -shared -o wavtag.so wavtag.c

clean:
	rm -f wavtag.so *.o
```

3. **Compile:** Run `make` in the directory containing `wavtag.c`. This will create a shared library file named `wavtag.so`.


**Public API Functions:**

* **`wavtag_open(const char *filename)`:** Opens a WAV file for reading or writing. Returns a `wavtag_context*` on success, NULL on failure.
* **`wavtag_close(wavtag_context *ctx)`:** Closes the file and releases resources associated with the context.
* **`wavtag_read_list(wavtag_context *ctx, char **language)`:** Reads the LIST tag from the WAV file. Returns a pointer to a dynamically allocated string containing the list data, or NULL if the tag is not found or an error occurs.  The caller must `free()` this string.
* **`wavtag_read_info(wavtag_context *ctx, char **language)`:** Reads the INFO tag from the WAV file. Returns a pointer to a dynamically allocated string containing the info data, or NULL if the tag is not found or an error occurs. The caller must `free()` this string.
* **`wavtag_write_list(wavtag_context *ctx, const char *language)`:** Writes the LIST tag to the WAV file.  Returns 0 on success, -1 on failure.
* **`wavtag_write_info(wavtag_context *ctx, const char *language)`:** Writes the INFO tag to the WAV file. Returns 0 on success, -1 on failure.

**Byte-Order Handling:**

wavtag automatically handles byte-order (endianness) of metadata fields within the WAV file for portability across different platforms (e.g., Intel x86 vs. ARM).  You don't need to explicitly convert data to big-endian or little-endian format.
