**Backupctl: Automated Nightly Backups for Your LAMP Stack Server**

Are you tired of manually backing up your server every night? Look no further than backupctl, a collection of bash scripts designed to automate the nightly backups of your LAMP stack server. With backupctl, you can rest assured that your database and web root are safely backed up every evening.

**Required Environment Variables**

Before running backupctl, you'll need to set three environment variables:

* `DB_HOST`: The hostname or IP address of your MySQL server.
* `DB_USER`: The username for the MySQL account used to dump databases.
* `DB_PASSWORD`: The password for the MySQL account used to dump databases.

These variables should be added to your system's shell configuration file (e.g. `/etc/bashrc` on Ubuntu/Debian) or set in a secure manner elsewhere in your environment.

**Installation**

To install backupctl, add the following cron entry to your crontab:
```bash
0 2 * * * /path/to/backupctl.sh > /dev/null 2>&1
```
This will run `backupctl.sh` every night at 2:00 AM. Make sure to replace `/path/to/backupctl.sh` with the actual path on your system.

**Backup Configuration**

By default, backupctl writes backups to a directory located at `/var/backups`. You can change this location by modifying the `BACKUP_DIR` variable in `backupctl.sh`.

Retention is configured using the `RETENTION` variable. This specifies how many daily backups should be retained before rotating old archives. For example, setting `RETENTION=7` will retain 7 daily backups and rotate the oldest one every night.

**Restore Script**

In case you need to restore your database or web root from a backup, simply run the `restore.sh` script:
```bash
/path/to/restore.sh <backup_date>
```
Replace `<backup_date>` with the date of the backup you wish to restore from. This will reverse the process and recover your database and web root.

**Notes**

* Make sure to test `backupctl.sh` before running it unattended.
* If you have a large number of databases, consider increasing the time allocated for backups or adjusting the retention period.
* Be aware that rotating old archives may overwrite existing files; make sure to verify the integrity of your backup repository.

With backupctl, you can rest assured that your LAMP stack server is properly backed up every night. Take advantage of this automation and save yourself from tedious manual backups!
