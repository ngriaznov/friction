Wavtag
================

A small C library for reading and writing metadata chunks (LIST/INFO tags) in WAV audio files.

Introduction
------------

This library allows you to access the artist, title, comment, and other metadata associated with a WAV file without modifying the audio sample data. It uses platform-agnostic byte-order handling to ensure portability across different operating systems.

Building
--------

To build wavtag, use the following command:

```bash
make
```

This will compile the library into `wavtag.so` (or similar on your platform) and create a header file `wavtag.h`.

Public API Functions
--------------------

### Opening a File

*   `wavtag_open(char *filename, int &filehandle)`:
    Opens a WAV file for reading or writing. Returns a file handle if successful.

```c
int wavtag_open(char *filename, int &filehandle) {
    // ...
}
```

### Reading Tags

*   `wavtag_read_tag(filehandle, char *tagname, size_t &tagsize)`:
    Reads the value of a specific tag from the WAV file. The tagname should be in lowercase.

```c
int wavtag_read_tag(int filehandle, char *tagname, size_t &tagsize) {
    // ...
}
```

*   `wavtag_read_all_tags(filehandle, char ***tags, size_t *tagcnt)`:
    Reads the metadata of the entire WAV file and returns an array of tags. The returned array should be freed by the caller.

```c
int wavtag_read_all_tags(int filehandle, char ***tags, size_t *tagcnt) {
    // ...
}
```

### Writing Modified Tags

*   `wavtag_write_tag(filehandle, const char *tagname, const char *value)`:
    Writes a modified tag value back to the WAV file.

```c
int wavtag_write_tag(int filehandle, const char *tagname, const char *value) {
    // ...
}
```

Note: The above functions use `const` correctness and null-terminated strings for simplicity. They do not handle invalid or malformed data; such cases should be checked by the caller.

Byte-Order Handling
-----------------

For portability across platforms, wavtag uses a byte-order handling scheme that is compatible with most systems. However, when working with specific file formats (like WAV), it's essential to consider endianness differences between different operating systems. The library will handle this internally, but you should be aware of these differences if you're developing cross-platform applications.

Limitations
------------

This library only handles the LIST and INFO tags in WAV files. More complex metadata structures might not be supported or may require custom modifications to wavtag.
