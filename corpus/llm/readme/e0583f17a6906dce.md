# WAVtag - Metadata Manipulation Library for WAV Files

## Overview

WAVtag is a lightweight C library designed to read and write metadata chunks (such as LIST, INFO tags) like artist name, album title, and comments within WAV audio files without altering the actual audio sample data. This makes it ideal for applications that need to manage or extract metadata from WAV files efficiently.

## Building WAVtag

To build WAVtag on your system, follow these simple steps:

1. **Clone the Repository**:  
   First, clone the WAVtag repository using Git:
   ```bash
   git clone https://github.com/yourusername/wavtag.git
   cd wavtag
   ```

2. **Install Dependencies**: Ensure you have a C compiler (like GCC) and `make` installed on your system.

3. **Build the Library**: Run the following command to compile WAVtag:
   ```bash
   make
   ```
   This will generate the `wavtag.so` shared library (on Unix-like systems).

## Public API

### Opening a File

To open an existing WAV file for metadata manipulation, use the `wavtag_open_file` function:

```c
#include "wavtag.h"

wavtag_t *file = wavtag_open_file(const char *filename);
if (!file) {
    // Handle error: unable to open file
}
```

### Reading Tags

To read existing tags from a WAV file, utilize the `wavtag_read_tag` function:

```c
const char *name;
int length;

wavtag_t *current = wavtag_first_tag(file);
while (wavtag_read_tag(current, &name, &length)) {
    // Process tag name and data as needed
}
```

### Writing Modified Tags

To write back modified tags to the WAV file, first clear existing tags with `wavtag_clear_tags`, then add new tags using `wavtag_add_text`:

```c
wavtag_t *file = wavtag_open_file("example.wav");
wavtag_clear_tags(file);

// Add new metadata tags
wavtag_add_text(file, "ARTIST", "New Artist Name", 16);
wavtag_add_text(file, "TITLE", "Updated Song Title", 20);
wavtag_add_text(file, "COMMENT", "This is an updated comment.", 30);

// Save changes back to the file
file->write_tags(file);
```

## Byte-Order Handling

WAVtag is designed with portability across different platforms in mind. It uses a consistent byte order for all metadata operations to ensure compatibility regardless of whether your system operates on little-endian or big-endian architectures. This means you can safely use WAVtag on any operating system without worrying about endian-specific issues.
