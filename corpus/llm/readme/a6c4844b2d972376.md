## multitail-go: Watch Multiple Log Files Simultaneously

**multitail-go** is a Go command-line tool that allows you to efficiently monitor multiple log files concurrently. It's particularly useful when observing the logs of an application server and its database, or any other set of interconnected services where synchronized viewing is beneficial.  

Unlike traditional `tail` commands, multitail-go color-codes each line based on its source file, providing instant visual clarity and facilitating quick identification of relevant information. 

**Key Features:**

* **Simultaneous Multi-File Tailing:** Observe multiple log files at once without needing to open several terminal windows.
* **Color-Coded Source Labels:** Each line is prefixed with a color that corresponds to its originating file, making it easy to distinguish between logs from different sources.
* **Glob Pattern Support:** Easily manage rotating log files by using glob patterns to match multiple files based on their names. 
* **Regex Filtering:** Focus on specific lines of interest by filtering output based on regular expressions.

**Installation:**

`multitail-go` is built as a self-contained Go binary. You can easily install it using the following command:

```bash
go install github.com/yourusername/multitail-go@latest
```

Replace `github.com/yourusername/multitail-go` with the actual repository URL if different.

**Basic Usage:**

To start tailing multiple log files, simply run `multitail-go` followed by a list of file paths:

```bash
multitail-go /var/log/app1.log /var/log/app2.log database.log
```

This will display the contents of `app1.log`, `app2.log`, and `database.log` simultaneously, with each line prefixed by its corresponding color label.

**Glob Pattern Support:**

Use glob patterns to match multiple files based on their names:

```bash
multitail-go /var/log/*.log
```

This will tail all log files within the `/var/log` directory.


**Regex Filtering:**

Filter lines based on regular expressions using the `-f` or `--filter` flag:

```bash
multitail-go -f "ERROR" /var/log/app1.log
```

This will only display lines containing the word "ERROR" from `app1.log`.



**Color Assignment:**

Log files are assigned colors based on their paths using a simple hashing algorithm. The first few characters of each path are used to generate a unique color code, ensuring distinct identification for different sources. You can customize these mappings if needed by modifying the internal logic within the `multitail-go` source code.



**Additional Notes:**

* `multitail-go` continues to monitor log files until manually interrupted (Ctrl+C).
* The tool supports ANSI escape codes for color rendering, ensuring compatibility with most terminals.
* For further customization and advanced usage scenarios, refer to the detailed documentation available within the repository.

Let `multitail-go` streamline your log monitoring experience and enhance your development workflow!
