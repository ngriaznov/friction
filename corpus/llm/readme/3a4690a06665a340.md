Backupctl
================

A collection of bash scripts designed to automate nightly backups of a LAMP stack server.

Environment Variables
--------------------

To use backupctl, you will need to set the following environment variables:

* `DB_USER`: The MySQL username used to connect to the database.
* `DB_PASSWORD`: The password for the MySQL user specified above.
* `DB_HOST`: The hostname or IP address of the MySQL server.
* `DB_NAME`: The name of the MySQL database to back up.
* `WEB_ROOT`: The path to the web root directory on the server.
* `BACKUP_DIR`: The directory where backups are written (default: `/var/backups`).
* `RETENTION`: The number of days to retain old backups (default: 30).

Cron Entry
------------

To schedule backupctl to run nightly, add the following cron entry:

```bash
0 2 * * * /usr/local/bin/backupctl
```

This will run backupctl at 2am every day.

Backup Script
-------------

The backup script performs the following tasks:

1. Dumps the MySQL database using `mysqldump`.
2. Rsyncs the web root directory to a new archive.
3. Rotates old archives by removing files older than the specified retention period.

Backups are written to the directory specified in `BACKUP_DIR`. The script also rotates old backups, keeping only the last `RETENTION` days' worth of data.

Restore Script
--------------

The restore script reverses the process:

1. Restores the MySQL database from a backup.
2. Rsyncs the web root directory from an archive.
3. Rotates new archives by removing files older than the specified retention period.

To use the restore script, simply run it with the following command:

```bash
/usr/local/bin/restorectl
```

This will restore the server to its state at the time of the last backup.

Configuration
-------------

The `RETENTION` variable controls how many days old backups are kept. You can adjust this value to suit your needs.

The `BACKUP_DIR` variable specifies where backups are written. If you want to change this location, simply update the value of this environment variable.

Troubleshooting
---------------

If you encounter any issues with backupctl, please check the following:

* Make sure that all required environment variables are set.
* Verify that the cron entry is correctly configured.
* Check the logs in `/var/log/backupctl.log` for any error messages.

Conclusion
----------

Backupctl provides a simple and efficient way to automate backups of your LAMP stack server. By setting the required environment variables and configuring the cron entry, you can ensure that your data is safely backed up every night. If you need to restore your server from a backup, simply run the restore script with the `restorectl` command.
