**Multitail-go: A Go Command-Line Tool for Tailing Multiple Log Files**

Multitail-go is a simple command-line tool written in Go that allows you to tail multiple log files at once, providing a clear and concise view of your application's logs. It's particularly useful when monitoring both the server and database logs side by side.

**Installation**

To install multitail-go, simply run:
```bash
go install github.com/your-username/multitail-go@latest
```
This will download and compile the tool, making it available in your system's executable path.

**Basic Usage**

The basic usage of multitail-go is straightforward. Simply provide a list of file paths as arguments to the command:
```bash
multitail-go /var/log/app.log /var/log/db.log
```
This will start tailing both files, prefixing each line with a color-coded label indicating its source.

**Glob Pattern Support**

For rotating logs or when dealing with multiple log files in different directories, you can use glob patterns to specify the files:
```bash
multitail-go /var/log/app.log.1.gz /var/log/db/*.log
```
The `*` character is a wildcard that matches any characters (except for `/`) in the file path.

**Filtering Lines with Regex**

To filter out lines based on a regular expression, use the `-f` or `--filter` flag followed by the regex pattern:
```bash
multitail-go /var/log/app.log -f 'error|warning'
```
This will only display lines containing the words "error" or "warning".

**Color Assignment**

Colors are assigned to sources based on a simple heuristic:

* Red is used for errors (default)
* Yellow is used for warnings (default)

If you want to customize the color assignment, create a `multitail-go.config` file in your system's configuration directory (`~/.config/multitail-go` by default) with the following format:
```json
{
  "colors": {
    "/var/log/app.log": "#00FF00", // Green for app logs
    "/var/log/db.log": "#0000FF"   // Blue for db logs
  }
}
```
**Configuration File**

The configuration file can contain multiple sections, each corresponding to a log source. The color assignment is done on a per-source basis.

You can also specify a custom prefix and suffix for each log source:
```json
{
  "colors": {
    "/var/log/app.log": "#00FF00",
    "/var/log/db.log": "#0000FF"
  },
  "sources": [
    {
      "path": "/var/log/app.log",
      "prefix": "[APP]",
      "suffix": ""
    },
    {
      "path": "/var/log/db.log",
      "prefix": "[DB]",
      "suffix": "(db)"
    }
  ]
}
```
This will prefix each line from the app log with `[APP]` and suffix it with an empty string, while prefixing each line from the db log with `[DB] (db)`.

**Conclusion**

Multitail-go is a lightweight command-line tool designed to simplify the process of monitoring multiple log files simultaneously. With its simple configuration file format and glob pattern support, you can easily customize its behavior to fit your specific use case.
