# Multitail-go README

## Overview

**Multitail-go** is a powerful command-line tool written in Go designed to streamline the process of monitoring multiple log files simultaneously. It's particularly useful for developers and system administrators who need to watch application server logs alongside database logs, providing an intuitive way to differentiate between sources directly from the terminal.

With **Multitail-go**, each line from any number of specified log files is prefixed with a color-coded label that identifies its source file. This feature enhances readability and efficiency when dealing with large volumes of log data, making it easier to spot issues or track specific events across different services.

## Installation

### Using `go install`

To get started with **Multitail-go**, you first need to have Go installed on your system along with the `go` command available in your PATH. Once that's set up, installation is straightforward:

```bash
go install github.com/yourusername/multitail-go@latest
```

Replace `yourusername` with the GitHub username where the repository resides. This command will download and compile the tool, placing it in a location accessible via your terminal commands.

## Basic Usage

The primary function of **Multitail-go** is to tail multiple log files at once. Here's how you can use it:

```bash
multitail -t /path/to/logfile1.log /path/to/logfile2.log
```

Replace `/path/to/logfile1.log` and `/path/to/logfile2.log` with the actual paths to your log files. The `-t` flag is optional but recommended for clarity.

### Glob Pattern Support

For logs that follow a rotation pattern (e.g., `logfile-2023-10-01.log`, `logfile-2023-10-02.log`), you can use glob patterns to specify multiple files dynamically:

```bash
multitail -t /var/log/myapp/*.log
```

This command will automatically tail all log files matching the specified pattern in the given directory, making it ideal for environments where logs are rotated daily or hourly.

### Filtering Lines by Regex

Sometimes you might only be interested in specific types of events within your logs. **Multitail-go** allows you to filter lines using regular expressions:

```bash
multitail -t /path/to/logfile1.log /path/to/logfile2.log --regex 'ERROR'
```

This command will display only the lines containing the word "ERROR" from both log files, reducing noise and focusing on critical events.

## Color Assignment to Sources

One of the standout features of **Multitail-go** is its ability to assign distinct colors to each source file. This visual differentiation helps users quickly identify which log belongs to which service without even looking at the label prefix.

By default, the tool assigns a unique color to each new log file you specify. If you have multiple instances of the same type of log (e.g., several application server logs), you can manually assign colors using the `-c` flag followed by a color name:

```bash
multitail -t /path/to/logfile1.log /path/to/logfile2.log -c red -c green
```

This command will label `/path/to/logfile1.log` with red and `/path/to/logfile2.log` with green, respectively. You can use any valid color name supported by your terminal environment (e.g., `red`, `blue`, `yellow`).

## Advanced Usage

### Customizing Labels

If you want to customize the label prefix that appears before each line, you can do so using the `-l` flag followed by a custom string:

```bash
multitail -t /path/to/logfile1.log /path/to/logfile2.log -l "[AppServer]"
```

This will prepend `[AppServer]` to every line from `/path/to/logfile1.log`, making it easier to distinguish between different application servers or services.

### Monitoring Multiple Directories

For scenarios where logs are scattered across multiple directories, you can specify a directory pattern:

```bash
multitail -t /var/log/*/*.log --regex 'CRITICAL'
```

This command will tail all `.log` files within `/var/log/`, filtering for lines containing "CRITICAL".

## Conclusion

**Multitail-go** is an essential tool for anyone working with multiple log sources, offering a clean, color-coded interface to monitor application and database logs side by side. Its support for glob patterns, regex filtering, and customizable labels makes it highly adaptable to various logging environments.

By following the installation instructions and basic usage guidelines outlined in this README, you can quickly start leveraging **Multitail-go** to enhance your log monitoring capabilities, leading to faster issue resolution and more efficient system management.
