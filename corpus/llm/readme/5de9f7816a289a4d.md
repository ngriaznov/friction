## backupctl: Automated Backups for Your LAMP Stack

backupctl is a collection of bash scripts designed to simplify nightly backups of your LAMP stack server. It automates the creation of MySQL database dumps, synchronizes your web root directory using rsync, and implements a robust archive rotation policy to ensure efficient storage utilization and data retention.

**Features:**

* **Automated Nightly Backups:** Scripts are configured to run automatically at night, ensuring consistent and timely backups.
* **Comprehensive Backup Scope:** Includes MySQL database dumps and a full rsync of your web root directory, capturing all essential website files.
* **Robust Archive Rotation:** Employs a configurable retention policy for backup archives, eliminating unnecessary data accumulation and freeing up disk space.

**Prerequisites:**

* A LAMP stack server with SSH access.
* Bash scripting environment.
* Basic understanding of command-line operations.

**Environment Variables:**

Before using backupctl, configure the following environment variables:

* `MYSQL_USER`: Your MySQL username.
* `MYSQL_PASSWORD`: Your MySQL password.
* `MYSQL_DATABASE`: The name of your database to be backed up.
* `BACKUP_ROOT`: The directory where backups will be stored.
* ` Retention_Days`: Number of days to retain backup archives (e.g., 7 for 7 days).

**Installation & Usage:**

1. **Clone the Repository:**

```bash
git clone https://github.com/your-username/backupctl.git
```

2. **Set Environment Variables:** Edit your shell configuration file (e.g., .bashrc) to define the environment variables as described above.

3. **Create Cron Job:** Add the following line to your crontab to schedule nightly backups:

```bash
0 2 * * * /path/to/backupctl/backup.sh
```

4. **Test & Verify:** Run `backup.sh` manually to test the backup process and ensure it executes successfully.

**Backup Directory Structure:**

Backups are organized in the `BACKUP_ROOT` directory with a hierarchical structure based on date:

```
BACKUP_ROOT/
├── YYYY-MM-DD/
│   └── mysql_dump_YYYY-MM-DD.sql.gz
│       └── webroot_YYYY-MM-DD.tar.gz
├── ...
```

**Restore Process:**

The `restore.sh` script provides a reverse functionality, allowing you to restore your website from a specific backup archive:

```bash
./restore.sh YYYY-MM-DD
```

This will overwrite your current files and database with the contents of the specified backup.
