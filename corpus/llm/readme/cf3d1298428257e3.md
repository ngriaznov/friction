## phodedupe: Eliminate Duplicate Photos with Perceptual Hashing

**phodedupe** is an open-source command-line tool written in Python that helps you identify and manage duplicate photos within your directories. Unlike traditional checksum comparison methods, **phodedupe** leverages perceptual hashing to detect duplicates even if images have been resized, slightly re-compressed, or otherwise subtly altered.

### Why Perceptual Hashing?

Traditional checksums compare byte-by-byte data of image files. This means that even minor changes like resizing or compression can result in drastically different checksums, leading to false negatives (missed duplicates) and significant wasted storage space. 

Perceptual hashing, on the other hand, focuses on capturing the **visual content** of an image rather than its raw byte representation. It generates a unique "fingerprint" based on the dominant patterns and features within an image. This allows **phodedupe** to accurately identify visually identical images even if they have different file sizes or formats.

### Installation

Install **phodedupe** using pip:

```bash
pip install phodedupe
```

### Basic Usage

1. **Scan for Duplicates:**

```bash
phodedupe -d /path/to/directory 
```

Replace `/path/to/directory` with the actual path to the directory containing your photos. This will generate a report of duplicate groups, each containing visually similar images.

2. **Review Report:** The report provides detailed information about each duplicate group, including:

* Number of images in the group
* File names and sizes of representative images
* Perceptual hash values (for technical analysis)

3. **Dry Run:** To preview the potential deletions without actually removing any files, use the `-n` flag:

```bash
phodedupe -d /path/to/directory -n 
```

4. **Delete Duplicates:** Once you've reviewed the report and are satisfied with the results, remove duplicates using the `-r` flag:

```bash
phodedupe -d /path/to/directory -r 
```

### Configuration Options

* `--hash-algorithm`: Choose the hashing algorithm to use (e.g., "average", "difference"). Default is "average".
* `--similarity-threshold`: Adjust the similarity threshold for duplicate detection (0-1). Higher values require a closer visual match between images. Default is 0.85.
