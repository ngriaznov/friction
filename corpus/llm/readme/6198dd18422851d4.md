# csvtype - Rust CSV Reader with Type Inference

`csvtype` is a Rust crate designed to read CSV files and automatically infer the types of each column (integer, float, date, boolean, or string). It then provides a typed iterator over records, making it easier to work with data that has mixed or unknown types. This README will guide you through installing `csvtype`, using it in your project, and understanding its features.

## Installation

To add `csvtype` as a dependency in your Rust project, include the following in your `Cargo.toml` file:

```toml
[dependencies]
csvtype = "0.2"
```

This versioning information may change depending on the latest release of `csvtype`. Always check the crate's documentation or GitHub page for the most current version.

## Example Usage

Here’s a simple example to demonstrate how `csvtype` can be used:

```rust
use csvtype::{CsvReader, InferenceOptions};
use std::fs;

fn main() -> Result<(), Box<dyn std::error::Error>> {
    // Load your CSV file content here or use the provided path.
    let contents = fs::read_to_string("example.csv")?;

    // Create a CsvReader instance with default inference options.
    let mut reader = CsvReader::new(&contents)?;

    // Infer the column types by sampling rows. By default, it samples 10% of the rows for type inference.
    let schema = reader.infer(InferenceOptions::default())?;

    // Print out the inferred schema
    println!("{:?}", schema);

    // Iterate over the records with typed fields based on the inferred schema.
    for row in reader.records() {
        if let Some(row) = row? {
            println!("{:?}", row);
        }
    }

    Ok(())
}
```

In this example, we first load a CSV file's content into a string. We then create an instance of `CsvReader` and use the `infer` method to estimate the column types based on sampling some rows from the data.

## Sampling Size

The accuracy of type inference can be influenced by how many rows are sampled for each column. By default, `csvtype` samples 10% of the total rows in a CSV file:

```rust
let schema = reader.infer(InferenceOptions::default())?;
```

However, you can customize this behavior:

```rust
// Specify your own sample size as a percentage.
let custom_options = InferenceOptions { sample_percent: Some(5) };
let schema = reader.infer(custom_options)?;
```

Alternatively, you might want to manually override the inferred types for specific columns. You can do this by setting `infer_type` in the `ColumnOptions`:

```rust
use csvtype::column_types::ColumnType;

// Infer the rest of the schema as usual.
let mut reader = CsvReader::new(&contents)?;
reader.infer(InferenceOptions::default())?;

// Set a specific type for a column, e.g., 2nd column to be treated as an integer.
reader.set_column_type(1, ColumnType::Int);

for row in reader.records() {
    if let Some(row) = row? {
        println!("{:?}", row);
    }
}
```

## Notes and Limitations

- **Performance**: The sampling method used for type inference can have a performance impact on large CSV files. Adjusting the `sample_percent` might be necessary to balance between accuracy and speed.
  
- **Edge Cases**: Certain edge cases, such as rows with mixed types within the same column or very specific date formats, may not be accurately inferred.

- **Extensibility**: The type inference logic can be extended or modified if needed. You can contribute improvements back to the crate.

By leveraging `csvtype`, you can streamline your data processing tasks by automatically handling complex CSV files with varying types in a more robust and efficient manner.
