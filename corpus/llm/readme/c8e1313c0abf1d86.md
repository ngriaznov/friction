## Feedwell: A Self-Hosted RSS & Atom Feed Reader

**Feedwell** is a lightweight, self-hosted RSS and Atom feed reader designed for simplicity and ease of use. Built from the ground up to run behind any WSGI server (like Gunicorn or uWSGI), it offers a clean web UI and powerful features without unnecessary bloat.  This guide will walk you through installation, configuration, and getting started with Feedwell.

**Key Features:**

* **Lightweight Web UI:** A responsive and intuitive interface for browsing and managing your feeds.
* **Folder Organization:** Organize your feeds into folders for better control and navigation.
* **Read/Unread Tracking:**  Easily track which articles you've read, allowing you to focus on what’s new.
* **Full-Text Search:** Powerful search functionality across the cached content of all your feeds. No more scrolling through endless lists!
* **OPML Import/Export:** Seamlessly import and export your feed subscriptions in OPML format – compatible with virtually any other RSS reader.
* **WSGI Compatibility:** Designed to run behind any WSGI server, giving you maximum flexibility for deployment.
* **Flexible Database Backend:** Choose between SQLite (default) or PostgreSQL for data storage.

**1. Installation**

Feedwell is packaged as a Python application, so installation requires a Python environment. We recommend using a virtualenv to isolate the project dependencies:

```bash
# Create and activate a virtualenv (replace "feedwell" with your desired name)
python3 -m venv feedwell-env
source feedwell-env/bin/activate  # On Linux/macOS
.\feedwell-env\Scripts\activate   # On Windows
```

**2. Installation from Source:**

1. **Clone the Repository:**
   ```bash
   git clone https://github.com/yourusername/feedwell.git
   cd feedwell
   ```

2. **Install Dependencies:**
   ```bash
   pip install -r requirements.txt
   ```

3. **Run Migrations:** This creates the necessary database tables:
   ```bash
   python manage.py migrate
   ```

---

**3. First-Run Setup & Admin Account Creation**

1. **Start the Development Server:**
   ```bash
   python manage.py runserver 0.0.0.0:8000
   ```
   This will start a development server, usually on port 8000.  Open your web browser and navigate to `http://localhost:8000`.

2. **Create an Admin Account:** The first time you run the application, it will prompt you to create an admin user. This account has full access to all Feedwell features. Enter a username, email address (optional), password, and confirm your password.  The admin account is crucial for managing your feeds and configurations.

---

**4. Database Configuration**

Feedwell supports two database backends: SQLite and PostgreSQL.

* **SQLite (Default):**  By default, Feedwell uses SQLite, which stores the data in a single file (`feedwell.db`). No configuration is needed if you stick with this option.
* **PostgreSQL:** For larger installations or increased performance, consider using PostgreSQL. 
    1. **Install PostgreSQL:** Ensure you have PostgreSQL installed and running on your server.
    2. **Create Database & User:** Create a database and a user specifically for Feedwell:
       ```bash
       sudo -u postgres createuser feedwell
       sudo -u postgres createdb feedwell -O feedwell
       ```
    3. **Configure Settings:** Edit the `feedwell/settings.py` file and update the following settings:
       * `DATABASE_ENGINE = 'django.db.backends.postgresql'` 
       * `DATABASE_NAME = 'feedwell'`
       * `DATABASE_USER = 'feedwell'`
       * `DATABASE_PASSWORD = 'your_password'`  (Replace with your PostgreSQL password)
       * `DATABASE_HOST = 'localhost'` (or the IP address of your PostgreSQL server)

---

**5. Feed Management & Configuration**

1. **Adding Feeds:** In the web UI, navigate to "Feeds" and click "Add Feed". Enter the URL of the RSS or Atom feed you want to subscribe to.
2. **Folder Management:** Create folders to organize your feeds logically.  You can manage folders through the "Folders" section in the navigation menu.
3. **Feed Settings:**  Each feed has a dedicated settings page where you can change its name, description, and other options.

---

**6. Scheduling Feed Polling with Cron**

Because Feedwell relies on periodically fetching new content from your feeds, it needs to be scheduled. We recommend using `cron` for this purpose (Linux/macOS) or Task Scheduler (Windows).

1. **Determine the Interval:**  How often do you want Feedwell to check for updates? Common intervals are every 5 minutes, 15 minutes, or hourly.

2. **Add a Cron Job:**
   * **Linux/macOS:** Open your crontab editor: `crontab -e`
   * Add a line similar to the following (example for checking every 15 minutes):
      ```
      */15 * * * * python3 /path/to/feedwell/manage.py poll_feeds > /dev/null 2>&1
      ```
     Replace `/path/to/feedwell/manage.py` with the actual path to your `manage.py` file.

   * Save and exit the crontab editor.  The cron daemon will automatically execute this command at the specified interval.
