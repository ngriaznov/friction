## confkeep: Preserving Comments & Formatting in INI Files

**confkeep** is a Python library designed to safely parse and modify INI-style configuration files while meticulously preserving all comments and blank lines. Unlike the standard library's `configparser`, which often strips formatting during parsing, **confkeep** ensures that round-tripping a file after programmatic edits doesn't result in an undesirable loss of user-defined structure.

### Motivation

`configparser` is a powerful tool for handling INI files. However, it tends to flatten the input, discarding comments and blank lines when converting to a structured dictionary representation. This can lead to unexpected behavior when writing back to file: valuable formatting notes might be lost, making future edits harder to understand or potentially breaking scripts relying on specific file layouts.

**confkeep** addresses this issue by employing a different approach. Instead of blindly parsing and reconstructing the file, it reads each line individually, carefully identifying sections, key-value pairs, comments, and blank lines. This meticulous handling preserves all original formatting elements, guaranteeing that modifications are reflected accurately when writing back to the file.

### Installation

**confkeep** is easily installed via pip:

```bash
pip install confkeep
```

### Usage Example

Let's consider a simple example `config.ini` file:

```ini
[General]
name = John Doe
# This is a comment about the name section
version = 1.2.3

[Database]
host = localhost
port = 3306
; Inline comment for port setting
username = user_data
password = secret!
```

Now, let's use **confkeep** to read this file, modify the `name` value in the `General` section, and write the changes back to a new file:

```python
from confkeep import ConfigFile

# Read the config file
config = ConfigFile('config.ini')

# Modify the name value
config['General']['name'] = 'Jane Doe'

# Write the modified config to a new file
config.write_to_file('modified_config.ini')
```

This code snippet demonstrates the core functionality of **confkeep**:

1.  **Initialization:** We create a `ConfigFile` object by providing the path to our `config.ini` file.
2.  **Modification:** We access the desired key-value pair using dictionary-like syntax (`config['General']['name']`) and update its value.
3.  **Writing Back:** The `write_to_file` method elegantly rewrites the modified configuration to a new file, preserving all original comments, blank lines, and formatting.

### Supported Syntax Quirks

**confkeep** gracefully handles various INI syntax quirks:

*   **Inline Comments:** Single-line comments starting with ';' or '#' are preserved.
*   **Multi-Line Values:** Values spanning multiple lines are correctly handled by identifying continuation lines based on indentation.
*   **Section Inheritance:** **confkeep** can detect and handle section inheritance (e.g., `[SectionA] ... [SectionB] extends=SectionA`) allowing you to manage configurations with nested sections and inherited settings.

### Conclusion

**confkeep** offers a robust solution for parsing and modifying INI configuration files while preserving the original formatting. Its meticulous handling of comments, blank lines, and various syntax quirks ensures accurate round-tripping and maintains the user's intended structure.
