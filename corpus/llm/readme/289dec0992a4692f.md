## confkeep: A Python Library for Robust INI Configuration Parsing

**confkeep** is a lightweight Python library designed to parse and manipulate INI-style configuration files while meticulously preserving their original formatting – including comments and blank lines. Unlike the standard `configparser` module, `confkeep` prioritizes round-tripping, ensuring that changes made programmatically are accurately reflected back into the original file with all its familiar layout intact.

**Motivation: Beyond Basic Parsing**

The standard Python `configparser` excels at reading and writing basic INI files, but it’s notoriously fragile when it comes to preserving formatting.  Any modifications – even a simple change to a single value – can easily corrupt the structure of the file if you then attempt to write it back out. This often leads to frustrating debugging sessions spent trying to recreate the original configuration from scratch.

`confkeep` addresses this problem by treating INI files as structured documents rather than just data stores. It specifically focuses on recreating the *file* exactly as it was, preserving whitespace, comments, and blank lines – crucial elements in many real-world configuration scenarios. This makes `confkeep` ideal for situations where you need to:

* **Maintain user-defined formatting:** Users often format INI files with specific indentation or spacing for readability.
* **Round-trip edits reliably:**  Make changes programmatically and then write the file back out without losing the original layout.
* **Handle complex configurations:**  Work with INI files that contain multi-line values, extensive comments, and potentially inherited sections.

**Example Usage**

Here's a basic example demonstrating how to read, modify, and write an INI file with `confkeep`:

```python
from confkeep import Config

# Create a sample INI file (config.ini)
# This will be parsed by the library later.
with open("config.ini", "w") as f:
    f.write("""
; This is a comment at the top of the file
[Section1]
  key1 = value1
  key2 = value2
  comment_after_value = another comment

[Section2]
  another_key = 123
  multi-line_value = 
    This is a multi-line
    value that spans multiple lines.
""")


# Read the configuration file
config = Config("config.ini")

# Modify a value
config["Section1"]["key1"] = "new_value"

# Write the updated configuration back to the file
config.write("config.ini")

print("Configuration file updated successfully!")
```

This code first creates a sample `config.ini` file with comments and multi-line values. Then, it reads this file using `confkeep`, modifies the value of `key1` in `Section1`, and finally writes the changes back to the same file, preserving all formatting aspects.


**Supported Syntax Quirks**

* **Inline Comments:**  `confkeep` fully supports inline comments starting with a semicolon (`;`). These comments are treated as whitespace and preserved during parsing and writing.
* **Multi-Line Values:**  Values can span multiple lines, using blank lines to separate them. `confkeep` correctly interprets these multi-line values.
* **Section Inheritance:**  Sections can inherit settings from parent sections. This allows for a hierarchical configuration structure. If a key is defined in a parent section and not in the current section, it will be inherited into the child section. The library handles this intelligently, ensuring that only the specific value defined in the child section overrides those inherited from its parents.
* **Whitespace Preservation:**  `confkeep` meticulously preserves whitespace (spaces, tabs) around keys, values, and within multi-line values. This is crucial for maintaining consistent formatting after modifications.
