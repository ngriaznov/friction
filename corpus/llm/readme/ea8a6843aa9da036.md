## multitail-go - Multi-Log Tailer

**multitail-go** is a Go command-line tool designed to simultaneously tail multiple log files and display them side-by-side with color-coded prefixes, making it ideal for monitoring application servers and their associated database logs. It’s particularly useful when you need to correlate events across different systems – for example, tracking web requests alongside database queries during a performance investigation.

**Features:**

*   **Simultaneous Tailing:** Monitors multiple log files concurrently.
*   **Color-Coded Prefixes:** Each line from a log file is prefixed with a color based on the source file's name, enhancing readability and quick identification of origins.
*   **Glob Pattern Support:** Supports glob patterns for automatically managing log rotation and tailing related files (e.g., `app/*.log`, `db/*.log`).
*   **Regex Filtering:** Filter lines based on regular expressions – exclude irrelevant entries or highlight specific events.
*   **Go Installable:** Easily deployable using Go's built-in installation mechanism.
*   **Simple and Efficient:** Designed for performance and a straightforward user experience.

---

### Installation

The easiest way to install `multitail-go` is through Go’s `go install` command:

```bash
go install github.com/yourusername/multitail-go@latest
```

Replace `github.com/yourusername/multitail-go` with the actual repository URL if you're using a different location.  Ensure your `$GOPATH/bin` directory is in your system’s PATH environment variable for the tool to be accessible from any terminal. You can usually find instructions on how to do this online based on your operating system (e.g., "add go bin path to PATH").

After installation, you should be able to run `multitail-go` from any directory.

### Basic Usage

The basic syntax is:

```bash
multitail-go [flags] <file1> [<file2> ...]
```

Where `<file1>`, `<file2>`, etc., are the paths to your log files.  You can include multiple file paths separated by spaces.

**Example:**

To tail `app.log` and `db.log`, you would run:

```bash
multitail-go app.log db.log
```

This will display the contents of both files in your terminal, with each line prefixed with a color based on its source file (e.g., lines from `app.log` will be colored one way, and those from `db.log` another).

---

### Key Flags & Options

*   `-f, --follow`:  Continuously tail files – automatically follow new log entries as they are appended. This is the default behavior.
*   `-s, --silent`: Suppresses the prefix coloring; just displays the raw lines from the logs. Useful for debugging color issues or when you only need to see the content.
*   `-h, --help`: Displays this help message and exits.
*   `-r, --regex <pattern>`:  Filters log lines using a regular expression.  Only lines matching the pattern will be displayed. Example: `-r "ERROR"` will filter for lines containing “ERROR”.  You can use multiple `-r` flags to apply multiple filters.
*   `-c, --colors <color_map>`:  Specifies a custom color map. This allows you to override the default colors assigned to file names. See section below on Color Assignment.

---

### Glob Pattern Support

`multitail-go` supports glob patterns for automatically tailing multiple log files. This is especially useful when dealing with log rotation schemes where log files are named sequentially (e.g., `app/access.log`, `app/error.log`).

**Example:**

```bash
multitail-go app/*.log db/*.log
```

This command will tail all `.log` files within the `app` directory and all `.log` files within the `db` directory.  The colors assigned to each file will be based on its name (e.g., `app.log` gets one color, `error.log` gets another).

---

### Color Assignment

By default, `multitail-go` assigns colors to source files based on their names using a simple mapping:

*   File names containing "app" or "server" are typically colored **green**.
*   File names containing "db" or "database" are typically colored **blue**.
*   Other file names are assigned a default color (usually **red**).

This mapping is configurable using the `-c` flag with a custom color map. The syntax for the color map is key-value pairs, where the key is the filename (or part of it) and the value is the desired color in ANSI escape code format.  You can specify multiple colors to cover different parts of your file names.

**Example:**

```bash
multitail-go -c "app:0;server:0;db:3" app.log db.log
```

In this example:

*   `app.log` will be colored green (0).
*   `server.log` will be colored green (0).
*   `db.log` will be colored blue (3).

You can use any valid ANSI escape code for colors. Some common codes include:

*   **Red:**  `\033[31m`
*   **Green:** `\033[32m`
*   **Blue:**  `\033[34m`
*   **Yellow:** `\033[33m`
*   **Magenta:** `\033[35m`
*   **Cyan:** `\033[36m`

Remember to include the escape code and the reset code (`\033[0m`) to return to the default color.  The reset code is crucial to prevent subsequent output from inheriting the specified color.
