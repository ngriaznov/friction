**packlite**
================

A lightweight Node.js command-line tool for bundling and minifying JavaScript and CSS assets in simple static sites.

**Installation**
---------------

To install packlite, run:

```bash
npm install -g packlite
```

**Configuration**
-----------------

Create a `packlite.config.js` file in the root of your project with the following structure:
```javascript
module.exports = {
  // Entry points for JavaScript and CSS files
  entry: {
    js: [
      'src/js/app.js',
      'src/js/utils.js'
    ],
    css: [
      'src/css/style.css',
      'src/css/layout.css'
    ]
  },
  // Output file names and paths
  output: {
    js: 'dist/bundle.js',
    css: 'dist/style.css'
  }
};
```
This example assumes you have a `js` and `css` directory containing your project's JavaScript and CSS files, respectively.

**Usage**
---------

Run packlite from the command line:

```bash
packlite build
```

This will bundle and minify your assets according to the configuration specified in `packlite.config.js`.

### Watch Mode

For local development, run packlite with the `-w` flag:

```bash
packlite -w build
```
packlite will monitor your files for changes and rebuild automatically.

### Source Maps

To generate source maps for debugging purposes, add the following option to your `packlite.config.js` file:
```javascript
module.exports = {
  // ... existing configuration ...
  sourcemap: true
};
```

**Comparison with Other Tools**
------------------------------

While heavier bundling tools like Webpack or Rollup can also bundle and minify assets, they often come with a higher overhead due to their complex feature sets. packlite is designed specifically for simple static sites, making it a lightweight alternative for projects that don't require the full-featured capabilities of these tools.

Here's a rough estimate of build times for various toolchains:

| Toolchain | Build Time (s) |
| --- | --- |
| packlite | 0.5-1.5 |
| Webpack | 2-5 |
| Rollup | 1.5-4 |

Keep in mind that these numbers are approximate and may vary depending on your project's size and complexity.
