**confkeep: Preserving Configuration File Formatting with Python**

When working with configuration files in Python, it's all too easy to lose the formatting and comments that made the file readable in the first place when editing programmatically. `confkeep` is a small library designed to mitigate this issue by preserving the original file structure while still allowing for programmatic modification of values.

**Why Choose confkeep Over configparser?**

Python's standard library includes `configparser`, which can parse and write INI-style configuration files. However, it has one major drawback: when writing a modified configuration back out, it discards any blank lines or comments not directly associated with the modified value. This makes it difficult to round-trip a file – making changes programmatically and then expecting the user's formatting to be preserved.

`confkeep`, on the other hand, is designed from the ground up to preserve the original formatting of the configuration file. It uses a simple, human-readable format that's easy to parse and modify programmatically, while still allowing for comments and blank lines to be maintained.

**Installation**

You can install `confkeep` via pip:
```bash
pip install confkeep
```
**Example Usage**

Here's an example of using `confkeep` to read a configuration file, modify a value, and write it back out:
```python
import confkeep

# Read the config from a file
config = confkeep.Config('example.ini')

# Modify a value
config['section1']['key'] = 'new value'

# Write the modified config back out
with open('example.ini', 'w') as f:
    config.write(f)
```
**Supported Syntax Quirks**

`confkeep` supports several syntax quirks to make working with configuration files easier:

* **Inline comments**: You can include inline comments in your configuration file by prefixing a line with `#`. These will be preserved when writing the modified config back out.
```ini
[section1]
key = value # This is an inline comment
```
* **Multi-line values**: If you need to store a large amount of text in a configuration option, you can use multi-line values by indenting each line with spaces. `confkeep` will preserve these when writing the modified config back out.
```ini
[section1]
key = This is a long value that spans multiple lines
  and needs to be preserved
```
* **Section inheritance**: If you have a section that inherits from another, you can specify this using the `include` keyword. For example:
```ini
[base]
key = value

[section1]
include = base
```
In this case, the `[section1]` section will inherit all values from the `[base]` section.

Overall, `confkeep` is designed to make working with configuration files in Python easier and more flexible. Its focus on preserving formatting and comments means you can round-trip your configuration files without worrying about losing valuable information.
