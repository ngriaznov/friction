Feedwell
================

A self-hosted RSS and Atom feed reader with a lightweight web UI, packaged to run behind any WSGI server.

Features
---------

### Folder Organization

* Feed articles are organized into folders which can be created manually or through the OPML import feature.
* Each folder can have its own set of read/unread tracking settings.

### Read/Unread Tracking

* Articles can be marked as read or unread, allowing users to keep track of their reading progress.
* Unread articles will appear in the "To Read" section of the feed reader's front page.

### Full-Text Search

* All cached articles are indexed for full-text search across all folders and tags.
* Users can search for specific keywords within an article or browse through a list of matching articles.

### OPML Import/Export

* Feedwell supports importing and exporting OPML files, allowing users to easily transfer their feed subscriptions between devices or services.
* The OPML import feature allows users to import feeds from popular services like Google Reader or Apple News.

Installation
-------------

To install Feedwell from source, follow these steps:

1. Clone the repository: `git clone https://github.com/username/feedwell.git`
2. Create a virtual environment: `python -m venv feedwell-env` (or `virtualenv feedwell-env`)
3. Activate the virtual environment: `source feedwell-env/bin/activate` (on Linux/Mac) or `feedwell-env\Scripts\activate` (on Windows)
4. Install dependencies: `pip install -r requirements.txt`
5. Run the migrations to create the database schema: `python manage.py migrate`

First-Run Setup
---------------

After installing Feedwell, you'll need to run the first-run setup script to create an admin account:

1. Navigate to the `manage.py` file in the root of the repository.
2. Run the following command to create a new admin user:
   ```bash
python manage.py createsuperuser --username admin --email admin@example.com --password 'your_admin_password'
```
3. Update your database configuration (see below for options) and run `python manage.py migrate` again.

Database Backend Options
------------------------

Feedwell supports the following database backends:

* **SQLite**: The default choice, SQLite is a lightweight and easy-to-use relational database that comes bundled with Python.
* **PostgreSQL**: A more powerful and flexible option, PostgreSQL is well-suited for larger feeds and more complex queries. To use PostgreSQL, you'll need to install it on your server and configure the database settings in `settings.py`.

Scheduling Feed Polling
----------------------

To keep Feedwell's internal feed aggregator up-to-date, you can schedule a cron job to run the following command at regular intervals:

* Windows:
  ```cmd
python manage.py feedpoll --interval=60
```
* Linux/Mac (using crontab):
  ```
0 * * * * python /path/to/feedwell/manage.py feedpoll --interval=60
```

This will poll the feeds every minute, refreshing the cached articles and keeping Feedwell's internal aggregator up-to-date.

Troubleshooting
-------------

If you encounter any issues during installation or setup, check the following:

* Check the database configuration to ensure it matches your chosen backend.
