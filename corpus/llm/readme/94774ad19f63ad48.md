# Invoicer4J README

## Overview

Invoicer4J is a Java library designed to simplify the generation of PDF invoices from templates combined with data objects. It leverages an existing PDF layout engine to ensure high-quality and customizable invoice outputs. This documentation covers Maven dependency integration, a minimal code example for constructing and rendering an invoice, template definition using XML layouts, supported currencies and locale-aware number formatting, as well as known limitations regarding multi-page invoices.

## Maven Dependency Coordinates

To incorporate Invoicer4J into your Java project, add the following Maven coordinates to your `pom.xml`:

```xml
<dependency>
    <groupId>com.example</groupId>
    <artifactId>invoicer4j</artifactId>
    <version>1.0.0</version>
</dependency>
```

*Note: Replace `com.example` with the actual Maven coordinates of Invoicer4J if available on a repository like Maven Central.*

## Minimal Code Sample

Below is an illustrative example demonstrating how to create an invoice with line items and render it to a PDF file:

```java
import com.example.Invoicer;
import com.example.model.Invoice;
import com.example.model.LineItem;

public class InvoiceGenerator {

    public static void main(String[] args) {
        // Create the invoice object
        Invoice invoice = new Invoice();
        invoice.setNumber("INV-001");
        invoice.setDate(new java.util.Date());

        // Define line items for the invoice
        LineItem item1 = new LineItem();
        item1.setDescription("Product A");
        item1.setQuantity(2);
        item1.setPrice(19.99);

        LineItem item2 = new LineItem();
        item2.setDescription("Product B");
        item2.setQuantity(1);
        item2.setPrice(29.50);

        invoice.setItems(Arrays.asList(item1, item2));

        // Configure the template (assuming a default template is available)
        Invoicer invoicer = new Invoicer();

        try {
            // Render the invoice to PDF
            String pdfPath = invoicer.render(invoice);
            System.out.println("Invoice rendered successfully: " + pdfPath);
        } catch (Exception e) {
            e.printStackTrace();
        }
    }
}
```

## Template Definition

Invoicer4J uses a simple XML format for defining templates. The template file maps placeholders to data fields, ensuring flexibility and ease of maintenance. Here's an example snippet:

```xml
<template>
    <header>
        <company name="Acme Corp"/>
        <address street="123 Main St" city="Anytown" state="CA" zip="90210"/>
    </header>

    <body>
        <invoiceNumber>${invoice.number}</invoiceNumber>
        <date>${invoice.date}</date>

        <items>
            ${invoice.items.forEach { item ->
                "<item>"
                    +"<description>${item.description}</description>"
                    +"<quantity>${item.quantity}</quantity>"
                    +"<price>${item.price}</price>"
                "</item>"
            }}
        </items>

        <total>${invoice.items.sumBy(item -> item.quantity * item.price)}</total>
    </body>
</template>
```

This XML structure allows for easy customization and maintenance of invoice layouts without altering the underlying Java code.

## Supported Currencies and Locale-Aware Number Formatting

Invoicer4J supports a wide range of currencies, enabling you to generate invoices compliant with international standards. For number formatting, it utilizes locale-aware formats, ensuring that monetary values are displayed correctly according to regional settings (e.g., using commas for decimals in Europe).

## Known Limitations Around Multi-Page Invoices

While Invoicer4J is designed to handle complex invoicing scenarios, including multi-page documents, there are limitations to be aware of:

1. **Template Constraints**: The current version of the template engine may not fully support dynamic page breaks based on content length. Manual adjustments or additional logic might be required for intricate layouts.
2. **Performance Considerations**: Generating large, complex invoices with many items or extensive text blocks can impact performance. Optimize by reducing unnecessary data processing and consider batch rendering where applicable.

Invoicer4J aims to streamline the process of generating professional PDF invoices from Java applications. By leveraging an existing PDF layout engine, it offers a robust solution that is both flexible and easy to integrate into your projects.
