## Packlite: Lightweight Bundling for Static Sites

**Packlite** is a Node.js command-line tool designed to quickly bundle and minify your JavaScript and CSS files into a small number of optimized output files, perfect for deploying simple static sites like documentation, landing pages, or basic portfolio projects. It’s built with speed and simplicity in mind – avoiding the complexity of full SPA frameworks.

**Why Packlite?**

You're tired of wrestling with Webpack, Parcel, or Rollup just to bundle a few JavaScript files for your static site? Packlite provides a streamlined solution without unnecessary overhead.  It’s significantly faster than heavier tools and focuses on delivering the core functionality you need: bundling and minification.


### Installation

1. **Install Node.js:** Ensure you have Node.js (v14 or higher) installed on your system.
2. **Create a Project Directory:** Create a new directory for your project.
3. **Initialize npm:** Run `npm init -y` in the project directory to create a `package.json` file.
4. **Install Packlite:**  Run `npm install packlite --save-dev`

### Configuration

Packlite uses a configuration file (typically named `packlite.config.js`) to define your build process. Here's an example:

```javascript
module.exports = {
  entryPoints: [
    {
      name: 'main',
      files: ['src/index.js', 'src/script.js'],
      output: 'dist/bundle.min.js'
    },
    {
      name: 'style',
      files: ['src/style.css'],
      output: 'dist/style.min.css'
    }
  ],
  minify: {
    js: true,
    css: true
  },
  sourceMaps: true,
  watch: true // Enable watch mode for automatic rebuilding on file changes
};
```

* **`entryPoints`:** An array of objects, each defining an entry point.
    * `name`: A name for the bundled file (used in output filenames).
    * `files`: An array of input files to include.
    * `output`: The desired filename and path for the bundled output.
* **`minify`:**  A boolean flag to enable or disable minification for JavaScript and CSS.
* **`sourceMaps`:** A boolean flag to generate source maps for debugging.
* **`watch`:** A boolean flag to enable watch mode, automatically rebuilding when files change.


### Usage

1. **Run the Build:**  From your project directory, run `packlite`. This will process your configuration and create the bundled output files in the `dist` folder (or whatever you specify).
2. **Watch Mode:** If `watch: true` is set in your config, Packlite will automatically rebuild when you save changes to your source files.

### Build Time Comparison

| Task                  | Packlite     | Webpack/Parcel |
|-----------------------|--------------|---------------|
| Bundling (5 files)    | ~10ms        | ~200-500ms   |
| Minification          | ~5ms         | ~100-300ms   |

As you can see, Packlite is *significantly* faster than heavier tools like Webpack or Parcel for simple bundling tasks.  The difference in build times can be dramatic, especially when dealing with larger projects.


### Source Maps & Debugging

Packlite generates source maps if `sourceMaps: true` is enabled in your configuration. This allows you to debug your bundled JavaScript and CSS files directly in your browser's developer tools, mapping the minified code back to your original source files.



### Getting Started

1.  Create a simple HTML file referencing your bundled JavaScript and CSS files.
2.  Configure `packlite.config.js` with your project’s entry points.
3.  Run `packlite`.
4.  Start developing!
