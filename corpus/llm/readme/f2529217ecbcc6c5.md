Feedwell
================

A self-hosted RSS and Atom feed reader with a lightweight web UI.

### Features

*   Folder organization: categorize feeds into folders for easier management.
*   Read/unread tracking: mark articles as read or unread, and track your progress.
*   Full-text search: quickly find specific articles across cached content using the built-in search function.
*   OPML import/export: transfer your feed subscriptions between different readers with ease.

### Installation

1.  **Clone the repository**: Use Git to clone the Feedwell repository onto your system:
    ```bash
git clone https://github.com/feedwell/feedwell.git /path/to/installation/directory
```
2.  **Set up a virtual environment**: Ensure you have pip installed, then create a new virtual environment using Python 3.x (currently recommended version) within the cloned repository:
    ```
python3 -m venv /path/to/venv
```
3.  **Activate the virtual environment**: Switch into your newly created virtual environment to manage dependencies without polluting your system's package database:
    ```bash
source /path/to/venv/bin/activate
```

### First-run Setup

1.  Navigate to the repository root and execute the following command to install Feedwell using pip:
    ```
pip install -r requirements.txt
```
2.  **Create an admin account**: Run the `setup.py` file with the `create-admin-account` option to establish a superuser, which will allow you to configure database settings and other critical options for your Feedwell instance:
    ```bash
python setup.py create-admin-account
```

### Database Backends

Feedwell supports two primary database backends:

*   **SQLite (default)**: A lightweight, self-contained solution suitable for small deployments or testing environments.
*   **PostgreSQL**: For larger-scale applications, PostgreSQL offers more robust performance and scalability.

To switch to PostgreSQL, you'll need to install the `psycopg2` package:
```bash
pip uninstall sqlite3
pip install psycopg2-binary
```
Update your database settings accordingly (consult [database configuration](#database-configuration) for details).

### Database Configuration

Once you've set up your desired database backend, update the `DATABASES` section in the `/feedwell/settings.py` file to reflect your chosen connection parameters:
```python
DATABASES = {
    'default': {
        'ENGINE': 'django.db.backends.sqlite3',
        'NAME': os.path.join(BASE_DIR, 'db.sqlite3'),
        'USER': '',
        'PASSWORD': '',
        'HOST': '',
        'PORT': ''
    }
}
```
If you're using PostgreSQL, modify the `ENGINE` to `'django.db.backends.postgresql_psycopg2'` and update the other connection parameters as necessary.

### Scheduling Feed Polling with Cron

1.  **Install cron**: Ensure your system has cron installed.
2.  **Configure cron job**: Create a new file in the `/etc/cron.d/` directory (or equivalent) to schedule feed polling at regular intervals:
    ```
crontab -e
```
3.  Add a line specifying the desired polling interval, such as every hour or daily:

```bash
0 * \* \* \* python /path/to/feedwell/manage.py poll_feeds
```

This setup assumes you've configured your Feedwell instance to use SQLite; adjust database connection parameters according to your chosen backend.

Feedwell is now ready for use. Access it by navigating to `http://localhost:8000` in a web browser (replace with the actual port number, if different).
