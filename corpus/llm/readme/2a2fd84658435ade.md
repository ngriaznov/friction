## phodedupe - Intelligent Duplicate Photo Finder

**phodedupe** is an open-source command-line tool designed to efficiently identify duplicate photos within one or more directories. It leverages perceptual hashing, a far superior technique compared to simple checksum comparisons, to accurately detect even subtly altered images like resized versions or slightly recompressed files. 

**Why Perceptual Hashing Over Checksums?**

Traditional checksums (like MD5 or SHA-256) are based on calculating the exact binary representation of a file.  Minor changes – resizing, compression adjustments, adding metadata – will drastically alter the checksum, leading to false negatives. Perceptual hashing (primarily using the AHash algorithm) works differently: it analyzes an image’s *content* rather than its raw data. It creates a fingerprint based on visual features like edges and colors, making it much more robust to these common image manipulations. This means phodedupe can reliably identify duplicates even if they've been slightly altered in size or quality.

**Installation**

phodedupe is packaged using `pip`:

```bash
pip install phodedupe
```

**Basic Usage**

1. **Scan a Folder:**  To scan a single directory, use:

   ```bash
   phodedupe scan /path/to/your/photos
   ```

   To scan multiple directories, you can specify them as arguments:

   ```bash
   phodedupe scan dir1 dir2 dir3
   ```

2. **Review the Report:**  After scanning, phodedupe generates a report detailing duplicate groups found in JSON format. This allows you to preview the matches before taking any action. The default output is `duplicates.json`.

3. **Deleting Duplicates (Proceed with Caution!)**  *Never* run this tool without reviewing the report first! To delete duplicates, use the `--delete` flag:

   ```bash
   phodedupe scan /path/to/your/photos --delete
   ```

   This will prompt you to confirm each deletion group before removing files. You can also specify a directory for the deleted files to be moved to with `-o <directory>`.

**Dry-Run Mode**

To preview the results without actually deleting anything, use the `--dry-run` flag:

```bash
phodedupe scan /path/to/your/photos --dry-run
```

This will display a report similar to the actual run but won't modify your file system.



**Configuration Options**

phodedupe offers several configuration options that can be set using command-line arguments or, eventually, through an optional config file:

*   `--hash-algorithm`:  Choose the hashing algorithm (default: `aHash`). Options include `aHash`, `dHash`.
*   `--similarity-threshold`: Sets the similarity threshold for grouping hashes. Lower values are more sensitive and may identify slightly different images as duplicates. The default is 0.95.
*   `-o <directory>` :  Specify a directory to move deleted files to.
*   `--verbose`:  Enable verbose output for debugging.

**Example:**

To perform a dry-run with `dHash` and a similarity threshold of 0.9:

```bash
phodedupe scan /path/to/your/photos --hash-algorithm dHash --similarity-threshold 0.9 --dry-run
```
