# Feedwell: Self-Hosted RSS & Atom Feed Reader

## Introduction

Feedwell is a lightweight, self-hosted web application designed to read and manage RSS and Atom feeds. It offers a clean, user-friendly interface that allows you to stay informed with your favorite news sources without relying on third-party services. Feedwell is packaged as a WSGI application, making it easy to deploy behind any WSGI server like Gunicorn or uWSGI.

## Features

### Folder Organization
Feedwell organizes feeds into folders for better categorization and management. You can create custom folders to group related feeds together, making navigation more intuitive.

### Read/Unread Tracking
Each article's read status is tracked automatically. Articles are marked as read when you navigate away from them or mark them as such manually. This feature helps you keep track of what you've already seen and what still needs attention.

### Full-Text Search Across Cached Articles
Feedwell includes a powerful full-text search functionality that allows you to quickly find articles based on keywords, titles, authors, or any other text content within the article body. The search is performed across all cached articles, providing an efficient way to locate specific information.

### OPML Import/Export
OPML (Outline Processor Markup Language) files are widely used for importing and exporting feed subscriptions. Feedwell supports both importing from and exporting to OPML files, making it easy to transfer your feed subscriptions between different devices or services.

## Installation

### Prerequisites
- Python 3.6+ installed on your system.
- A WSGI server (e.g., Gunicorn, uWSGI).
- SQLite for the default database backend.
- PostgreSQL support is optional but can be enabled if needed.

### Step-by-Step Guide to Install from Source with Virtualenv

1. **Create a virtual environment:**
   ```bash
   python3 -m venv feedwell_env
   source feedwell_env/bin/activate  # On Windows use `feedwell_env\Scripts\activate`
   ```

2. **Install the required dependencies:**
   ```bash
   pip install -r requirements.txt
   ```

3. **Clone the Feedwell repository:**
   ```bash
   git clone https://github.com/alibaba-cloud-platform/feedwell.git
   cd feedwell
   ```

4. **Run the first-run setup script to create an admin account:**
   ```bash
   python manage.py createsuperuser
   ```
   Follow the prompts to set up your admin user.

5. **Configure the database settings in `settings.py`:**
   - For SQLite (default):
     ```python
     DATABASES = {
         'default': {
             'ENGINE': 'django.db.backends.sqlite3',
             'NAME': BASE_DIR / 'db.sqlite3',
         }
     }
     ```
   - For PostgreSQL:
     ```python
     DATABASES = {
         'default': {
             'ENGINE': 'django.db.backends.postgresql',
             'NAME': 'your_database_name',
             'USER': 'your_username',
             'PASSWORD': 'your_password',
             'HOST': 'localhost',  # Or an IP Address that your DB is hosted on
             'PORT': '5432',
         }
     }
     ```

6. **Apply database migrations:**
   ```bash
   python manage.py migrate
   ```

7. **Collect static files (CSS, JavaScript):**
   ```bash
   python manage.py collectstatic
   ```

8. **Start the development server:**
   ```bash
   python manage.py runserver
   ```
   Visit `http://127.0.0.1:8000/` in your web browser to access Feedwell.

## Running with Cron for Scheduled Feed Polling

To ensure that feeds are regularly updated, you can use a cron job to periodically poll the feeds. Here’s how you can set it up:

### Step 1: Create a Management Command
Feedwell comes with a management command to handle feed polling. You can run this command manually or schedule it using cron.

1. **Run the management command to update all feeds:**
   ```bash
   python manage.py update_feeds
   ```

2. **Schedule the command using cron:**
   - Open your crontab file:
     ```bash
     crontab -e
     ```
   - Add a line to run the `update_feeds` command every hour (or as per your preference):
     ```cron
     0 * * * * /path/to/your/virtualenv/bin/python /path/to/feedwell/manage.py update_feeds >> /var/log/feedwell.log 2>&1
     ```
   - Replace `/path/to/your/virtualenv` and `/path/to/feedwell` with the actual paths on your system.

### Step 2: Configure Logging (Optional)
To keep track of feed updates, you can configure logging. The example above redirects output to a log file (`feedwell.log`). You can customize this by setting up a `logging.conf` file in your project directory or configuring Django’s logging settings.
