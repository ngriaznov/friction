**tasq: A Simple Command-Line Todo Manager**

tasq is a lightweight command-line todo manager that stores tasks as plain lines of text in a file. This approach keeps data readable and diffable, making it easy to track changes over time using version control.

**Getting Started**

1. Create a `tasks.txt` file in your project root or home directory.
2. Run `tasq add "your task description"` to create a new task.
3. Use `tasq complete <task number>` to mark a task as completed.

**Adding Tasks**

Add tasks using the following syntax:

```bash
$ tasq add "Buy milk"
```

tasq will append a numbered task line to your tasks file:
```
1. Buy milk
```

**Completing Tasks**

Mark a task complete with `tasq complete <task number>`:

```bash
$ tasq complete 1
```

This will prepend a strikethrough to the completed task line.

**Listing Tasks**

List all tasks using `tasq list`. You can filter by tag or due date using options like `-t` (tag) and `-d` (due date).

```bash
$ tasq list -t @urgent
```

This will display only tasks with the `@urgent` tag.

**Tag Syntax**

Use an `@` prefix to add tags to your tasks:

```bash
$ tasq add "Buy milk @groceries"
```

Tags can be comma-separated for multiple labels.

**Due-Date Syntax**

Specify a due date in the format `YYYY-MM-DD HH:MM`:

```bash
$ tasq add "Submit report by 2023-03-15 14:00"
```

tasq will display tasks sorted by their due dates.

**Configuring the Task File**

By default, tasq looks for a `tasks.txt` file in your project root or home directory. To point it elsewhere, set the `TASKS_FILE` environment variable:

```bash
$ export TASKS_FILE=/path/to/your/tasks/file.txt
```

Now, tasq will use this file instead of searching for the default location.

**Customization**

tasq's configuration is minimal and intended to be easy to read. If you need custom features or behavior, consider contributing to the project or using a separate tool in conjunction with tasq.

Enjoy using tasq to manage your tasks!
