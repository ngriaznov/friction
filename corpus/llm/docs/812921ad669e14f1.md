**Thistle Configuration File Format**

A Thistle configuration file is a simple YAML file that contains metadata about your site. It's where you define the global settings and overrides for individual pages. In this document, we'll break down the supported top-level keys, explain how to use front matter overrides, and provide two example config files.

**Top-Level Keys**

The following are the top-level keys you can use in your Thistle configuration file:

*   **title**: The title of your site.
*   **base_url**: The base URL for your site. This is used as a fallback when generating URLs for pages that don't have a slug.
*   **theme**: The name of the theme to use for your site. Themes are stored in the `themes` directory and contain CSS, JavaScript, and layout templates.
*   **output_dir**: The directory where Thistle will output the generated static files.

Here's an example of what the top-level keys might look like in a configuration file:

```yml
title: My Awesome Site
base_url: https://example.com/
theme: my-theme
output_dir: site
```

**Per-Page Front Matter Overrides**

You can override individual page settings by using front matter. Front matter is metadata that's placed at the top of each Markdown file, inside triple backticks. For example:

```yml
---
title: My Page
base_url: https://example.com/my-page/
---

# This is my page content
```

In this example, we're overriding the `title` and `base_url` settings for just this one page.

You can also use front matter to override settings for specific pages in your site. For instance:

```yml
---
title: My Page
base_url: https://example.com/my-page/
---

# This is my page content

---

---
title: Another Page
base_url: https://example.com/another-page/
---

# This is another page's content
```

This way, you can define different settings for each page without having to create a separate configuration file.

**Example Configuration Files**

Here are two complete example configuration files:

### Example 1: Simple Site

```yml
title: My Awesome Site
base_url: https://example.com/
theme: default-theme
output_dir: site

pages:
  - title: Home
    base_url: /
    layout: home.html
  - title: About
    base_url: /about/
    layout: about.html
```

This configuration file defines a simple site with two pages: Home and About. The `layout` key specifies which template to use for each page.

### Example 2: Site with Custom Theme

```yml
title: My Awesome Site
base_url: https://example.com/
theme: my-theme
output_dir: site

pages:
  - title: Home
    base_url: /
    layout: home.html
  - title: About
    base_url: /about/
    layout: about.html
```

This configuration file defines a site that uses the `my-theme` theme. The rest of the settings are the same as in the previous example.

These examples should give you an idea of how to configure Thistle for your static site generator needs. Happy building!
