## Ratchet Task Queue API Reference

Ratchet is a robust and scalable task queue service designed for handling asynchronous workloads efficiently. This document provides a comprehensive reference guide to the REST API endpoints available for interacting with Ratchet.

**Base URL:** `https://api.ratchet.example.com/v1`

All requests should be made using HTTPS and include an appropriate authorization header (see Authentication section).

### 1. Enqueuing Jobs

To add a new task to the queue, use the POST request to the `/jobs` endpoint. The request body must contain a JSON object with the following fields:

* **name:** A unique identifier for the job.
* **data:** Any data required by the worker function to execute the task. This can be a string, number, array, or any other valid JSON structure.
* **priority:** (Optional) An integer representing the priority of the job. Higher numbers indicate higher priority. Defaults to 10.

**Example Request:**

```bash
curl -X POST \
  https://api.ratchet.example.com/v1/jobs \
  -H "Authorization: Bearer your_access_token" \
  -d '{
    "name": "process_image",
    "data": {
      "image_url": "https://example.com/image.jpg"
    },
    "priority": 20
  }'
```

**Example Response (Success):**

```json
{
  "id": "12345678-9abcdef0-1234-5678-90abcdef0",
  "name": "process_image",
  "status": "PENDING",
  "created_at": "2023-10-26T10:00:00Z"
}
```

**Example Response (Error - Invalid data):**

```json
{
  "error": {
    "code": "INVALID_DATA",
    "message": "Invalid data format. 'data' field must be a JSON object."
  }
}
```

### 2. Checking Job Status

To retrieve the status of a specific job, use the GET request to the `/jobs/{job_id}` endpoint, replacing `{job_id}` with the unique identifier returned when the job was enqueued.

**Example Request:**

```bash
curl -X GET \
  https://api.ratchet.example.com/v1/jobs/12345678-9abcdef0-1234-5678-90abcdef0 \
  -H "Authorization: Bearer your_access_token"
```

**Example Response (Success - PENDING):**

```json
{
  "id": "12345678-9abcdef0-1234-5678-90abcdef0",
  "name": "process_image",
  "status": "PENDING",
  "created_at": "2023-10-26T10:00:00Z",
  "updated_at": "2023-10-26T10:05:00Z"
}
```

**Example Response (Success - COMPLETED):**

```json
{
  "id": "12345678-9abcdef0-1234-5678-90abcdef0",
  "name": "process_image",
  "status": "COMPLETED",
  "created_at": "2023-10-26T10:00:00Z",
  "updated_at": "2023-10-26T10:10:00Z",
  "result": {
    "message": "Image processing successful."
  }
}
```

**Example Response (Error - Job not found):**

```json
{
  "error": {
    "code": "JOB_NOT_FOUND",
    "message": "Job with ID '12345678-9abcdef0-1234-5678-90abcdef0' not found."
  }
}
```

### 3. Cancelling Pending Jobs

To cancel a job before it is executed, use the DELETE request to the `/jobs/{job_id}` endpoint. This will mark the job as cancelled and prevent it from being processed by workers.

**Example Request:**

```bash
curl -X DELETE \
  https://api.ratchet.example.com/v1/jobs/12345678-9abcdef0-1234-5678-90abcdef0 \
  -H "Authorization: Bearer your_access_token"
```

**Example Response (Success):**

```json
{
  "message": "Job cancelled successfully."
}
```

**Example Response (Error - Job not found):**

```json
{
  "error": {
    "code": "JOB_NOT_FOUND",
    "message": "Job with ID '12345678-9abcdef0-1234-5678-90abcdef0' not found."
  }
}
```

**Example Response (Error - Job already completed):**

```json
{
  "error": {
    "code": "JOB_ALREADY_COMPLETED",
    "message": "Job with ID '12345678-9abcdef0-1234-5678-90abcdef0' has already been completed."
  }
}
```



### Authentication

All API requests require authentication using a Bearer token. To obtain a token, you can use the OAuth 2.0 protocol or contact Ratchet support for API key credentials. Include the token in the `Authorization` header of your requests:

```
Authorization: Bearer your_access_token
```


**Note:** This documentation provides a basic overview of the Ratchet API. For detailed information on specific endpoints, parameters, and response formats, please refer to the official Ratchet API documentation available at [https://docs.ratchet.example.com/](https://docs.ratchet.example.com/).
