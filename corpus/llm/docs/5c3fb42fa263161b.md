## How to Configure a Reverse Proxy with Wharfgate for Multi-Backend Routing

Wharfgate is a lightweight, open-source load balancer perfect for simple use cases like routing traffic to different backend services based on URL path prefixes. This guide walks you through setting up a reverse proxy with Wharfgate, effectively managing traffic flow to three distinct backend services.

**Prerequisites:**

* A server running Linux (tested on Ubuntu)
* Basic familiarity with command line interface and text editors
* Three backend services accessible via their respective IP addresses or domain names

**Steps:**

1. **Install Wharfgate:**

   ```bash
   sudo apt update && sudo apt install wharfgate
   ```

2. **Create a Configuration File:**

   Create a configuration file named `wharfgate.conf` in the `/etc/wharfgate/` directory. This file defines how Wharfgate handles incoming traffic. 

3. **Sample Configuration (wharfgate.conf):**

```
# Define default backend for unmatched paths
default_backend = my-default-backend

# Configure backends and their corresponding URL prefixes
backends:
  backend1:
    url: http://backend1-ip:8080
    prefix: /api/v1
  backend2:
    url: http://backend2-ip:8080
    prefix: /static
  backend3:
    url: http://backend3-ip:8080
    prefix: /blog

# Listen on port 80 for HTTP traffic
listen:
  port: 80
```

* **`default_backend`:** Specifies the backend to use if no matching prefix is found. Replace `my-default-backend` with the actual backend name defined below.

* **`backends:`**: Defines each backend service with its URL and corresponding URL prefix.

    * **`url:`**: The address of your backend service (replace placeholders).
    * **`prefix:`**: The URL path prefix that triggers routing to this backend. For example, requests starting with `/api/v1` will be directed to `backend1`.
* **`listen:`**:  Configures Wharfgate to listen on port 80 for incoming HTTP traffic. Adjust as needed.

4. **Start Wharfgate:**

   ```bash
   sudo systemctl enable wharfgate
   sudo systemctl start wharfgate
   ```

5. **Test Your Configuration:**

   Access your website through the configured domain or IP address (e.g., `http://your-domain.com`). Different requests based on URL prefixes should now be routed to the appropriate backend services. 

**Health Checks:**

Wharfgate offers basic health checks for your backends. To configure a health check, add the following section within each backend definition:

```
health_check:
  path: /health
  interval: 5s # Check every 5 seconds
  timeout: 1s # Wait for a maximum of 1 second for a response
```

* **`path:`**: The URL path on your backend that represents its health status.
* **`interval:`**: Time interval between consecutive health checks.
* **`timeout:`**: Maximum time allowed for a successful health check response.

Wharfgate will automatically failover to other backends if the current one fails the health check.


**Note:** This guide provides a basic example. Wharfgate offers many advanced features like SSL termination, rate limiting, and detailed logging. Refer to the official documentation for further customization options: https://wharfgate.io/docs/.
