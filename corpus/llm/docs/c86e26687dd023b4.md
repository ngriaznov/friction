**Setting Up Automatic Database Backups for Corkboard**

As the administrator of your self-hosted note-taking app, Corkboard, it's essential to ensure that your users' data is safe in case of any unexpected issues or disasters. One crucial step towards achieving this goal is setting up automatic database backups. In this guide, we'll walk you through the process of configuring cron scheduling, selecting a backup storage location, and restoring from a backup file.

**Prerequisites**

Before starting, ensure that:

1. You have a self-hosted instance of Corkboard installed on your server.
2. You have access to the command line interface (CLI) or terminal.
3. You have a basic understanding of Linux/Unix commands and cron scheduling.

**Step 1: Choose a Backup Storage Location**

Decide where you want to store your backups. Options include:

* Local storage (e.g., /var/backups/corkboard)
* External hard drive or NAS
* Cloud storage services like AWS S3, Google Cloud Storage, or Microsoft Azure Blob Storage

For this guide, we'll assume local storage on the server.

**Step 2: Create a Backup Script**

Create a new file called `backup_corkboard.sh` in the `/usr/local/bin/` directory (or any other location of your choice). Add the following code to the file:
```bash
#!/bin/bash

# Set backup location and database name
BACKUP_LOCATION=/var/backups/corkboard
DB_NAME=corkboard

# Create backup directory if it doesn't exist
mkdir -p $BACKUP_LOCATION

# Dump database to a SQL file
mysqldump -u [username] -p[password] $DB_NAME > $BACKUP_LOCATION/$(date +'%Y-%m-%d_%H-%M-%S').sql

# Compress the backup file (optional)
tar -czf $BACKUP_LOCATION/backup_$(date +'%Y-%m-%d_%H-%M-%S').tgz $BACKUP_LOCATION/$(date +'%Y-%m-%d_%H-%M-%S').sql
```
Replace `[username]` and `[password]` with your actual database credentials.

**Step 3: Make the Script Executable**

Run the following command to make the script executable:
```bash
chmod +x /usr/local/bin/backup_corkboard.sh
```
**Step 4: Schedule the Backup using Cron**

Open the crontab editor by running `crontab -e` in the terminal. Add the following line to schedule the backup to run daily at 2 AM:
```bash
0 2 * * * /usr/local/bin/backup_corkboard.sh
```
This will execute the `backup_corkboard.sh` script every day at 2 AM.

**Step 5: Verify the Backup**

After scheduling the backup, verify that it's working correctly by checking the `/var/backups/corkboard` directory for new files. You should see a SQL file with a timestamp and an optional compressed tarball.

**Restoring from a Backup File**

To restore your database from a backup file:

1. Stop the Corkboard service to prevent any further writes to the database.
2. Move or copy the backup file to the `/var/backups/corkboard` directory (if it's not already there).
3. Run the following command to import the SQL file into your database:
```bash
mysql -u [username] -p[password] corkboard < /var/backups/corkboard/backup_$(date +'%Y-%m-%d_%H-%M-%S').sql
```
Replace `[username]` and `[password]` with your actual database credentials.

**Conclusion**

By following these steps, you've successfully set up automatic database backups for your self-hosted Corkboard instance. Regular backups ensure that your users' data is safe in case of any unexpected issues or disasters. Remember to test the backup and restore process periodically to ensure everything works as expected.
