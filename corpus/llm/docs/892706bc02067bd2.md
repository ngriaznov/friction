# Reference Page for Ratchet Task-Queue Service API

## Overview

Ratchet is an advanced task-queue service designed to handle asynchronous jobs efficiently. This document provides detailed information on the RESTful API endpoints used for interacting with the Ratchet service, including enqueuing a job, checking job status, and canceling a pending job. Each section includes example `curl` requests and corresponding JSON responses.

## Endpoint Documentation

### 1. Enqueue Job

#### URL:
```
POST /api/v1/jobs
```

#### Description:
Enqueues a new job in the Ratchet task-queue service.

#### Request Body (JSON):
```json
{
    "job_type": "string",
    "payload": {
        // Additional data related to the job type
    },
    "metadata": {
        // Metadata about the job, e.g., user_id: 12345
    }
}
```

#### Example `curl` Request:
```bash
curl -X POST \
     -H 'Content-Type: application/json' \
     -d '{
           "job_type": "email_notification",
           "payload": {
               "user_id": 12345,
               "subject": "Monthly Report",
               "body": "Please review the latest report."
           },
           "metadata": {
               "priority": "high"
           }
         }' \
     https://ratchet.example.com/api/v1/jobs
```

#### JSON Response:
```json
{
    "job_id": 98765,
    "status": "enqueued",
    "queue_name": "default"
}
```

### 2. Check Job Status

#### URL:
```
GET /api/v1/jobs/{job_id}
```

#### Description:
Retrieves the status of a specific job in the queue.

#### Path Parameters:
- `job_id`: Unique identifier for the job (required)

#### Example `curl` Request:
```bash
curl -X GET \
     https://ratchet.example.com/api/v1/jobs/98765
```

#### JSON Response:
```json
{
    "job_id": 98765,
    "status": "completed",
    "result": {
        // Result of the job execution, if any
    },
    "queue_name": "default"
}
```

### 3. Cancel Pending Job

#### URL:
```
DELETE /api/v1/jobs/{job_id}
```

#### Description:
Cancels a pending or enqueued job.

#### Path Parameters:
- `job_id`: Unique identifier for the job (required)

#### Example `curl` Request:
```bash
curl -X DELETE \
     https://ratchet.example.com/api/v1/jobs/98765
```

#### JSON Response:
```json
{
    "job_id": 98765,
    "status": "canceled",
    "reason": "Cancelled by user request"
}
```

### Error Handling

All API responses will follow a standardized error handling format. Here is an example of the structure for errors:

#### Example `curl` Request with Error:
```bash
curl -X DELETE \
     https://ratchet.example.com/api/v1/jobs/98765
```
#### JSON Response (Error):
```json
{
    "error": {
        "code": 404,
        "message": "Job not found",
        "details": [
            "The job with the specified ID does not exist."
        ]
    }
}
```

### Additional Notes

- **Authentication**: This API supports both Basic Authentication and OAuth2. For more information, refer to the Ratchet documentation on authentication.
- **Rate Limiting**: The service limits requests per second to prevent abuse. Detailed rate limiting rules can be found in the Ratchet API guidelines.
- **Documentation Updates**: Regular updates will be made to this document as new features are added or changes are implemented.
