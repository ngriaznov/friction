Ratchet Task Queue Service Reference Page
=============================================

Introduction
------------

The Ratchet task queue service provides a REST API for managing jobs in the system. This document outlines the available endpoints, including enqueuing a job, checking job status, and canceling a pending job.

Endpoint 1: Enqueue Job
----------------------

### Description

Enqueue a new job in the system.

### Request Parameters

*   `job_name`: The name of the job to be enqueued.
*   `task_data`: The data associated with the task.
*   `priority`: The priority of the task (optional).
*   `retry_policy`: The retry policy for the task (optional).

### Example Request

```bash
curl -X POST \
  http://localhost:8080/enqueue_job \
  -H 'Content-Type: application/json' \
  -d '{"job_name": "my_task", "task_data": {"key": "value"}, "priority": 1, "retry_policy": "at_least_once"}'
```

### Response

```json
{
  "job_id": 123,
  "created_at": "2023-02-20T14:30:00.000Z",
  "status": "pending"
}
```

Endpoint 2: Get Job Status
-------------------------

### Description

Check the status of a job in the system.

### Request Parameters

*   `job_id`: The ID of the job to check.

### Example Request

```bash
curl -X GET \
  http://localhost:8080/job_status/123 \
  -H 'Content-Type: application/json'
```

### Response

```json
{
  "job_id": 123,
  "status": "running",
  "created_at": "2023-02-20T14:30:00.000Z"
}
```

Endpoint 3: Cancel Job
----------------------

### Description

Cancel a pending job in the system.

### Request Parameters

*   `job_id`: The ID of the job to cancel.

### Example Request

```bash
curl -X DELETE \
  http://localhost:8080/cancel_job/123 \
  -H 'Content-Type: application/json'
```

### Response

```json
{
  "message": "Job cancelled successfully",
  "status": "cancelled"
}
```

Additional Endpoints
--------------------

### Enqueue Job with Retry Policy

Enqueue a job with a custom retry policy.

#### Request Parameters

*   `job_name`: The name of the job to be enqueued.
*   `task_data`: The data associated with the task.
*   `priority`: The priority of the task (optional).
*   `retry_policy`: The retry policy for the task.

#### Example Request

```bash
curl -X POST \
  http://localhost:8080/enqueue_job \
  -H 'Content-Type: application/json' \
  -d '{"job_name": "my_task", "task_data": {"key": "value"}, "priority": 1, "retry_policy": {"attempts": 3}}'
```

#### Response

```json
{
  "job_id": 123,
  "created_at": "2023-02-20T14:30:00.000Z",
  "status": "pending"
}
```

### Get Job Status with Detailed Information

Get the detailed status of a job in the system.

#### Request Parameters

*   `job_id`: The ID of the job to check.

#### Example Request

```bash
curl -X GET \
  http://localhost:8080/job_status/123?include_details=true \
  -H 'Content-Type: application/json'
```

#### Response

```json
{
  "job_id": 123,
  "status": "running",
  "created_at": "2023-02-20T14:30:00.000Z",
  "progress": {
    "percentage": 50
  }
}
```

Error Handling
--------------

The Ratchet task queue service returns HTTP error codes and JSON responses with detailed error messages.

*   `400 Bad Request`: Invalid request parameters or body.
*   `401 Unauthorized`: Authentication failed or token is invalid.
*   `404 Not Found`: Job not found.
*   `500 Internal Server Error`: Server-side error.

Security Considerations
-----------------------

The Ratchet task queue service uses HTTPS to encrypt data in transit. It also implements rate limiting and IP blocking to prevent abuse and denial-of-service attacks.

Conclusion
----------

The Ratchet task queue service provides a robust and scalable solution for managing jobs in the system. This reference page documents the available endpoints, including enqueuing a job, checking job status, and canceling a pending job. By following this guide, developers can leverage the Ratchet service to build high-performance applications that rely on task queues.
