By the end of this guide, you should have a functional webhook server that can handle and respond to incoming requests.

### Prerequisites

Before we begin, ensure you have the following tools installed:

1. **Node.js**: You can download it from [nodejs.org](https://nodejs.org/).
2. **npm (Node Package Manager)**: Comes bundled with Node.js.
3. A text editor or IDE of your choice.

### Step 1: Setting Up Your Project

First, create a new directory for your project and navigate into it:

```bash
mkdir webhook-receiver
cd webhook-receiver
```

Initialize a new Node.js project by running:

```bash
npm init -y
```

This command will generate a `package.json` file in your project root.

### Step 2: Installing Hookline

Hookline is a middleware library for handling webhooks. Install it using npm:

```bash
npm install hookline
```

### Step 3: Creating the Webhook Receiver

Now, let's create the basic structure of our webhook receiver application. Create a file named `app.js` and open it in your text editor.

#### Importing Dependencies

At the top of `app.js`, add the following imports:

```javascript
const express = require('express');
const hookline = require('hookline');
```

#### Setting Up Express

Initialize an instance of the Express application:

```javascript
const app = express();
```

### Step 4: Configuring Hookline

Configure Hookline to handle incoming webhook requests. You will also set up a secret for signing and verifying webhooks.

Add the following code inside your `app.js` file:

```javascript
// Secret key for HMAC SHA256 signature verification
const SECRET_KEY = 'your-secret-key-here';

// Configure Hookline with the Express app and the secret key
hookline(app, { secret: SECRET_KEY });
```

### Step 5: Defining Webhook Routes

Next, define a route to handle incoming webhooks. For simplicity, we'll use `/webhook` as our webhook endpoint.

```javascript
app.post('/webhook', (req, res) => {
    // Extract the raw body of the request
    const requestBody = req.body;

    try {
        // Verify the signature using Hookline's verify method
        if (!hookline.verify(requestBody, SECRET_KEY)) {
            return res.status(401).send('Invalid webhook');
        }

        // Process your business logic here

        // Respond with a 200 status to indicate successful processing
        res.status(200).send('Webhook received successfully');

    } catch (error) {
        console.error(error);
        res.status(500).send('Internal Server Error');
    }
});
```

### Step 6: Running the Application

To run your application, add a script to your `package.json` file:

```json
"scripts": {
  "start": "node app.js"
}
```

Now you can start your server by running:

```bash
npm start
```

Your webhook receiver should now be listening on port 3000 (by default). You can test it using tools like Postman or cURL.

### Step 7: Testing the Webhook

To test, send a POST request to `http://localhost:3000/webhook` with a JSON body. For example:

```bash
curl -X POST http://localhost:3000/webhook \
     -H "Content-Type: application/json" \
     -d '{"event": "some_event", "data": {"key": "value"}}'
```

If the signature is valid, your server should respond with `Webhook received successfully`.

### Step 8: Handling Invalid Signatures

Ensure that you handle cases where the signature verification fails. In our example, we return a 401 status if the webhook is invalid.

```javascript
if (!hookline.verify(requestBody, SECRET_KEY)) {
    return res.status(401).send('Invalid webhook');
}
```
