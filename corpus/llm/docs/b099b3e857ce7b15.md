## Building a Simple Webhook Receiver with Hookline (Node.js)

Webhooks are a fantastic way for applications to automatically receive updates when events occur in another system. This tutorial will guide you through building a basic webhook receiver in Node.js, leveraging the power of the `Hookline` library for secure and reliable handling. We’ll cover installation, signature verification, and responding with a success status code.

**Prerequisites:**

*   Node.js and npm installed on your system.
*   Basic understanding of HTTP requests and responses.

**Step 1: Project Setup & Installation**

1.  **Create a Project Directory:** Create a new directory for your project, let’s call it `webhook-receiver`. Navigate into this directory in your terminal.

2.  **Initialize npm:** Run `npm init -y` to create a default `package.json` file that manages your project's dependencies.

3.  **Install Hookline:** Install the `Hookline` library using npm:
    ```bash
    npm install hookline
    ```

4.  **Create `index.js`:** Create a file named `index.js` within your project directory, this will contain our webhook receiver logic.


**Step 2: Setting Up the Basic Receiver Structure**

Let’s start with the basic structure of our Node.js application using Express.js – a popular framework for building web applications and APIs in Node.js.  We'll use `Hookline` to handle incoming requests securely.

```javascript
const express = require('express');
const HookLine = require('hookline');

const app = express();
const port = process.env.PORT || 3000; // Use environment variable for port or default to 3000

// Create a new Hookline instance
const hookline = new HookLine({
  port: 8000, // You can change this to the port you want your webhook receiver to listen on.
  // This is crucial for security - keep this secret!
  secret: 'your-super-secret-key'
});

app.use(express.json()); // Middleware to parse JSON request bodies

hookline.start((err, server) => {
  if (err) {
    console.error('Error starting HookLine:', err);
    process.exit(1);
  }
  console.log(`HookLine server listening on port ${server.address().port}`);
});


// Example route to handle the webhook request (optional, but good practice)
app.get('/webhook', (req, res) => {
  if (req.headers['x-signature']) {
    hookline.verify(req); // Verify the signature using HookLine's built in mechanisms
    res.status(200).send('Webhook request received and verified!');
  } else {
    res.status(400).send('Missing X-Signature header');
  }
});
```

**Explanation:**

*   **`express` & `HookLine` Imports:** We import the necessary modules from Express and Hookline.
*   **Express Setup:** We create an Express app, define a port (using an environment variable for flexibility), and include middleware to parse JSON request bodies using `express.json()`.
*   **`HookLine` Instance:** We instantiate a new `HookLine` object, providing the secret key – **this is absolutely critical for security**.  Never expose this secret publicly! The port number also needs to be configured here.
*   **`hookline.start()`:** This starts the HookLine server and listens for incoming requests on the specified port.  The callback handles errors gracefully.
*   **`/webhook` Route (Example):** An example route is provided, but it's not strictly necessary if you are relying entirely on `HookLine` to handle the request. It’s included here to demonstrate how you might handle a specific event or respond with custom data.

**Step 3: Signature Verification**

The core of `Hookline`’s security is its signature verification mechanism.  This ensures that the incoming webhook requests are genuinely from the expected source, preventing malicious actors from injecting fake events.

*   **`hookline.verify(req)`:**  This function does the heavy lifting. It performs these steps:
    1.  **Header Check:** Verifies the presence of the `X-Signature` header in the incoming request. If it's missing, a 400 status code is returned.
    2.  **Hashing:**  `Hookline` calculates a hash based on the entire HTTP request body (including headers and data) using a secret key that *you* define when creating the `HookLine` instance.
    3.  **Signature Generation:** It then generates a signature from this hash, signing it with a cryptographic algorithm (typically HMAC-SHA256).
    4.  **Verification:** Finally, it compares the signature in the `X-Signature` header with the one generated by `HookLine`. If they match, the signature is verified, and the request is processed.

**Step 4: Responding to Webhooks**

In this simple example, we respond with a 200 status code confirming that the webhook request was received and verified.  More complex applications will likely process the data from the incoming event and perform actions accordingly.


**Running Your Receiver:**

1.  **Start your application:** Run `node index.js` in your terminal.
2.  **Configure your sending application:** The application that sends webhook events needs to be configured to send requests to `http://localhost:8000/webhook`. (Adjust the port if you changed it).

**Important Security Considerations:**

*   **Secret Key:** *Never* expose your secret key in client-side code or public locations. Store it securely in environment variables.
*   **HTTPS:**  Always use HTTPS for your webhook receiver to encrypt communication between the sending application and your server, preventing eavesdropping and tampering.
*   **Rate Limiting:** Implement rate limiting to prevent abuse and denial-of-service attacks. `Hookline` offers built-in rate limiting features that can be configured.

This tutorial provides a basic foundation for building a webhook receiver with Hookline in Node.js. Remember to prioritize security best practices and expand upon this example to meet the specific needs of your application.
