**Ratchet Task Queue API Reference**

Welcome to the Ratchet Task Queue API documentation! This guide provides detailed information on how to interact with our service using RESTful endpoints. We cover enqueuing a job, checking its status, and canceling a pending job, complete with example cURL requests and JSON responses.

---

### 1. Enqueueing a Job

To start processing tasks asynchronously, you need to enqueue them into the Ratchet queue. This is done using the `POST /jobs` endpoint.

**Endpoint:**  
`POST https://api.ratchet.example.com/v1/jobs`

**Request Headers:**
- `Content-Type: application/json`
- `Authorization: Bearer YOUR_ACCESS_TOKEN`

**Request Body (JSON):**
```json
{
  "name": "example_job",
  "data": {
    "description": "This is a sample job.",
    "priority": "high"
  }
}
```

**Example cURL Request:**
```bash
curl -X POST https://api.ratchet.example.com/v1/jobs \
-H "Content-Type: application/json" \
-H "Authorization: Bearer YOUR_ACCESS_TOKEN" \
-d '{
  "name": "example_job",
  "data": {
    "description": "This is a sample job.",
    "priority": "high"
  }
}'
```

**Successful Response (Status Code: 201 Created):**
```json
{
  "job_id": "123e4567-e89b-12d3-a456-426614174000",
  "status": "queued",
  "message": "Job has been successfully enqueued."
}
```

---

### 2. Checking Job Status

To monitor the progress of a job, use the `GET /jobs/{job_id}` endpoint. This allows you to retrieve detailed information about the current status and any associated data.

**Endpoint:**  
`GET https://api.ratchet.example.com/v1/jobs/{job_id}`

**Request Headers:**
- `Authorization: Bearer YOUR_ACCESS_TOKEN`

**Example cURL Request:**
```bash
curl -X GET https://api.ratchet.example.com/v1/jobs/123e4567-e89b-12d3-a456-426614174000 \
-H "Authorization: Bearer YOUR_ACCESS_TOKEN"
```

**Possible Responses:**

- **Status Code: 200 OK (Job Found):**
```json
{
  "job_id": "123e4567-e89b-12d3-a456-426614174000",
  "status": "processing",
  "data": {
    "description": "This is a sample job.",
    "priority": "high"
  },
  "progress": 45,
  "created_at": "2023-10-01T12:00:00Z",
  "updated_at": "2023-10-05T14:30:00Z"
}
```

- **Status Code: 404 Not Found (Job Not Available):**
```json
{
  "error": "Job not found or does not exist."
}
```

---

### 3. Canceling a Pending Job

If you need to cancel a job that is still in the `queued` or `processing` state, use the `DELETE /jobs/{job_id}` endpoint.

**Endpoint:**  
`DELETE https://api.ratchet.example.com/v1/jobs/{job_id}`

**Request Headers:**
- `Authorization: Bearer YOUR_ACCESS_TOKEN`

**Example cURL Request:**
```bash
curl -X DELETE https://api.ratchet.example.com/v1/jobs/123e4567-e89b-12d3-a456-426614174000 \
-H "Authorization: Bearer YOUR_ACCESS_TOKEN"
```

**Possible Responses:**

- **Status Code: 200 OK (Job Cancelled):**
```json
{
  "message": "Job has been successfully cancelled."
}
```

- **Status Code: 404 Not Found (Job Already Processed or Completed):**
```json
{
  "error": "Job cannot be cancelled. It is either completed or already processed."
}
```

### Additional Notes

- **Authentication:** Ensure your access token (`YOUR_ACCESS_TOKEN`) is valid and has the necessary permissions to interact with job endpoints.
- **Error Handling:** If you encounter any errors, check the response body for detailed error messages that can guide you on how to resolve the issue.
- **Rate Limiting:** Be mindful of rate limits imposed by the API to avoid service disruptions. Excessive requests may be throttled or temporarily blocked.
