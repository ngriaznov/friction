## Ratchet Task Queue - REST API Reference

This document details the REST API endpoints for interacting with the Ratchet task queue service. Ratchet provides a robust and scalable solution for managing asynchronous tasks within your applications, allowing you to offload time-consuming operations and improve responsiveness. This documentation covers key functionalities including enqueueing jobs, retrieving job statuses, and canceling pending jobs.

**Base URL:** `https://ratchet.example.com/api/v1` (Replace `ratchet.example.com` with your actual Ratchet instance's address)

**Authentication:** All API requests require an API key passed in the `X-API-Key` header.  You will receive this key upon registering a new application with Ratchet. 

**JSON Format:** All requests and responses utilize JSON for data exchange.

---

### 1. Enqueueing a Job (POST /jobs)

This endpoint allows you to add a task to the queue for processing by one of Ratchet’s worker processes.

**Endpoint:** `https://ratchet.example.com/api/v1/jobs`

**Method:** POST

**Request Headers:**
*   `Content-Type: application/json`
*   `X-API-Key: YOUR_API_KEY`

**Request Body (JSON):**

```json
{
  "job_type": "email_sender", // Required. Name of the task handler. Must match a registered handler.
  "payload": { 
    "to_email": "recipient@example.com",
    "subject": "Hello from Ratchet!",
    "body": "This is a test email sent via the Ratchet queue."
  },
  "options": { // Optional configuration for the job handler
     "priority": "high", 
     "timeout": 60 // Timeout in seconds. Defaults to 30 if not specified.
  }
}
```

**Description of Fields:**

*   `job_type`: (String, Required) The name of the task handler that will process this job. This must correspond to a registered handler within your Ratchet configuration. Examples include "email_sender", “image_processor”, “data_loader”.  Consult your Ratchet documentation for available handlers.
*   `payload`: (JSON, Required) A JSON object containing the data needed by the task handler. The structure of this object depends on the specific `job_type`.
*   `options`: (JSON, Optional) A JSON object providing configuration options to the task handler. This may include priority levels, timeout values, and other task-specific settings.

**Successful Response (201 Created):**

```json
{
  "id": "a1b2c3d4e5f6", 
  "status": "pending", 
  "created_at": "2023-10-27T10:00:00Z", 
  "updated_at": "2023-10-27T10:00:00Z",
  "job_type": "email_sender",
  "queue_name": "default" // The queue the job was added to.
}
```

**Error Responses:**

*   `400 Bad Request`: Invalid request body format or missing required fields.  The response body will contain details about the error.
    Example: `{"error":"Missing job_type field."}`
*   `401 Unauthorized`: Invalid API key.


### 2. Checking Job Status (GET /jobs/{job_id})

This endpoint allows you to retrieve the current status of a specific job.

**Endpoint:** `https://ratchet.example.com/api/v1/jobs/{job_id}`

**Method:** GET

**Request Headers:**
*   `X-API-Key: YOUR_API_KEY`

**Parameters:**

*   `job_id`: (String, Required) The unique identifier of the job to retrieve.

**Successful Response (200 OK):**

```json
{
  "id": "a1b2c3d4e5f6", 
  "status": "completed", // Possible values: pending, completed, failed, cancelled
  "created_at": "2023-10-27T10:00:00Z", 
  "updated_at": "2023-10-27T10:05:00Z",
  "job_type": "email_sender",
  "payload": { 
    "to_email": "recipient@example.com",
    "subject": "Hello from Ratchet!",
    "body": "This is a test email sent via the Ratchet queue."
  },
  "options": {
     "priority": "high", 
     "timeout": 60 
  },
  "result": "Email successfully sent!" // Only present if status is 'completed'
}
```

**Error Responses:**

*   `401 Unauthorized`: Invalid API key.
*   `404 Not Found`: Job with the specified ID does not exist.
    Example: `{"error":"Job with id a1b2c3d4e5f6 not found."}`



### 3. Cancelling a Pending Job (DELETE /jobs/{job_id})

This endpoint allows you to cancel a job that is currently in the “pending” state.  Attempting to cancel a completed or failed job will have no effect.

**Endpoint:** `https://ratchet.example.com/api/v1/jobs/{job_id}`

**Method:** DELETE

**Request Headers:**
*   `X-API-Key: YOUR_API_KEY`

**Parameters:**

*   `job_id`: (String, Required) The unique identifier of the job to cancel.

**Successful Response (204 No Content):**

No response body is returned upon successful cancellation.  The status field of the job will be updated to 'cancelled'.

**Error Responses:**

*   `401 Unauthorized`: Invalid API key.
*   `404 Not Found`: Job with the specified ID does not exist.
    Example: `{"error":"Job with id a1b2c3d4e5f6 not found."}`
*   `409 Conflict`: The job is already in a completed or failed state and cannot be cancelled.
