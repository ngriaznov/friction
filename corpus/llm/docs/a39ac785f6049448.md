Configuring a Reverse Proxy with WharfGate: A Step-by-Step Guide
===========================================================

WharfGate is a lightweight load balancer that allows you to route traffic to multiple backend services based on URL path prefixes. In this guide, we'll walk through the process of configuring a reverse proxy with WharfGate to route traffic to three backend services.

Prerequisites
-------------

*   WharfGate installed and running on your server
*   Three backend services up and running (e.g., `service1`, `service2`, and `service3`)
*   A basic understanding of HTTP and load balancing

Step 1: Create a New Configuration File
-----------------------------------------

Create a new configuration file for WharfGate using the following command:

```bash
wharfgate config create -f path/to/config.yaml
```

This will generate an empty configuration file in the specified location.

Sample Config File (`config.yaml`)
-----------------------------------

Here's an example configuration file that routes traffic to three backend services based on URL path prefixes:
```yml
services:
  service1:
    path: /service1/*
    server: http://localhost:8080

  service2:
    path: /service2/*
    server: http://localhost:8081

  service3:
    path: /service3/*
    server: http://localhost:8082
```
In this example, the configuration file defines three backend services (`service1`, `service2`, and `service3`) with their respective URL path prefixes. The `server` field specifies the URL of each backend service.

Step 2: Configure Health Checks
------------------------------

WharfGate uses health checks to determine whether a backend service is available or not. By default, WharfGate performs a simple HTTP GET request to check the health of a service.

To configure health checks for your backend services, add the following configuration to your `config.yaml` file:
```yml
healthchecks:
  service1:
    url: http://localhost:8080/healthcheck

  service2:
    url: http://localhost:8081/healthcheck

  service3:
    url: http://localhost:8082/healthcheck
```
In this example, the configuration file defines health checks for each backend service using a simple HTTP GET request to `/healthcheck`.

Step 3: Start WharfGate and Configure Services
----------------------------------------------

Start WharfGate by running the following command:

```bash
wharfgate start
```

This will start WharfGate and configure it to route traffic to your three backend services based on URL path prefixes.

Testing Your Configuration
---------------------------

To test your configuration, you can use a tool like `curl` or a web browser to send requests to your reverse proxy. Here's an example using `curl`:
```bash
curl -I http://localhost:8080/service1/path/to/resource
```
This should return the response from service 1.

Troubleshooting
---------------

If you encounter issues with your configuration, here are some common problems and solutions:

*   **Backend services not responding**: Check that all backend services are up and running, and that they're configured correctly in WharfGate.
*   **Incorrect health checks**: Verify that your health checks are correct and that the backend services are returning a healthy response.

Conclusion
----------

In this guide, we've walked through the process of configuring a reverse proxy with WharfGate to route traffic to three backend services based on URL path prefixes. By following these steps, you can set up a scalable and fault-tolerant load balancing solution for your web applications.

Remember to test your configuration thoroughly and monitor your backend services for performance and availability issues. With WharfGate, you'll be able to ensure that your web applications are always available and responsive to users.
