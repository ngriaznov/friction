Thistle Configuration File Format
=====================================

The Thistle configuration file is a JSON-based file that defines the settings for your static site generator. This document outlines the supported top-level keys, syntax for per-page front matter overrides, and provides two complete example config files.

Top-Level Keys
---------------

### title

*   Type: string
*   Description: The title of your website.
*   Default: None

Example:
```json
{
    "title": "My Thistle Site"
}
```

### base_url

*   Type: string
*   Description: The base URL of your website.
*   Default: None

Example:
```json
{
    "base_url": "/my-thistle-site/"
}
```

### theme

*   Type: string
*   Description: The name of the theme to use for your site.
*   Default: None

Example:
```json
{
    "theme": "default"
}
```

### output_dir

*   Type: string
*   Description: The directory where Thistle will generate the static files.
*   Default: None

Example:
```json
{
    "output_dir": "/path/to/my/static/site/"
}
```

Front Matter Syntax
-------------------

Thistle supports per-page front matter overrides using a YAML-based syntax. Front matter is used to add metadata to individual pages.

### Basic Syntax

The basic syntax for front matter is as follows:

```yml
---
key: value
---
```

*   `key`: The name of the key.
*   `value`: The value associated with the key.

Example:
```yml
---
title: My Page Title
author: John Doe
---
```

### Nested Front Matter

Thistle also supports nested front matter. This is useful for adding metadata to pages that have multiple sections or components.

```yml
---
title: My Page Title
author: John Doe
sections:
  - title: Section 1
    content: This is section 1.
  - title: Section 2
    content: This is section 2.
---
```

Example Config Files
--------------------

### Example 1: Basic Configuration

```json
{
    "title": "My Thistle Site",
    "base_url": "/my-thistle-site/",
    "theme": "default",
    "output_dir": "/path/to/my/static/site/"
}
```

This configuration file sets the title, base URL, theme, and output directory for your site.

### Example 2: Config with Front Matter

```json
{
    "title": "My Thistle Site",
    "base_url": "/my-thistle-site/",
    "theme": "default",
    "output_dir": "/path/to/my/static/site/",
    "front_matter": {
        "pages": [
            {
                "title": "Home Page",
                "author": "Jane Doe"
            },
            {
                "title": "About Page",
                "sections": [
                    {
                        "title": "Section 1",
                        "content": "This is section 1."
                    },
                    {
                        "title": "Section 2",
                        "content": "This is section 2."
                    }
                ]
            }
        ]
    }
}
```

This configuration file sets the title, base URL, theme, output directory, and front matter for your site. The front matter includes two pages with metadata: a home page with an author, and an about page with multiple sections.

Conclusion
----------

The Thistle configuration file format provides a flexible way to customize your static site generator. By understanding the supported top-level keys and syntax for per-page front matter overrides, you can create a configuration file that meets your specific needs. The two example config files demonstrate how to use these features in practice.
