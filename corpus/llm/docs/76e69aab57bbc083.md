## Thistle Configuration File Format Reference

This document details the configuration file format used by Thistle, a static site generator built for simplicity and flexibility. The configuration file, named `config.toml`, governs various aspects of your website's generation and deployment.

**File Location:**

The `config.toml` file should be placed at the root level of your Thistle project directory.

**General Syntax:**

Thistle uses TOML (Tom's Obvious, Minimal Language) for its configuration format. TOML is a human-readable data serialization language known for its simplicity and ease of use. 

**Key Features:**

* **Indentation:** Indentation is significant in TOML and determines the hierarchy of values. Use spaces or tabs consistently (spaces are recommended).
* **Comments:** Lines starting with `#` are treated as comments.

**Top-Level Keys:**

The following are the supported top-level keys within your `config.toml` file:

* **title:** (String) The title of your website, displayed in the browser tab and potentially on pages.
    ```toml
    title = "My Awesome Website"
    ```

* **base_url:** (String) The base URL of your website. This is crucial for generating relative links within your site. If you're deploying to a subdirectory (e.g., `yourdomain.com/my-site`), include it in the base URL:
    ```toml
    base_url = "/my-site" 
    ```

* **theme:** (String) The name of the theme you've selected for your website. Thistle supports various built-in themes and allows for custom theme development. 
    ```toml
    theme = "default" 
    ```

* **output_dir:** (String) Specifies the directory where generated HTML files will be saved. By default, this is set to `docs`.  
    ```toml
    output_dir = "dist"
    ```

**Per-Page Front Matter Overrides:**

Each Markdown file in your content directory can have its own front matter section. This allows for page-specific overrides of the settings defined in your main `config.toml` file.

Front matter is enclosed within YAML-style delimiters (`---`) at the beginning of a Markdown file:

```yaml
title: My Page Title
description: A brief description of this page
author: John Doe
theme: dark  # Overrides the global theme setting 
```

These overrides will take precedence over the values in `config.toml` for the specific page.


**Example Configuration Files:**

 **Example 1: Basic Configuration**

```toml
title = "My Personal Blog"
base_url = "/"
theme = "classic"
output_dir = "public"
```

**Explanation:**

* Sets the website title to "My Personal Blog."
* Defines the base URL as the root (`/`).
* Uses the "classic" theme.
* Generates HTML files in the "public" directory.

 **Example 2: With Per-Page Overrides**

```toml
title = "Travel Adventures"
base_url = "/travels/"
theme = "minimal"


# content/about.md - Example front matter override
---
title: About Me
description: Learn more about the person behind this blog.
author: Jane Doe
theme: light  
---

```

**Explanation:**

* Sets the website title to "Travel Adventures," a base URL of `/travels/`, and uses the "minimal" theme. 
* The `about.md` file overrides the global theme with "light".



This reference guide provides a foundation for understanding Thistle's configuration file format. For more advanced customization options and details about specific keys, refer to the comprehensive Thistle documentation available online.
