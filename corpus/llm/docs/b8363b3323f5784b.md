Setting Up Automatic Database Backups for Your Self-Hosted Note-Taking App, Corkboard
=====================================================================================

Corkboard is a powerful self-hosted note-taking app that allows you to organize your thoughts, ideas, and projects in one place. However, as with any valuable data, it's essential to protect it from loss or corruption. In this guide, we'll show you how to set up automatic database backups for Corkboard using cron scheduling.

**Step 1: Determine Your Backup Storage Location**

To store your backup files, you have several options:

*   **Local Hard Drive**: Store your backups on a local hard drive connected to your server or computer. This option is easy and requires minimal setup.
*   **Cloud Storage**: Use cloud storage services like Google Drive, Dropbox, or Amazon S3 to store your backups remotely. This option provides an additional layer of security and allows you to access your backups from anywhere.
*   **External Disk**: Store your backups on an external disk connected to your server or computer. This option is more secure than local hard drives but may require additional setup.

For this example, we'll use a local hard drive as our backup storage location.

**Step 2: Configure Cron Scheduling**

Cron scheduling allows you to automate tasks, such as backing up your database. To set up cron scheduling for Corkboard, follow these steps:

*   Log in to your server or computer and access the terminal.
*   Open the `crontab` file using the command `crontab -e`.
*   Add a new line at the end of the file with the following format: `minute hour day month day_of_week command`

For example, to backup your database every Sunday at 2 AM, you would add the following line:

```
0 2 * * SUN /usr/bin/bash -c "/path/to/corkboard/script/backup.sh"
```

**Step 3: Create a Backup Script**

To create a backup script for Corkboard, follow these steps:

*   Create a new file called `backup.sh` in the directory where your Corkboard installation is located.
*   Open the `backup.sh` file using a text editor and add the following code:
    ```bash
#!/bin/bash

# Backup database
mysqldump -u $USER -p$PASSWORD -h localhost -d corkboard > /path/to/backup/backup.sql

# Compress backup file
gzip backup.sql

# Rename backup file
mv backup.sql.gz /path/to/backup/
```
    Replace `$USER`, `$PASSWORD`, `localhost`, and `/path/to/corkboard` with your actual database credentials, server IP address, and installation directory.

**Step 4: Make the Backup Script Executable**

To make the backup script executable, follow these steps:

*   Open a terminal and navigate to the directory where you created the `backup.sh` file.
*   Run the command `chmod +x backup.sh` to make the file executable.

**Step 5: Verify Your Backup Configuration**

To verify that your backup configuration is working correctly, follow these steps:

*   Log in to your server or computer and access the terminal.
*   Open a new terminal window and run the following command: `crontab -l`
*   Check the crontab file for the new entry you added earlier.
*   Run the following command to verify that the backup script has executed successfully: `ls /path/to/backup/`

**Step 6: Restore from a Backup File**

To restore your database from a backup file, follow these steps:

*   Download the compressed backup file from the designated backup storage location.
*   Extract the compressed file using the command `gunzip backup.sql.gz`.
*   Run the following command to import the restored database into your Corkboard installation: `mysql -u $USER -p$PASSWORD -h localhost < /path/to/backup/backup.sql`

For this example, you would run the following command:

```
mysql -u $USER -p$PASSWORD -h localhost < /path/to/backup/backup.sql
```

Replace `$USER`, `$PASSWORD`, `localhost`, and `/path/to/corkboard` with your actual database credentials, server IP address, and installation directory.

Conclusion
----------

Automatic database backups for Corkboard are an essential security measure to protect your valuable data. By following the steps outlined in this guide, you can set up a reliable backup system using cron scheduling and store your backups on a local hard drive or cloud storage service. Remember to verify that your backup configuration is working correctly and to regularly restore from your backups in case of any issues.
