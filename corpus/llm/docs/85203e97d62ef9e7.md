**Building a Simple Webhook Receiver with Node.js and Hookline**

In this tutorial, we'll walk through building a simple webhook receiver in Node.js using the Hookline middleware library. We'll cover installing the package, setting up the server, verifying incoming requests, and responding to successful verification.

### Prerequisites

Before starting, make sure you have the following:

* Node.js (version 14 or higher) installed on your system
* A code editor or IDE of your choice
* Basic understanding of Node.js and web development concepts

### Step 1: Install Hookline

To get started, we need to install the Hookline middleware library. Run the following command in your terminal:
```bash
npm install hookline
```
This will install the `hookline` package and its dependencies.

### Step 2: Create a New Project

Create a new directory for your project and navigate into it using your terminal or command prompt. Initialize a new Node.js project by running:
```bash
npm init -y
```
This will create a `package.json` file in your project directory.

### Step 3: Set Up the Server

Create a new file called `app.js` and add the following code to set up an Express server:
```javascript
const express = require('express');
const app = express();

app.use(express.json());

const port = 3000;

app.listen(port, () => {
  console.log(`Server listening on port ${port}`);
});
```
This sets up a basic Express server that listens for incoming requests.

### Step 4: Set Up Hookline

Create a new file called `hook.js` and add the following code to set up Hookline:
```javascript
const hook = require('hookline');

// Replace with your public key
const publicKey = 'your-public-key';

// Set up the webhook endpoint
app.post('/webhook', hook({
  secret: publicKey,
  verifyOptions: {
    algorithm: 'HS256',
  },
}), (req, res) => {
  // Handle incoming request
});
```
This sets up a new webhook endpoint at `/webhook` that uses Hookline to verify incoming requests. Replace `your-public-key` with your actual public key.

### Step 5: Verify Incoming Requests

In the previous step, we set up an endpoint for incoming webhooks using Hookline. Now, let's walk through how verification works:

1. The client (e.g., a third-party service) sends a request to our `/webhook` endpoint.
2. Hookline receives the request and checks if it contains a valid `X-Hook-Signature` header.
3. If the signature is valid, Hookline verifies the incoming request by calculating its own signature using the provided secret key (public key in this case).
4. If both signatures match, the incoming request is considered legitimate.

To verify an incoming request, we can access the `req.hookline` object and check if verification was successful:
```javascript
app.post('/webhook', hook({
  // ...
}), (req, res) => {
  const { verified } = req.hookline;

  if (!verified) {
    return res.status(401).send('Invalid signature');
  }

  console.log('Incoming request verified!');

  // Handle the incoming request
});
```
In this example, we check if `req.hookline.verified` is truthy. If it's not, we respond with a 401 status code and an error message.

### Step 6: Respond to Successful Verification

If verification is successful, we can proceed with handling the incoming request. In our case, we'll simply send a response back to the client:
```javascript
app.post('/webhook', hook({
  // ...
}), (req, res) => {
  const { verified } = req.hookline;

  if (!verified) {
    return res.status(401).send('Invalid signature');
  }

  console.log('Incoming request verified!');

  res.status(200).send('Webhook received successfully!');
});
```
In this example, we send a response with a 200 status code and a success message.

### Conclusion

Congratulations! You've just built a simple webhook receiver in Node.js using Hookline. This tutorial walked through setting up an Express server, installing Hookline, verifying incoming requests, and responding to successful verification.

Remember to replace `your-public-key` with your actual public key and adjust the verification options according to your needs. With this setup, you can now receive webhooks from clients and verify their legitimacy using Hookline's built-in functionality.
