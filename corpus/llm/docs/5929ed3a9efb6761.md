## Thistle Static Site Generator Configuration File

This document outlines the configuration file format used by the Thistle static site generator. The configuration file allows you to customize various aspects of your generated website, including its title, base URL, theme, output directory, and individual page settings.  Thistle utilizes a YAML-based configuration system for readability and ease of modification.

**Top-Level Keys:**

The core configuration consists of the following top-level keys:

* **`title` (string):** This key defines the overall title of your website. It’s used in the `<head>` section of each page, as well as for any global branding within the theme.  This is a mandatory field.
    * Example: `"My Awesome Blog"`

* **`base_url` (string):** Specifies the root URL of your site. This is crucial for generating correct links and absolute URLs in generated assets. It’s recommended to use a full, protocol-relative URL (e.g., `https://example.com`).
    * Example: `"https://www.mywebsite.com"`

* **`theme` (string):**  This key designates the Thistle theme you want to utilize for your website. Supported themes are defined within the Thistle installation and typically include options like light/dark mode, font choices, and overall styling.  The default theme is ‘minimal’.
    * Example: `"dark"` or `"classic"`

* **`output_dir` (string):**  Determines where the generated static site files will be placed after processing. The default value is `public`. You can override this to suit your project's organization.
   * Example: `"docs"` 


**Per-Page Front Matter Overrides:**

Each page in your Thistle site has a front matter section, which allows you to customize settings on a per-page basis.  Front matter is defined within the YAML block at the beginning of each `.md` or `.njk` file (depending on the template engine you’re using). 

The syntax for overriding configuration keys in front matter is straightforward: simply use the key name followed by an equals sign (`=`) and then the desired value.  If a top-level key is not defined in the main configuration, it will be overridden by its corresponding entry within the front matter of a specific page. 

**Example:**

```yaml
---
title: "My Awesome Blog Post"
description: "A detailed explanation of something interesting."
featured_image: "/images/blog-post-image.jpg"
tags: ["blog", "technology"]
---
```

Here, the `title` and `description` fields for this specific page would override the global title and description defined in the main configuration file.  Note that front matter keys can be used to define any setting supported by Thistle (e.g., font size, color schemes).


**Example Configuration Files:**

**1. Basic Configuration:**

```yaml
title: "My Simple Website"
base_url: "https://example.com/my-site/"
theme: "minimal"
output_dir: "static"
```

This configuration will generate a static site with the title "My Simple Website", using the 'minimal' theme, and placing all generated files in the `static` directory.



**2. Advanced Configuration:**

```yaml
title: "Company Blog - Thistle Demo"
base_url: "https://companyblog.example.com/"
theme: "dark"
output_dir: "docs"
page_metadata: # A custom top-level key to add page-specific metadata
  og_title: "Thistle Demo Blog"
  og_description: "This is a demonstration of the Thistle static site generator."

blog_posts: # Demonstrates overriding a theme setting per-page
  - title: "Introduction to Thistle"
    description: "Learn about the basics of using Thistle."
    featured_image: "/images/thistle-logo.png"
    tags: ["static site generator", "Thistle"]
  - title: "Customizing Your Blog"
    description: "Explore advanced customization options with Thistle."
    featured_image: "/images/customize.jpg"
    tags: ["theme", "configuration"]
```

This more complex example demonstrates several features: a different `title` and `base_url`, the use of the `dark` theme, placing output in the `docs` directory,  and the addition of a custom top-level key - `page_metadata`. It also illustrates how to override settings within front matter for specific pages (e.g., using `blog_posts`).

**Important Considerations:**

* **YAML Syntax:**  Ensure your configuration file adheres to YAML syntax rules, including proper indentation and quotation marks.
* **Key Naming:** Key names are case-sensitive.
* **Conflict Resolution:** If a key is defined in both the top-level configuration and within page front matter, the value from the front matter takes precedence.

By understanding these configurations, you can effectively tailor Thistle to your specific needs and create a stunning static website.  For more detailed information, please refer to the official Thistle documentation.
