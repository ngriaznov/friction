# Reference Page: Configuration File Format for Thistle Static Site Generator

Thistle is a static site generator designed to streamline the process of creating and managing websites with minimal setup and configuration overhead. This reference page documents the format and key elements of Thistle's configuration file, which is typically named `thistle.yml` or `config.yml`.

## Top-Level Keys in Configuration File

The top-level keys are essential for defining global settings that apply to all pages generated by Thistle. These include:

1. **title**: A string representing the title of your site.
2. **base_url**: The base URL used for linking purposes, such as relative URLs and anchor tags.
3. **theme**: Specifies the theme or style sheet directory path where custom CSS is located.
4. **output_dir**: Defines the output directory where static files are saved after generation.

### Example 1: Basic Configuration
```yaml
title: My Awesome Site
base_url: https://example.com/
theme: themes/my-custom-theme
output_dir: ./public
```

### Example 2: Advanced Configuration with Themes and Custom Settings
```yaml
title: Thistle Documentation Site
base_url: http://docs.thistle.com/
theme: themes/documentation
output_dir: ./published-docs

# Additional global settings
enable_sitemap: true
sitemap_path: /sitemap.xml
```

## Per-Page Front Matter Overrides

In addition to the top-level keys, Thistle supports front matter in each Markdown file. This allows for page-specific overrides of global configurations or additional metadata that can be used within templates.

### Syntax for Per-Page Front Matter
Front matter is placed at the beginning of a Markdown file, surrounded by three dashes (`---`). The syntax follows:

```yaml
---
key: value
another_key: another_value
---
```

### Example Usage in a Markdown File

Consider a scenario where you have several pages but one needs specific adjustments. Here’s how to override settings for such a page.

#### Default Configuration (thistle.yml)
```yaml
title: Thistle Documentation Site
base_url: http://docs.thistle.com/
theme: themes/documentation
output_dir: ./published-docs
```

#### Custom Page with Front Matter Overrides (page.md)
```markdown
---
title: Introduction to Thistle
base_url: https://thistle.io/docs/introduction
---

# Introduction to Thistle

Welcome to the documentation for Thistle. This page is specifically configured to have a different title and base URL.
```

### Full Example Configuration File with Overrides

Here’s an example where multiple pages have varying configurations, some of which are overridden in specific files.

#### thistle.yml (Global Settings)
```yaml
title: My Awesome Site
base_url: https://example.com/
theme: themes/my-custom-theme
output_dir: ./public

# Additional global settings
enable_sitemap: true
sitemap_path: /sitemap.xml
```

#### page1.md (Default Configuration)
```markdown
---
title: Home Page
---
<h1>Welcome to the Home Page</h1>
<p>This is the default home page for your site.</p>
```

#### page2.md (Page-Specific Overrides)
```markdown
---
title: About Us
base_url: https://example.com/about-us
theme: themes/classic
output_dir: ./custom-output
---
<h1>About Us</h1>
<p>This is the about us page, with its own specific configuration.</p>
```

### Summary

Thistle’s configuration file provides a flexible and powerful way to manage global settings while allowing for per-page customization through front matter. By leveraging these features, you can tailor your static site generation process to fit various project requirements efficiently.

This document serves as a comprehensive guide for setting up Thistle and customizing its behavior according to the specific needs of different pages within a website.
