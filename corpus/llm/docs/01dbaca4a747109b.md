# Getting Started Guide: Dedupe-Shots - Removing Duplicate Photos with Perceptual Hashing

Welcome to the getting-started guide for `dedupe-shots`, a powerful command-line tool designed to help you identify and remove duplicate photos from your local photo library using perceptual hashing. This guide will walk you through installing `dedupe-shots`, performing a basic scan, and previewing matches before making any deletions.

## Prerequisites

Before diving into the installation process, ensure that your system meets the following requirements:

- **Operating System**: `dedupe-shots` is compatible with Linux, macOS, and Windows (using WSL).
- **Python Version**: Python 3.8 or higher.
- **Dependencies**: `Pillow`, `imagehash`, and `click`.

## Installation

To install `dedupe-shots`, follow these steps:

1. **Clone the Repository**:
   ```bash
   git clone https://github.com/alibaba/dedupe-shots.git
   cd dedupe-shots
   ```

2. **Install Dependencies**:
   ```bash
   pip install -r requirements.txt
   ```

3. **Run the Tool**:
   You can now run `dedupe-shots` from the command line.
   ```bash
   python -m dedupe_shots
   ```

Alternatively, you can use a virtual environment to manage dependencies:

1. **Create and Activate a Virtual Environment** (optional but recommended):
   ```bash
   python3 -m venv venv
   source venv/bin/activate  # On Windows: venv\Scripts\activate
   ```

2. **Install Dependencies in the Virtual Environment**:
   ```bash
   pip install -r requirements.txt
   ```

3. **Run the Tool**:
   ```bash
   python -m dedupe_shots
   ```

## Basic Scan Command

To start scanning your photo library for duplicates, use the following command:

```bash
dedupe-shots scan /path/to/your/photo/library
```

Replace `/path/to/your/photo/library` with the actual path to your photo directory. This command will generate a database of perceptual hashes and store it in a temporary location.

### Example Command

If you have photos stored in `~/Photos`, run:

```bash
dedupe-shots scan ~/Photos
```

## Preview Matches Before Deleting

After scanning, `dedupe-shots` provides an option to preview matches before making any deletions. This is crucial for ensuring that only duplicates are removed.

To preview matches, use the following command after a successful scan:

```bash
dedupe-shots preview /path/to/your/photo/library
```

Replace `/path/to/your/photo/library` with the path you used during the initial `scan` command.

### Example Command

If your scan was performed on `~/Photos`, run:

```bash
dedupe-shots preview ~/Photos
```

This will display a list of potential duplicates, allowing you to review and confirm which files should be deleted. You can also specify a threshold for similarity if needed:

```bash
dedupe-shots preview --threshold 0.95 /path/to/your/photo/library
```

### Reviewing Matches

When `preview` is run, it will show you pairs of images that are considered duplicates based on their perceptual hash values. You can review these matches and decide which ones to keep or delete.

## Deleting Duplicates

Once you have reviewed the matches and confirmed that they are indeed duplicates, you can proceed with deleting them using:

```bash
dedupe-shots clean /path/to/your/photo/library
```

Replace `/path/to/your/photo/library` with the path used during the initial `scan` command.

### Example Command

If your scan was performed on `~/Photos`, run:

```bash
dedupe-shots clean ~/Photos
```

This will delete the duplicates based on the matches you reviewed in the preview step. Ensure that you have backed up any important files before running this command to avoid accidental deletion of valuable photos.

## Conclusion

Congratulations! You've now set up and used `dedupe-shots` to identify and manage duplicate photos in your local library. By following these steps, you can keep your photo collection organized without losing any important images. If you encounter any issues or have further questions, feel free to reach out for support.
