## Setting Up Automatic Database Backups for Corkboard

Corkboard is a fantastic self-hosted note-taking application, but keeping your data safe requires proactive maintenance. This guide outlines how to set up automatic database backups for your Corkboard instance, ensuring you can restore your notes in case of any issues. 

**Understanding the Backup Process**

This setup utilizes cron scheduling – a time-based job scheduler – to automatically execute a backup script at regular intervals. We’ll be backing up the Corkboard database (typically SQLite) and storing the backups securely. 

**1. Prerequisites:**

* **Corkboard Installation:** You need a fully functional Corkboard installation, accessible via its web interface.
* **Command-Line Access:**  You'll need command-line access to your server where Corkboard is running (e.g., SSH).
* **Backup Script (Example):** We’ll create a simple bash script for backing up the database. Adapt this to fit your specific environment and desired backup frequency.

```bash
#!/bin/bash

# Configuration - ADJUST THESE!
DATABASE_PATH="/var/lib/corkboard/corkboard.db" # Path to your Corkboard database file
BACKUP_DIR="/mnt/backups/corkboard"          # Directory where backups will be stored
DATE=$(date +%Y-%m-%d_%H-%M-%S)              # Timestamp for backup filename

# Create Backup Directory if it doesn't exist
mkdir -p "$BACKUP_DIR"

# Backup the database using mysqldump (adapt command to your DB type)
mysqldump --user=corkboarduser --password='corkboardpassword' --all-databases > "$BACKUP_DIR/corkboard_$DATE.sql" 

# Optional: Verify backup size
# du -h "$BACKUP_DIR/corkboard_$DATE.sql"
```

**2. Setting Up the Backup Directory:**

* **Create the Directory:** Ensure the `BACKUP_DIR` exists and has appropriate permissions (read & write for the user running the script).  Consider using a dedicated storage location like an external drive or network share for redundancy. 
   ```bash
   mkdir -p /mnt/backups/corkboard
   chmod 755 /mnt/backups/corkboard # Read/Write for owner, read for group and others
   ```

**3. Scheduling the Backup with Cron:**

* **Edit Crontab:** Open your crontab file using the `crontab -e` command. This will open a text editor where you can add cron entries.

* **Add the Cron Entry:** Add a line to schedule the backup script, for example, to run daily at 2:00 AM:
   ```bash
   0 2 * * * /path/to/your/backup_script.sh > /dev/null 2>&1
   ```
   * **Explanation:**
     * `0 2 * * *`: This specifies the schedule (minute, hour, day of month, month, day of week).  In this case, it runs at 2:00 AM every day.
     * `/path/to/your/backup_script.sh`: Replace with the actual path to your backup script file.
     * `> /dev/null 2>&1`: This redirects standard output and standard error to the null device, preventing cron from sending you emails about the script's execution (unless there’s an error).

**4. Restoring from a Backup:**

To restore from a backup, use the following command:

```bash
mysql --user=corkboarduser --password='corkboardpassword' --all-databases < /mnt/backups/corkboard/corkboard_YYYYMMDD_HHMMSS.sql
```

* **Replace placeholders:**  `YYYYMMDD_HHMMSS` with the date and time of your backup file (e.g., `2023-10-27_14-30-00`).

**Important Considerations & Best Practices:**

* **Database Type:** The example script uses `mysqldump`. If you’re using a different database system, adjust the command accordingly.
* **Permissions:**  Ensure the user executing the backup script has sufficient permissions to read the database file and write to the backup directory.
* **Testing:** Regularly test your backups by restoring them to a temporary environment to confirm they are valid and complete.
* **Retention Policy:** Implement a retention policy – how many backups you keep – based on your data needs and storage capacity.  Consider deleting older backups automatically. 
* **Monitoring:** Monitor the backup script’s execution and ensure it's running successfully.



By following these steps, you can establish a robust automated backup system for your Corkboard instance, providing peace of mind and safeguarding your valuable note-taking data. Remember to customize the script and schedule according to your specific requirements.
