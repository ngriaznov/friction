## Secure Your Notes: A Guide to Automatic Backups for Corkboard

Corkboard is a powerful tool for organizing your thoughts and ideas, but like any digital asset, it requires protection against data loss. This guide will walk you through setting up automatic backups for your self-hosted Corkboard instance, ensuring your valuable notes are always safe and accessible. 

**Why Back Up?**

Imagine losing all your carefully crafted notes due to a hardware failure, accidental deletion, or even a cyberattack.  Regular backups provide a safety net, allowing you to quickly restore your data if disaster strikes. 

**Step-by-Step Guide:**

**1. Choose Your Backup Location:**

* **External Hard Drive:** A reliable and affordable option for storing large backups offline. Connect an external hard drive to your server and ensure it's physically separate from your main system.
* **Cloud Storage:** Services like Dropbox, Google Drive, or Amazon S3 offer secure and scalable storage solutions. Choose a plan that meets your backup size requirements.

**2. Utilize Cron Scheduling:**

Cron is a powerful scheduling tool built into most Linux systems. It allows you to automate tasks, including backups, at specific intervals. 

* **Open the crontab:**
    * On most systems, run `crontab -e` in the terminal. This will open your crontab file in an editor. 
* **Add a Backup Command:**

    Let's assume you want to back up your Corkboard database daily at 3 AM. Add the following line to your crontab:

```bash
0 3 * * * /path/to/your/backup_script.sh
```

  * `0 3`: Specifies the time (minutes and hours) for the backup to run. 
  * `* * *`: Indicates that the backup should occur every day of the week, month, and year.
  * `/path/to/your/backup_script.sh`: Replace this with the full path to your backup script file.

**3. Create a Backup Script:**

You'll need a shell script to perform the actual backing up process. Here's a basic example:

```bash
#!/bin/bash

# Set the database credentials
DATABASE_HOST="localhost"
DATABASE_USER="your_database_user"
DATABASE_PASSWORD="your_database_password"
DATABASE_NAME="corkboard_db"  # Replace with your Corkboard database name

# Create a timestamp for the backup file
BACKUP_DATE=$(date +%Y-%m-%d_%H-%M-%S)

# Backup the database using mysqldump (adjust for your database type)
mysqldump -u $DATABASE_USER -p$DATABASE_PASSWORD $DATABASE_NAME > /path/to/backup/corkboard_$BACKUP_DATE.sql

echo "Backup created: corkboard_$BACKUP_DATE.sql" 
```

* **Replace Placeholders:** Update the script with your actual database credentials and backup location. 
* **Customize Script:** Tailor the script to include additional options like compressing the backup file or sending notifications upon completion.


**4. Restore from a Backup File:**

If you need to restore your Corkboard data, follow these steps:

1. **Stop Corkboard:** Ensure the application is not running.
2. **Import the Database:** Use the appropriate database import tool for your system (e.g., `mysql` for MySQL) to import the backed-up SQL file into your active database. 
3. **Start Corkboard:** Restart your Corkboard application. Your notes should be restored.

**5. Regularly Test Backups:**


Periodically test your backup and restore process to ensure everything is working correctly. This will give you peace of mind knowing that your data is truly secure.



By following these steps, you'll establish a robust backup system for Corkboard, safeguarding your valuable notes and ensuring smooth recovery in case of unforeseen circumstances.
